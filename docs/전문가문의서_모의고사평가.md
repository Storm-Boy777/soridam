# OPIc 모의고사 평가 피드백 설계 — 전문가 자문 요청서

## 1. 목적

AI(GPT-4.1) 기반 OPIc 모의고사의 **평가 결과를 사용자에게 어떻게 보여줘야 학습에 가장 도움이 되는지**에 대한 전문가 평가를 요청합니다.

현재 시스템은 **개별 문항 평가**(14문항)와 **종합 진단 리포트**(1건)를 생성하며, GPT가 생성하는 데이터 자체는 풍부하나, **사용자에게 어떤 정보를 어떤 순서와 형태로 보여줘야 실제 학습 효과를 극대화할 수 있는지** 검증받고자 합니다.

---

## 2. 시스템 아키텍처 개요

### 2-1. 모의고사 평가 파이프라인

```
[사용자]  15문항 응시 (Q1 자기소개 + Q2~Q15 평가 대상 14문항)
    ↓
[Stage A]  음성 → 텍스트 변환 (Whisper STT) + 발음 평가 (Azure Speech)
    ↓
[Stage B]  개별 평가 — GPT-4.1이 문항별로 체크박스 통과 여부 판정 + 교정 + 심층 분석
    ↓
[Stage C]  종합 평가 — 규칙엔진(ACTFL 기반 등급 산출) + GPT-4.1 종합 진단 리포트
```

### 2-2. 두 가지 평가 유형

| 구분 | 개별 평가 (문항별) | 종합 평가 (전체) |
|------|-------------------|-----------------|
| **대상** | Q2~Q15 각 문항 | 14문항 전체를 종합 |
| **생성 주체** | GPT-4.1 (question_type별 전용 프롬프트) | 규칙엔진(등급) + GPT-4.1(리포트) |
| **목적** | 이 문항에서 잘한 점/못한 점/교정/연습 안내 | 전체 등급 진단, 패턴 분석, 훈련 로드맵 |
| **제공 시점** | 훈련 모드: 문항 직후 즉시 / 실전 모드: 시험 종료 후 | 전체 시험 종료 + 모든 개별 평가 완료 후 |

### 2-3. question_type (10가지)

| # | 유형 | 영어 | 체크박스 등급 |
|---|------|------|-------------|
| 1 | 묘사 | description | INT (18개) |
| 2 | 루틴 | routine | INT (18개) |
| 3 | 질문하기 | asking_questions | INT (18개) |
| 4 | 비교 | comparison | ADV (42개) |
| 5 | 특정 경험 | experience_specific | ADV (42개) |
| 6 | 습관 경험 | experience_habitual | ADV (42개) |
| 7 | 과거 경험 | experience_past | ADV (42개) |
| 8 | 대안제시 | suggest_alternatives | ADV (42개) |
| 9 | 비교변화 | comparison_change | AL (42개) |
| 10 | 사회적이슈 | social_issue | AL (42개) |

## 3. 개별 평가 — 프롬프트 구조 (평가 대상)

> 아래는 `eval_routine` (루틴 유형) 프롬프트의 핵심 구조입니다. 10가지 question_type마다 유사한 구조의 전용 프롬프트가 존재합니다.

### 3-1. GPT에게 요청하는 출력 항목

GPT-4.1에게 하나의 JSON으로 아래 4가지를 동시에 생성하도록 요청합니다:

#### (1) 체크박스 평가 (checkboxes)

question_type에 따라 INT(18개) / ADV(42개) / AL(42개) 체크박스를 판정합니다.

**평가 원칙: INTELLIGIBILITY (의사소통 가능성)**
- Pass: 오류가 있어도 의미가 전달되면 통과
- Fail: 의미 전달이 실패한 경우에만 불합격
- 문법 오류 자체가 아니라 "의사소통 단절 여부"로 판정

**INT 체크박스 예시 (18개):**

| 카테고리 | 코드 | 판정 기준 |
|---------|------|----------|
| Language Creation (GATE) | INT-1-1 | 개인 정보를 적절한 어휘로 제시 |
| | INT-1-2 | 질문의 의도를 정확히 이해 |
| | INT-1-3 | 문장 수준의 발화 생성 |
| Speaking in Sentences | INT-2-1 | 단어와 암기된 구문 사용 |
| | INT-2-2 | 일부 문장 생성 가능 |
| | INT-2-3 | 대부분 문장 수준 유지 |
| Intelligibility — Fluency | INT-4-F1~F5 | 말하기 속도, 쉼, 미완성, 재시작, 반복 |
| Intelligibility — Pronunciation | INT-4-P1~P4 | 조음, 음높이, 강세, 억양 |
| Intelligibility — Grammar | INT-4-G1~G2 | 단순 문법, 완전한 문장 |
| Intelligibility — Syntax | INT-4-S1 | 어순 |

각 체크박스에는 한국어 evidence(근거)를 함께 생성합니다.

#### (2) 문장 분할 (sentences)

학생의 답변(transcript)을 논리적 문장으로 분리하고, 의미 맥락별로 group 번호를 부여합니다.

```json
[
  { "index": 0, "text": "Okay, I'd like to talk about internet.", "group": 1 },
  { "index": 1, "text": "My friends and I use internet to find...", "group": 2 },
  { "index": 2, "text": "For me, I'm spending a lot of time...", "group": 3 },
  { "index": 3, "text": "Thanks to the internet, it makes our life...", "group": 4 }
]
```

#### (3) 교정 사항 (corrections)

문법/표현 오류별로 교정을 제공합니다:

```json
{
  "sentence_index": 0,
  "error_parts": ["about internet"],
  "tip_korean": "특정 명사 앞에는 정관사 'the'가 필요해요.",
  "corrected_segment": "about the internet."
}
```

- `error_parts`: 틀린 부분만 추출 (올바른 단어 포함 금지)
- `tip_korean`: **왜 틀렸는지만** 1문장으로 설명 (교정 내용은 포함하지 않음)
- `corrected_segment`: 수정된 부분만 (전체 문장 아님)

#### (4) 심층 분석 (deep_analysis) — 5섹션

OPIc 전문 튜터가 학생에게 1:1 피드백하는 톤으로 작성합니다.

| 섹션 | 목적 |
|------|------|
| overall_assessment | 이 문항에서 잘한 점 + 부족한 점 2~3문장 요약 |
| linguistic_analysis | 오류를 유형별로 분류 + L1 간섭 원인 분석 |
| communicative_effectiveness | 청자 관점에서 잘 전달된 내용 vs 흐려진 부분 |
| proficiency_gap | 현재 능력 → 목표 등급 도달에 필요한 능력 + 예시 문장 |
| recommendation | 이 답변의 오류에 직접 연결된 구체적 연습 안내 |

**금지 사항:** 체크박스 코드, pass/fail 결과, 등급 코드, 내부 메트릭(WPM 등), 평가 로직 언급

## 4. 종합 평가 — 프롬프트 구조 (평가 대상)

### 4-1. 규칙엔진 (GPT 전 단계)

GPT를 호출하기 전에, 14문항의 개별 평가 결과를 규칙엔진이 집계합니다:

- INT/ADV/AL 체크박스별 누적 통과율 산출
- ACTFL 기준 Floor/Ceiling 판정
- Sympathetic Listener 필요 여부 판정
- **최종 등급(final_level) 결정** — NH, IL, IM1, IM2, IM3, IH, AL
- FACT 점수 산출 (F: Fluency, A: Accuracy, C: Content, T: Task / 각 0~10점, 총 100점)

### 4-2. GPT에게 요청하는 출력 항목

규칙엔진 결과를 입력으로 받아 GPT가 아래를 생성합니다:

#### (1) 전문가 진단서 (overall_comments_ko)

ACTFL OPIc 공인 평가사가 학생에게 보내는 **정식 진단 리포트** 스타일. 8-12문장.

**구조:**
1. 현재 등급 요약 (1-2문장)
2. INT 수준 평가 상세 (2-3문장) — 어떤 과제를 어떻게 충족했는지
3. ADV 수준 평가 상세 (2-3문장) — 어디서 수행이 무너졌는지
4. Sympathetic Listener 분석 (1문장)
5. 전체적 언어 발달 단계 진단 (1-2문장)
6. 결론 및 등급 판정 근거 (1-2문장)

등급별로 미리 정의된 ACTFL 진단 표현을 기반으로 커스터마이징합니다. 예(IM2):

> "응시자는 Intermediate 수준에서 안정적인 수행을 보여주었으며, 중간 범위를 유지했습니다. [구체적 강점]. Advanced 수준의 과제에서는 수행이 무너졌으며, [구체적 약점] 능력이 부족했습니다. 동정적 청자가 필요했습니다."

#### (2) 종합 피드백 (comprehensive_feedback) — 5파트

| 파트 | 내용 | 분량 |
|------|------|------|
| overall_assessment | 등급 맥락, Floor/Ceiling 참조 | 3-4문장 |
| strengths_analysis | 체크박스 상위 통과 카테고리에서 도출한 강점 3가지 | 2-3문장 |
| weaknesses_analysis | 하위 통과 카테고리에서 도출한 약점 패턴 3가지 | 2-3문장 |
| gap_analysis | 현재 등급 → target_level 갭, Ceiling 상태 참조 | 2-3문장 |
| action_plan | 3가지 우선 액션 아이템, 오늘 당장 연습 가능한 수준 | 3-4문장 |

#### (3) 반복 실수 패턴 (recurring_mistakes) — 최대 5개

14문항을 교차 분석하여 3회 이상 반복되는 실수 패턴을 도출합니다.

```json
{
  "pattern_name_ko": "과거 시제 전환 불안정",
  "category": "grammar",
  "severity": "high",
  "frequency": 8,
  "affected_questions": [3, 5, 7, 10],
  "description_ko": "14문항 중 8문항에서 과거 시제를 써야 하는 상황에서 현재 시제로...",
  "example_wrong": "Yesterday I go to the store and buy some groceries.",
  "example_correct": "Yesterday I went to the store and bought some groceries.",
  "improvement_tip_ko": "과거 경험을 말할 때 동사의 과거형을 먼저 떠올리는 연습을..."
}
```

#### (4) 훈련 권장 (training_recommendations) — 3개

```json
{
  "priority": 1,
  "category_ko": "시제별 서사/묘사",
  "weak_areas_ko": ["과거 시제 서술이 불안정합니다"],
  "practice_tip": "과거 경험을 말할 때 'First, I went to... Then, I realized that...' 순서 표현을 사용해 보세요.",
  "recommended_type": "past_experience",
  "model_sentence": "When I was in college, I had an unforgettable experience...",
  "tutoring_connection": {
    "tutor_type": "dam_sam",
    "session_focus": "과거 시제 서사 집중 훈련",
    "expected_improvement": "3-4주 후 과거 시제 통제력 향상 기대"
  }
}
```

## 5. 실제 생성 결과물

### 5-1. 개별 평가 결과 예시 (실제 DB 데이터)

**질문**: 가족이나 친구의 인터넷 사용 습관에 대해 설명하세요 (routine/INT)

**학생 답변 (STT 원문):**

> Okay, I'd like to talk about internet. My friends and I, yeah, almost people use internet to find some information or just using for social networking service or, yeah, want to buy something through the web. For me, I'm spending a lot of time to watch YouTube, you know, tell us a lot of videos and sometimes I'm using for chat time and I can do a lot of things through the internet. Thanks to the internet, yeah, it makes our life more convenient.

**메타 정보:**
- 음성 길이: 66초 | WPM: 76 | 필러(yeah/you know): 2회 | 긴 침묵(3초+): 2회
- 발음 정확도: 55 | 유창성: 42 | 운율: 32.6

**체크박스 결과: 18/18 통과 (INT)**

| 코드 | 결과 | 근거 (한국어) |
|------|------|-------------|
| INT-1-1 | PASS | 인터넷, 소셜 네트워킹, 정보 검색 등 일상적 인터넷 사용에 대한 어휘를 적절히 사용함 |
| INT-1-2 | PASS | 질문에서 요구한 가족과 친구의 인터넷 사용 습관에 대해 답변함 |
| INT-1-3 | PASS | 단어 나열이 아닌 문장 단위로 자신의 생각을 표현함 |
| INT-4-F1 | PASS | 발화 속도가 다소 느리지만 의미 전달에 지장 없음 |
| INT-4-F2 | PASS | 긴 쉼이 있으나 전체 흐름이 끊기지 않음 |
| INT-4-G1 | PASS | 문법 오류가 있으나 의미 전달에 큰 장애 없음 |
| INT-4-P1 | PASS | 발음 정확도가 낮지만 단어 인지가 가능함 |
| ... | ... | (18개 모두 PASS) |

**문장 분할 결과:**

| # | 문장 | 그룹 |
|---|------|------|
| 0 | Okay, I'd like to talk about internet. | 1 (도입) |
| 1 | My friends and I, yeah, almost people use internet to find some information or just using for social networking service or, yeah, want to buy something through the web. | 2 (타인 습관) |
| 2 | For me, I'm spending a lot of time to watch YouTube, you know, tell us a lot of videos and sometimes I'm using for chat time and I can do a lot of things through the internet. | 3 (본인 습관) |
| 3 | Thanks to the internet, yeah, it makes our life more convenient. | 4 (결론) |

**교정 결과 (4건):**

| # | 오류 부분 | 교정 | 설명 |
|---|----------|------|------|
| 1 | about internet | about **the** internet. | 특정 명사 앞에는 정관사 'the'가 필요해요. |
| 2 | almost people, use internet, using for social networking service | almost **all** people use **the** internet... | 'almost'는 'all'과 함께 써야 하고, 동사 뒤에 목적어가 필요해요. |
| 3 | spending a lot of time to watch, tell us a lot of videos, using for chat time | I **spend** a lot of time **watching** YouTube, it **shows** us... | 'spend time' 뒤에는 동명사를 써야 하고, 'tell'의 주어가 불분명해요. |
| 4 | our life | our **lives** more convenient. | 'life'는 복수형으로 써야 자연스러워요. |

**심층 분석 결과 (5섹션):**

**전체 평가:**
> 오픽톡닥님, 이번 답변에서는 가족과 친구의 인터넷 사용 목적(정보 검색, SNS, 쇼핑 등)과 본인의 온라인 습관(유튜브 시청, 채팅 등)을 구체적으로 언급해 주셔서 내용 전달이 명확했습니다. 순서 연결은 다소 약했지만, 다양한 활동을 예시로 들어 설명한 점이 인상적이었습니다. 다만, 관사와 명사 수, 동사 형태, 동명사 사용 등에서 한국어식 표현이 반복되어 문장의 자연스러움과 정확성이 아쉬웠습니다.

**언어적 분석:**
> 주요 오류 유형은 관사 누락(about internet, use internet), 명사 수 일치 오류(our life), 동사 형태 및 동명사 사용 오류(spending a lot of time to watch, using for chat time), 그리고 어색한 주어-동사 연결(tell us a lot of videos)입니다. 이는 한국어에는 관사, 명사 복수, 동명사 구문 등이 명확하지 않아 직역하는 경향에서 비롯됩니다. 또한, 'almost people'처럼 빈도 부사와 명사의 결합 방식이 영어와 달라 발생하는 L1 간섭이 보입니다.

**의사소통 효과:**
> 친구와 가족이 인터넷을 어떻게 활용하는지, 본인의 온라인 습관까지 청자가 전체적으로 이해할 수 있었습니다. 다양한 예시(정보 검색, SNS, 유튜브, 채팅 등)로 내용 전달이 풍부했으나, 일부 문장에서 어색한 어순과 구문(예: using for social networking service, tell us a lot of videos) 때문에 잠시 의미 파악이 더뎌지는 부분이 있었습니다.

**숙련도 갭:**
> 현재는 다양한 활동을 나열하고 예시를 드는 능력이 있지만, AL 도달을 위해서는 'Most of my friends rely on the internet for work, while my family mainly uses it to stay in touch with relatives.'처럼 대상별로 차이점을 명확히 비교하거나, 'For example, my father checks the news every morning, but my sister prefers watching dramas online.'처럼 구체적 사례와 연결어를 자연스럽게 활용하는 능력이 필요합니다.

**학습 권장:**
> 이번 답변의 오류에 맞춘 연습 두 가지를 추천합니다. 첫째, 관사+명사+동사 패턴: 'the internet', 'the office', 'the news'처럼 특정 명사 앞에 항상 'the'를 붙여 연습하세요. 둘째, 동명사 구문 연습: 'I spend time ___ing', 'I use it for ___ing'처럼 동명사 형태로 자신의 활동을 매일 3문장씩 만들어 보세요.

### 5-2. 종합 평가 — 프롬프트 예시 (아직 실행된 적 없음)

종합 평가는 아직 실행 결과가 없으므로, 프롬프트 기반으로 **예상 출력 구조**를 제시합니다.

**입력 데이터 예시 (규칙엔진 결과):**
```
Final Level: IM2
Floor Status: ESTABLISHED
Ceiling Status: BROKE_DOWN
Sympathetic Listener: ALWAYS_NEEDED
INT Pass Rate: 89.3%
ADV Pass Rate: 41.2%
FACT: F=6.5, A=5.8, C=7.2, T=6.0 (Total=63.8)
```

**예상 출력 (프롬프트 설계 기반):**

**(1) 전문가 진단서 — 8~12문장:**

> 응시자는 Intermediate 수준에서 안정적인 수행을 보여주었으며, 중간 범위를 유지했습니다. 일상적 주제(인터넷 사용, 루틴 등)에 대해 문장 단위로 자신의 경험과 의견을 표현할 수 있었으며, 다양한 예시를 들어 내용을 보충하는 능력을 보였습니다. 그러나 Advanced 수준의 과제에서는 수행이 무너졌으며, 시제 전환이 불안정하고 문단 수준의 서사를 유지하는 능력이 부족했습니다. ...

**(2) 반복 실수 패턴 — 최대 5개:**

> 패턴 1: 관사 누락 (grammar, severity: high, 10/14문항)
> 패턴 2: 동명사 구문 오류 (grammar, severity: medium, 6/14문항)
> 패턴 3: 연결어 단조로움 (structure, severity: medium, 8/14문항)

**(3) 훈련 권장 — 3개:**

> 우선순위 1: 시제별 서사/묘사 → 과거 경험 말하기 연습
> 우선순위 2: 문단 구성 → 비교 문항 연습
> 우선순위 3: 관사/전치사 정확성 → 전체 유형 연습

## 7. 구체적 질문 사항

### 개별 평가 피드백 관련

1. **정보 표시 순서** — 현재는 "통과율 → 답변 원문 → 발음 점수 → 체크박스 → 교정 → 심층분석(접힘)" 순서입니다. **OPIc 학습자에게 가장 먼저 보여줘야 할 정보**는 무엇인가요? 실제 학원이나 1:1 튜터링에서 피드백을 줄 때 어떤 순서로 전달하시나요?

2. **심층 분석의 5섹션** (전체평가, 언어분석, 의사소통효과, 숙련도갭, 학습권장) — 이 5가지 분류가 적절한가요? 빠진 관점이 있나요? 각 섹션의 비중(분량)은 어떻게 되어야 하나요?

3. **체크박스 코드 노출 여부** — 현재 `INT-1-1`, `INT-4-F2` 등의 코드를 그대로 보여주고 있습니다. 학습자에게 체크박스 코드를 보여주는 것이 학습에 도움이 되나요? 아니면 **한국어 라벨** (예: "언어 생성 — 적절한 어휘") 또는 **카테고리 요약** (예: "유창성 영역 5/5 통과")으로 변환하는 것이 나은가요?

4. **통과율 수치의 의미** — "체크박스 통과율 78%"라는 정보가 학습자에게 직관적인가요? OPIc 학습 맥락에서 이 수치를 어떻게 해석해 줘야 의미가 있나요?

5. **교정(corrections) 표시 방식** — 현재는 오류 부분(취소선) → 교정 → 한국어 팁을 나열합니다. 대신 **문장별로 답변 텍스트 위에 오류를 인라인 하이라이트** 하는 방식이 더 효과적일까요? 학원에서 첨삭할 때 어떤 형태를 사용하시나요?

6. **발음 점수 활용** — Azure Speech의 3점수(정확도/유창성/운율, 0~100)와 오발음 단어 목록을 보여주고 있습니다. 이 정보가 OPIc 학습자에게 유용한가요? OPIc 채점에서 발음의 비중은 어느 정도인가요?

### 종합 평가 피드백 관련

7. **전문가 진단서의 톤과 내용** — 프롬프트가 ACTFL 공인 평가사 스타일의 8-12문장 진단서를 생성합니다. 이 형식이 **한국 OPIc 학습자**에게 효과적인가요? 너무 딱딱하거나, 반대로 너무 부드러운 부분이 있나요?

8. **FACT 점수 체계** — 자체 설계한 F(Fluency)/A(Accuracy)/C(Content)/T(Task) 체계입니다(각 0~10점, 총 100점). 이런 점수 체계가 학습자에게 유용한가요? 실제 OPIc에서는 이런 세분화된 점수를 제공하지 않는데, 자체 점수를 주는 것의 장단점은?

9. **반복 실수 패턴 분석** — 14문항을 교차 분석하여 "관사 누락이 10/14 문항에서 반복됨" 같은 패턴을 도출합니다. 이 정보가 학습에 얼마나 도움이 되나요? 실제 OPIc 학습에서 가장 흔한 반복 실수 유형은 무엇인가요?

10. **훈련 권장과 학습 연결** — 현재 약점에 기반해 "이 유형의 스크립트를 만들어 연습하세요" CTA를 제공합니다. 모의고사 → 스크립트 → 쉐도잉으로 이어지는 학습 루프가 효과적인가요? 추가해야 할 학습 활동이 있나요?

### 전체 구조 관련

11. **개별 평가 vs 종합 평가의 역할 분담** — 개별 평가에서 이미 5섹션 심층 분석을 제공하는데, 종합 평가에서는 어떤 **추가적 가치**를 줘야 하나요? 두 평가가 중복되지 않으면서 보완적이려면 각각의 초점이 어떻게 다르면 좋겠나요?

12. **등급 피드백의 구체성** — "IM2입니다"라는 결과만으로는 학습 방향이 불명확합니다. **"IM2인데 IM3으로 올라가려면 구체적으로 무엇을 해야 하는지"**를 얼마나 상세하게 안내해야 하나요? 등급 간 핵심 차이를 어떻게 설명하면 효과적인가요?

13. **피드백 분량과 깊이** — 현재 개별 평가 1건의 GPT 출력이 약 1,000~1,500단어 수준입니다. 학습자가 실제로 읽고 활용하기에 적절한 분량인가요? 너무 많으면 어떤 부분을 줄이고, 부족하면 어떤 부분을 늘려야 하나요?

14. **시각적 표현** — 통과율 바, 점수 차트, 색상 코딩(초록/노랑/빨강) 등 시각적 요소를 사용하고 있습니다. OPIc 학습 맥락에서 **수치/차트 중심 피드백**과 **텍스트/서술 중심 피드백** 중 어느 쪽이 더 효과적인가요?

### 개선 방향 관련

15. **가장 먼저 개선해야 할 부분** — 위에 제시한 개별 평가와 종합 평가의 현재 화면 구성에서, **학습 효과를 위해 가장 시급하게 바꿔야 할 1~2가지**는 무엇인가요?

16. **빠진 피드백 요소** — 현재 제공하지 않지만, OPIc 학습자에게 **꼭 보여줘야 하는 정보**가 있다면 무엇인가요? (예: 답변 시간 배분, 도입/본론/결론 구조 평가, 콤보 대비 전략 등)

17. **실제 OPIc 결과와의 차이** — 실제 OPIc 시험은 등급(NH~AL)만 알려줍니다. 우리 시스템은 상세 피드백을 제공하는데, 이것이 **실제 시험 결과와 괴리**를 만들 위험이 있나요? (예: 우리 시스템에서 IM3인데 실제 시험에서 IM1이 나오는 경우)

| ADV-3: Vocabulary | ADV-3-V1 ~ V5 | 5 | 어휘 다양성 |
| ADV-4: Complication | ADV-4-C1 ~ C5 | 5 | 문제 해결 능력 |
| ADV-5: Readily Understood — Fluency | ADV-5-RF1 ~ RF3 | 3 | 고급 유창성 |
| ADV-5: Readily Understood — Pronunciation | ADV-5-RP1 ~ RP3 | 3 | 고급 발음 |
| ADV-5: Readily Understood — Grammar | ADV-5-RG1 ~ RG4 | 4 | 고급 문법 |

### 8-3. FACT 점수 체계

| 영역 | 풀네임 | 산출 기준 | 배점 |
|------|--------|----------|------|
| F | Fluency (유창성) | INT/ADV Fluency 체크박스 통과율 + WPM + 필러 + 침묵 | 0~10 |

| **Total** | | F×2.5 + A×2.5 + C×2.5 + T×2.5 | **0~100** |
