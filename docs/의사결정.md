### P-5. 리브랜딩: 오픽톡닥 → 하루오픽 ⏳

**핵심 질문**: 서비스 이름과 도메인을 변경할 것인가?

**배경 (2026-02-25)**:
- "오픽톡닥" 문제점: 4음절 발음 어려움, 도메인 11자(opictalkdoc) 타이핑 오타 잦음
- 메타포 혼재: 이름/로고=의료, 랜딩=연극/무대, 메뉴=학습 — 서비스 본체와 이름이 동떨어짐
- 실제 메뉴(시험후기, 스크립트, 모의고사, 튜터링)는 "체계적 훈련 과정"이며 의료 메타포와 무관

**검토 과정**:
1. 의료 메타포 유지 (오픽닥, 오픽룸) → 서비스 본체와 안 맞음
2. 훈련/연구소 계열 (오픽랩, 오픽캠프) → 랜딩 톤(따뜻한, 위로)과 안 맞음
3. 일상/감성 계열 → 랜딩 카피("평범한 하루가 대본")와 직결

**결정 (2026-02-25)**: **하루오픽 (HaruOPIc)** 확정
- **이름**: 하루오픽
- **도메인**: haruopic.com (8자, 구매 가능 확인)
- **발음**: 하-루-오-픽 (4음절, 부드러운 진입)
- **의미**: "하루의 OPIc" — 일상이 곧 대본, 매일의 OPIc 준비
- **컨셉**: "나는 내 삶의 주인공이다" (유지)
- **태그라인**: "말하다, 나답게." (유지)
- **도메인 가독성**: haru + opic → 음절 경계 명확 (opicharu는 "오피차루" 오독 위험)

**왜 "하루"가 앞인가**:
- "하루"로 시작 → 일상/따뜻함이 먼저 전달 → 20대 감성
- "오픽"으로 끝 → 서비스 성격 확인
- 랜딩 카피 "평범한 **하루**가 가장 완벽한 대본입니다"와 직결

**변경 범위**:
| 항목 | 내용 | 난이도 |
|------|------|--------|
| 도메인 | haruopic.com 구매 + DNS(Vercel/Spaceship) | 낮음 |
| 코드 텍스트 | "오픽톡닥" → "하루오픽" 전체 치환 | 낮음 |
| 로고 | 반창고 아이콘 + "하루오픽" 텍스트 재생성 | 중간 |
| CORS | T-4 도메인 허용 목록에 haruopic.com **추가** (기존 유지) | 낮음 |
| GitHub | 저장소명 변경 (선택) | 낮음 |
| Supabase | 변경 불필요 | 없음 |

**도메인 전환 전략 (양쪽 동시 운영)**:
- **주 도메인**: haruopic.com (새 브랜드)
- **리다이렉트**: opictalkdoc.com → haruopic.com (301 리다이렉트)
- **기존 도메인 유지**: opictalkdoc.com 1년 구매 완료, PG사 심사 완료까지 유지
- **이유**: 카카오페이 카드사 심사 진행 중, KG이니시스 본계약 대기, 토스페이 MID 대기

**유지 항목**: 디자인 시스템(테라코타, 크림, 반창고 로고 아이콘), 슬로건, 랜딩 컨셉, 무대 메타포

| # | 문제 | 위험도 | 결정 |

### T-9. 하이브리드 백엔드 전략 ✅ (큰 방향만 확정, 모듈별 상세는 미정)

**핵심 질문**: Server Actions만 쓸 것인가, Edge Functions도 활용할 것인가?

**배경 (T-8 재검토)**:
- T-8에서 "Edge Functions 사용 안 함"으로 결정했으나, 실제 모듈 분석 결과 AI API 호출(GPT, Whisper, Gemini TTS, Azure Speech)이 다수 존재
- 이런 AI 작업은 10~60초 이상 소요 → Vercel Hobby 서버리스 함수 제한(10초)에 걸릴 수 있음
- Supabase Pro($25)에 Edge Functions 무료 포함 → 추가 비용 없음

**결정 (2026-02-23)**: 하이브리드 — 작업 성격에 따라 분리

| 작업 성격 | 백엔드 방식 | 예시 |
|----------|-----------|------|
| **가벼운 CRUD** | Server Actions | DB 읽기/쓰기, 세션 관리, 목록/상세 조회 |
| **무거운 AI API** | Edge Functions | GPT 생성/채점, Whisper STT, Gemini TTS, Azure 발음 |

**모듈별 상세 적용 (⏳ 초안 — 각 모듈 이관 시 소리담 파악 후 확정)**:

| 모듈 | Server Actions (예상) | Edge Functions (예상) | 확정 |
|------|----------------------|----------------------|------|
| 시험후기 | 전부 (DB CRUD만) | 없음 | ✅ 구현 완료 |
