# OPIc 시험 구조

> **목적**: OPIc 시험의 구조를 정확히 파악하여 master_questions 테이블 재설계, 후기 검증, 모의고사 시험풀 구축의 기반 자료로 활용한다.
> **참조**: [`docs/이현석DB_분석.md`](이현석DB_분석.md) — 이현석 DB 431질문, 198세트, 28토픽, 41RP 추출 완료
> **최종 업데이트**: 2026-02-28

---

## 1. 핵심 전략

```
사용자 시험 응시
    ↓
후기 제출 (14문항 기록)
    ↓
OPIc 구조 기반 검증 ← master_questions가 검증 기준
  ✓ 일반 콤보 3개인가?
  ✓ 롤플레이 1개, 어드밴스 1개인가?
  ✓ 선택형 3 : 공통형 2 비율인가?
  ✓ 같은 parent_topic이 중복되지 않는가?
    ↓
검증 통과 → 기출문제로 확정
    ↓
모의고사 시험풀로 활용
```

| 개념 | 설명 |
|------|------|
| **후기 = 기출문제** | 사용자가 실제 시험에서 받은 질문을 수집 |
| **master_questions = 검증 기준** | 후기가 OPIc 구조에 맞는지 판단하는 레퍼런스 DB (직접 출제 X) |
| **모의고사 = 기출 재활용** | 검증 통과된 기출에서 콤보 단위로 시험 구성 |
| **추천 서베이** | 특정 서베이를 고정 추천하여 출제 범위를 좁히는 전략 |

## 2. 시험 흐름

### 2.1 사전 단계

| 순서 | 단계 | 설명 |
|------|------|------|
| 1 | **Background Survey** | 배경 설문 — 최소 12개 항목 체크. 선택형 질문 풀 결정 |
| 2 | **Self Assessment** | 난이도 선택 (1~6 중 택 1) → 콤보 구성 결정 |
| 3 | **스피커 테스트** | 음성 녹음 테스트 |

### 2.2 시험 본편 (난이도 5-5 기준)

총 **15문항**, 약 **40분**, 음성 답변 녹음.

```
Q1       자기소개 (고정) ─────────────────────────── 시스템
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
Q2  ┐
Q3  ├── 일반 콤보 ① (주제 A)     ┐
Q4  ┘                            │
Q5  ┐                            │
Q6  ├── 일반 콤보 ② (주제 B)     ├── 3세트 × 3문항 = 9문항
Q7  ┘                            │   배치 순서는 시험마다 유동
Q8  ┐                            │
Q9  ├── 일반 콤보 ③ (주제 C)     ┘
Q10 ┘
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
Q11  roleplay_11 (질문하기)   ┐
Q12  roleplay_12 (문제해결)   ├── 롤플레이 × 1세트 (고정 위치)
Q13  roleplay_13 (과거경험)   ┘
─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
Q14  advanced_14 (사회적 변화) ┐
Q15  advanced_15 (심화 의견)   ┘── 어드밴스 × 1세트 (고정 위치)

총 15문항 = 1(자기소개) + 9(일반) + 3(롤플레이) + 2(어드밴스)
```

## 3. 출제 규칙

### 3.1 콤보 구성

OPIc 시험에서 **하나의 주제에 대해 연속으로 출제되는 질문 묶음**을 콤보라고 부른다.

| 콤보 유형 | 위치 | 문항수 | 구성 |
|----------|------|--------|------|
| **일반** | Q2~Q10 (유동) | 3문항 | description + routine + 경험 |
| **롤플레이** | Q11~Q13 (고정) | 3문항 | 질문하기 + 문제해결 + 과거경험 |
| **어드밴스** | Q14~Q15 (고정) | 2문항 | 사회적 변화 + 심화 의견 |

### 3.2 선택형 3 : 공통형 2 (고정 비율)

```
5개 콤보 = 선택형 3개 + 공통형 2개 (항상 고정)

선택형 ── Background Survey에서 선택한 주제에서 출제
공통형 ── 서베이 선택과 무관하게 누구에게나 출제

⚠️ 비율 3:2는 고정, 어떤 콤보 유형(일반/롤플레이/어드밴스)에 배치되는지는 랜덤

예시)
  선택형: 음악(일반) + 쇼핑(일반) + 해외여행(일반)   ← 서베이에서 선택한 주제
  공통형: 건강(롤플레이) + 인터넷(어드밴스)            ← 서베이 무관
```

### 3.3 주제 중복 불가 (절대 규칙)

> ⚠️ **하나의 parent_topic은 같은 시험에서 한 번만 출제된다.**

- 일반으로 나오면 → 같은 주제가 롤플레이/어드밴스로 **절대 안 나옴**
- 롤플레이로 나오면 → 같은 주제가 일반/어드밴스로 **절대 안 나옴**
- 어드밴스로 나오면 → 같은 주제가 일반/롤플레이로 **절대 안 나옴**

**예시 — "음악 감상하기" 서베이 선택 시:**

```
parent_topic: 음악 감상하기

┌─ 일반 (Q2~Q10 중 배치) ─────────────────────────────────┐
│  topic: 음악                                             │
│  → description + routine + past_exp  (3문항)              │
└──────────────────────────────────────────────────────────┘

┌─ 롤플레이 (Q11~Q13 고정) ───────────────────────────────┐
│  topic: MP3 Player 구매  → roleplay_11 + 12 + 13 (3문항) │
│  topic: 공연 예매        → roleplay_11 + 12 + 13 (3문항) │
└──────────────────────────────────────────────────────────┘

┌─ 어드밴스 (Q14~Q15 고정) ───────────────────────────────┐
│  topic: 음악                                             │
│  → advanced_14 + advanced_15  (2문항)                     │
└──────────────────────────────────────────────────────────┘

⚠️ 위 3가지 중 단 하나만 출제. 절대 중복 없음.
```

### 3.4 배치 규칙 요약

| 항목 | 고정/유동 | 설명 |
|------|----------|------|
| Q1 자기소개 | 고정 | 항상 첫 문항 |
| Q2~Q10 일반 콤보 배치 순서 | **유동** | 같은 질문이 시험마다 다른 위치에 배치 |
| Q11~Q13 롤플레이 | 고정 | 항상 11-12-13번 |
| Q14~Q15 어드밴스 | 고정 | 항상 14-15번 |
| 선택형:공통형 비율 | **고정** | 항상 3:2 |
| 어떤 콤보가 선택형/공통형인지 | **랜덤** | 시험마다 다름 |

→ master_questions에 고정 `question_number`를 넣는 것은 **부적절** (일반 콤보 위치가 시험마다 다르므로)

## 4. 질문 유형 (answer_type)

### 4.1 10가지 유형

| 대분류 | answer_type | 설명 | 예시 |
|--------|-------------|------|------|
| **묘사/설명** | `description` | 장소, 사물, 사람 묘사 | "Describe your home" |
| **습관/루틴** | `routine` | 일상 활동, 반복 행동 | "What do you usually do at the park?" |
| **비교** | `comparison` | 과거↔현재, A↔B 비교 | "How has technology changed?" |
| **경험** | `past_experience_memorable` | 기억에 남는 경험 | "Tell me about a memorable trip" |
| | `past_experience_recent` | 최근 경험 | "Tell me about the last time you..." |
| | `past_experience_childhood` | 어린 시절 경험 | "What did you do as a child?" |
| **롤플레이** | `roleplay_11` | 질문하기 (Ask) | "Call and ask 3-4 questions" |
| | `roleplay_12` | 문제해결 (Resolve) | "Explain the problem and suggest alternatives" |
| | `roleplay_13` | 과거경험 (Past) | "상황극 종료 후 과거 경험 서술" |
| **어드밴스** | `advanced_14` | 사회적 이슈/변화 | "How has X changed in your country?" |
| | `advanced_15` | 심화 분석/의견 | "What do you think about...?" |

### 4.2 콤보별 질문 순서

| 콤보 | 순서 | 상태 |
|------|------|------|
| **롤플레이** | `roleplay_11`(질문하기) → `roleplay_12`(문제해결) → `roleplay_13`(과거경험) | ✅ 확정 |
| **일반** | [Int] 묘사/설명 → [Int/Adv] 상세/비교 → [Adv] 과거경험/변화 | ✅ 확정 (이현석 DB) |
| **어드밴스** | advanced_14(시간 변화/비교/사회적 이슈) → advanced_15(뉴스/여론/심층 분석) | ✅ 확정 (이현석 DB) |

### 4.3 roleplay_13 DB 현황 (⚠️ 정합성 이슈)

| answer_type | DB 행수 | 상태 |
|-------------|---------|------|
| `roleplay_11` | 62개 | ✅ 정상 |
| `roleplay_12` | 65개 | ✅ 정상 |
| `roleplay_13` | **0개** | ⚠️ `past_experience_memorable`로 잘못 분류됨 |

- 식별 기준: 한국어 텍스트가 **"상황극이 끝났습니다"**로 시작하는 행
- 조치: `question_korean LIKE '상황극이 끝났습니다%'` → `roleplay_13`으로 재분류 필요

## 5. 주제 체계

### 5.1 3계층 구조

```
Background Survey 항목 (서베이 원문)
    ↓  parent_topic으로 연결
DB topic (현재 master_questions.topic)
    ↓
answer_type별 질문들
```

**예시:**
```
서베이: "음악 감상하기"
    ├── topic: 음악           → description, routine, comparison, past_exp, advanced_14, advanced_15
    ├── topic: MP3 Player 구매 → roleplay_11, roleplay_12, roleplay_13
    └── topic: 공연 예매       → roleplay_11, roleplay_12, roleplay_13
```

### 5.2 카테고리 (topic_category)

| 카테고리 | 설명 | 비고 |
|---------|------|------|
| **일반** | 서베이 선택 주제 + 공통 주제 | 일부 topic이 어드밴스 질문도 보유 |
| **롤플레이** | 상황극 주제 | Q11~Q13 고정 |
| **어드밴스** | 심화 토론 주제 | Q14~Q15 고정 |
| **시스템** | 자기소개, 스피커 테스트 | 2개 |

### 5.3 서베이 유형 (survey_type)

| 유형 | 설명 | 현재 행수 |
|------|------|----------|
| **선택형** | Background Survey에서 선택한 주제에서 출제 | 197 |
| **공통형** | 서베이 선택과 무관하게 누구에게나 출제 | 312 |
| **시스템** | 자기소개, 스피커 테스트 | 2 |

### 5.4 주제 중복/분리 현황

같은 대상에 대해 일반/롤플레이/어드밴스가 별도 topic으로 존재 → **parent_topic 필요**:

| 일반 주제 | 롤플레이 주제 | 어드밴스 |
|----------|-------------|---------|
| 공원 | 공원 방문 | - |
| 쇼핑 | 쇼핑 정보 문의 / 세일 상품 구매 / 옷 구매 | - |
| 영화 | 영화 관람 계획 / 친구들과 영화 관람 준비 | - |
| 카페 | 카페 이용 | - |
| 호텔 | 호텔 이용 | - |
| 은행 | 은행 계좌 개설 | - |
| 가구 | 가구 구매 | - |
| 가전제품 | - | 가전/전자제품 |
| 재활용 | 재활용 규정 | - |
| 전화기 | - | 휴대폰 |
| 건강 | 건강식품 구매 | - |
| 해외여행 | 해외여행 날씨 / 여행 계획 / 차량 렌트 / 공항 / 기차표 예매 / 해외 친구 방문 / 친구의 미국 여행 | - |

## 6. Background Survey 원문

> ✅ 실제 OPIc 시험 서베이 원문 기준. 최소 12개 이상 체크.

### 여가 활동 (27개, 두 개 이상 선택)
영화보기, 클럽/나이트클럽 가기, 박물관 가기, 공원 가기, 스포츠 관람, 주거 개선, 게임하기(비디오/카드/보드/휴대폰 등), 공연보기, SNS에 글 올리기, 캠핑하기, 구직활동하기, 술집/바에 가기, 요리 관련 프로그램 시청하기, 친구들과 문자대화하기, 스파/마사지샵 가기, 당구 치기, 리얼리티쇼 시청하기, 자원봉사하기, 쇼핑하기, 차로 드라이브하기, TV 시청하기, 뉴스를 보거나 듣기, 콘서트 보기, 시험 대비 과정 수강하기, 해변가기, 체스하기, 카페/커피전문점 가기

### 취미/관심사 (14개, 한 개 이상 선택)
아이에게 책 읽어주기, 음악 감상하기, 글쓰기(편지/단문/시 등), 그림그리기, 애완동물 기르기, 독서, 주식 투자하기, 신문 읽기, 사진 촬영하기, 혼자 노래 부르거나 합창하기, 악기 연주하기, 요리하기, 춤추기, 여행 관련 잡지나 블로그 읽기

### 운동 (24개, 한 개 이상 선택)
농구, 조깅, 야구/소프트볼, 걷기, 축구, 요가, 미식축구, 하이킹/트레킹, 하키, 낚시, 크로켓, 헬스, 골프, 태권도, 배구, 운동 수업 수강하기, 테니스, 아이스 스케이트, 배드민턴, 탁구, 수영, 자전거, 스키/스노우보드, 운동을 전혀 하지 않음

### 휴가/출장 (5개, 한 개 이상 선택)
국내출장, 해외출장, 집에서 보내는 휴가, 국내여행, 해외여행

## 7. 오픽톡닥 추천 서베이

출제 범위를 좁혀 **최소 노력으로 최대 성과**를 내는 전략. 전 서비스(스크립트/모의고사/튜터링)에 적용.

### 7.1 기본 정보

| 항목 | 설정 |
|------|------|
| 직업 | 일 경험 없음 |
| 학생 | 아니오 |
| 수강 | 수강 후 5년 이상 지남 |
| 거주 | 개인 주택이나 아파트에 홀로 거주 |

### 7.2 배경 설문 선택

| 카테고리 | 선택 항목 |
|---------|----------|
| 여가 활동 | 영화보기, 쇼핑하기, TV 시청하기, 공연보기, 콘서트 보기 |
| 취미/관심사 | 음악 감상하기 |
| 운동 | 조깅, 걷기, 운동을 전혀 하지 않음 |
| 휴가/출장 | 집에서 보내는 휴가, 국내여행, 해외여행 |

## 10. master_questions 테이블 재설계

### 10.1 왜 재설계인가

현재 master_questions는 **"질문 저장소"**일 뿐, 시험 구조(콤보, 출제 규칙, 서베이 매핑)를 모른다.

| 역할 | 현재 | 필요 |
|------|------|------|
| 질문 내용 저장 | ✅ | ✅ |
| 주제별 분류 | ✅ | ✅ |
| 유형별 분류 | ⚠️ (roleplay_13 미분류) | ✅ |
| 서베이 ↔ topic 연결 | ❌ | ✅ |
| 후기 구조 검증 기준 | ❌ | ✅ 핵심 |
| 상위 주제 중복 방지 | ❌ | ✅ |

### 10.2 확정 사항

- **`question_number` 불필요**: 일반 콤보 위치가 유동적이므로 고정 속성 아님
- **`parent_topic` 필요**: 서베이 항목과 topic을 연결하여 중복 검증 기반
  ```
  parent_topic: "음악 감상하기"
  ├── topic: 음악          (일반 + 어드밴스)
  ├── topic: MP3 Player 구매 (롤플레이)
  └── topic: 공연 예매      (롤플레이)
  ```

### 10.3 추가 논의 필요 항목

| 항목 | 현재 | 논의 사항 |
|------|------|----------|
| `topic_category` | ENUM (일반/롤플레이/어드밴스/시스템) | 하나의 topic이 여러 카테고리에 걸침. 구조 적절한지? |
| `survey_type` | ENUM (선택형/공통형/시스템) | 활용도 낮음. 역할 재정의 또는 삭제? |
| `question_id` 체계 | `COM_APP_N_C1_01` 코드 | 현행 유지 vs 재설계? |
| `roleplay_13` 분류 | `past_experience_memorable`로 혼재 | 재분류 실행 여부 |
| 테이블 분리 | 단일 테이블 510행 | topics 테이블 별도 분리 필요성? |
| 추천 서베이 | DB에 반영 안 됨 | 추천 서베이 항목을 DB로 관리할지? |

### 10.4 모듈별 관계

| 모듈 | master_questions 역할 | 재설계 시 필요 |
|------|---------------------|--------------|
| **후기 제출** | 주제/질문 선택 UI + OPIc 구조 검증 기준 | parent_topic (중복 검증) |
| **빈도 분석** | 통계 집계 기준 | parent_topic (상위 주제 그룹 통계) |
| **스크립트 생성** | 주제/질문 선택 UI + GPT 프롬프트 분기 | 현행 유지 가능 |
| **모의고사** | 검증 통과된 기출에서 콤보 재활용 | parent_topic (중복 방지) |

```
master_questions (검증 기준) → 후기 제출 (기출 수집 + 검증) → 모의고사 (검증된 기출 재활용)
                            → 스크립트 (주제/질문 선택)
                            → 빈도 분석 (통계)
```
