## 개발 원칙

- **하이브리드 백엔드 (T-9)**: CRUD = Server Actions, AI API 호출 = Edge Functions, 웹훅 = API Routes
- **모듈별 풀스택 이관**: DB → RLS → SA → 타입 → UI → 테스트
- **업계 표준 우선**: 편의성보다 검증된 정석 방식. 확신 없으면 조사 후 제안
- **환경 제약 먼저 확인**: 코드 수정 전 실행 환경 자체의 제약(인앱 브라우저 등) 검토
- **TanStack Query 필수**: 클라이언트에서 `useState + useEffect` 금지
