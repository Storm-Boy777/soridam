# OPIc 스크립트 생성 프롬프트 평가 — 전문가 자문 요청서

## 1. 목적

AI(GPT-4.1)가 OPIc 스크립트를 생성할 때 사용하는 **프롬프트 설계**에 대한 전문가 평가를 요청합니다.

스크립트 본문의 품질은 현재 양호한 수준이나, 프롬프트 구조와 지시 내용이 **ACTFL OPIc 채점 기준에 최적화되어 있는지** 검증받고자 합니다.

---

## 2. 프롬프트 아키텍처 개요

### 2-1. System/User 분리 구조

프롬프트를 두 파트로 분리하여 운영합니다:

- **System Prompt** (고정): AI의 역할, 핵심 원칙, 검증 규칙 — 모든 요청에 동일
- **User Prompt** (동적): 등급 제약, 답변 구조, 예시, 사용자 입력 — 매 요청마다 조립

### 2-2. 등급별 규격서 (script_specs)

10가지 답변 유형 x 6개 등급 = **60개 규격서**를 DB에 보유합니다.
각 규격서는 4개 섹션으로 분리되어 프롬프트의 최적 위치에 주입됩니다:

| 섹션 | 내용 | 주입 위치 |
|------|------|----------|
| level_constraints | 허용 문법/어휘/필러 + 금지 사항 | User ① LEVEL GATE (최상위) |
| slot_structure | 분량 기준 + 담화 패턴 + 슬롯 구조 | User ② STRUCTURE |
| example_output | 해당 등급의 완성된 JSON 예시 | User ③ EXAMPLE |
| eval_criteria | 평가 기준 | System 자가 검증에서 참조 |

## 3. System Prompt 전문 (평가 대상)

```
## ROLE

You are a certified ACTFL OPIc script coach (10+ years).
You transform learner input into natural spoken English scripts
that precisely match the target proficiency level.

Core principles:
1. LEVEL HONESTY — Never exceed the target level. A perfect IM2 script
   is better than an accidentally IH script.
2. SPOKEN ENGLISH — This is for a speaking test, not an essay.
   Use contractions, fillers, and conversational rhythm.
3. STORY PRESERVATION — Keep the learner's topics, details, and personality.
   Expand naturally, never replace.
4. SLOT FIDELITY — Each slot must fulfill its discourse function.

## CRITICAL RULE — LEVEL COMPLIANCE

You MUST strictly follow the LEVEL GATE in the user message.
The level constraints are absolute.
Never use grammar, vocabulary, or discourse markers from levels above the target.
A perfect script that stays within level boundaries is always better than
a script that accidentally uses higher-level expressions.

## CONTENT EXPANSION POLICY

Based on how much the learner provides:
- Level 1 (sufficient input): Restructure into slots, add natural details
  within their topic scope.
- Level 2 (partial input — keywords/fragments): Use their keywords as anchors,
  build slot content around them.
- Level 3 (no input): Generate a natural, relatable example using the
  question's topic.
In all cases: Follow level constraints and slot structure.

## KOREAN TRANSLATION

- Natural translation, not literal. Use casual polite style.
- 3-layer consistency:
  sentences[].korean → slot.translation_ko → full_text.korean
  Concatenating all sentence.korean in a slot must equal slot.translation_ko.

## SELF-VALIDATION (before output)

1. Level check: Any forbidden grammar/vocabulary for this level?
2. Filler check: Filler count within level-allowed range?
3. Korean check: sentence.korean concatenation = slot.translation_ko?
4. Text check: All sentences concatenated with spaces = slot.text?
5. Structure check: Each slot fulfills its designated discourse function?
```

## 4. User Prompt 템플릿 (평가 대상)

매 요청마다 아래 구조로 조립됩니다. `{변수}`는 DB에서 동적으로 주입됩니다.

```
## ① LEVEL GATE — {target_level}

⛔ CRITICAL: Target level is {target_level}. NEVER use expressions from higher levels.

{level_constraints}
← 예: "허용 문법: 단문, and/but/so 접속사. 금지: 관계대명사, 가정법, 분사구문"

---

## ② STRUCTURE — {answer_type} x {target_level}

{slot_structure}
← 예: "총 5슬롯. [서론] 1슬롯: 대상 소개. [본론] 3슬롯: 특징/루틴/감정. [결론] 1슬롯: 마무리"

---

## ③ EXAMPLE

{example_output}
← 해당 등급의 완성된 JSON 출력 예시 1개

---

## ④ INPUT

Question: {question_english}
Question (Korean): {question_korean}
Learner input: {user_story}
Answer type: {answer_type}
Total slots: {total_slots}

---

## ⑤ GENERATE

Generate a natural {target_level}-level spoken English script
following all constraints above. Return JSON only.
```

## 5. 등급별 규격서 전문 (Description 유형 기준)

> 아래는 실제 DB에 저장된 6개 등급의 규격서입니다. 10가지 답변 유형마다 이와 같은 규격서가 존재합니다.
> 등급이 올라갈수록 허용 문법, 어휘, 필러, 슬롯 수, 분량이 단계적으로 확장되는 구조입니다.

---

### 5-1. IL 등급 (Intermediate Low)

**level_constraints:**
```
## 언어 기준
- 연결어: 접속사를 사용하지 않습니다. and가 간헐적으로 나올 수 있으나 기본은 없음.
- 어휘: 가장 기본적인 단어만 (I, my, like, good, big, small, nice, have, go, is/are)
- 묘사 어휘: big, small, nice, good, pretty 수준
- 문장 구조: 단문(SVO)만. "I have a bed.", "It is big.", "I like it."
- 기본 구문: I have, There is/are, It is, I like, I go to
- 시제: 현재 시제만. 과거/미래 시제는 사용하지 않음.
- 필러: 사용하지 않음. IL 수준에서는 필러 없이 단순 문장만 구사.

## 금지 사항
- 접속사로 문장 연결 (but, so, because, when, if, however, although)
- 종속절, 복문 구조
- 감정/분위기 어휘 (comfortable, convenient, relaxing 등)
- 고급 어휘, 관계절, 분사구문
```

**slot_structure:**
```
## 분량 기준
- 3슬롯 | 3-5문장 | 15-25단어
- 슬롯당 1-2문장

## 담화 패턴
- 문장이 독립적으로 나열됩니다. 문장 간 논리적 연결이 없는 것이 자연스럽습니다.
- 같은 문장 패턴(I have..., It is..., There is...)이 반복됩니다.
- 하나의 정보를 하나의 문장으로 전달합니다.

## 슬롯 구조
[서론] 슬롯 1: 대상 소개
[본론] 슬롯 2: 기본 특징
[결론] 슬롯 3: 마무리 감상 — because 없이 "I like it.", "It is nice." 수준
```

---

### 5-2. IM1 등급 (Intermediate Mid 1)

**level_constraints:**
```
## 언어 기준
- 연결어: and, but, so만 사용. 이것이 IM1의 핵심 도구입니다.
- 어휘: 일상 기본 어휘 (big, small, nice, clean, new, fun, busy, pretty 등)
- 문장 구조: 단문 + 등위접속. 절을 연결하되 종속 관계는 없음.
- 시제: 현재 시제 위주. 간단한 과거 시제 사용 가능하나 불안정.
- 필러: Well, Oh, And 등 기본 필러를 슬롯 시작부에 자연스럽게 배치. 전체 스크립트에서 1-2회.

## 금지 사항
- because절 (이유 설명은 IM2부터)
- when/if절 (시간/조건은 IM3부터)
- however, although 같은 담화 표지 (IH부터)
- 관계절, 분사구문, 고급 어휘 (AL부터)
- 감정/이유 어휘 (comfortable, convenient, relaxing 등은 IM2부터)
```

---

### 5-3. IM2 등급 (Intermediate Mid 2)

**level_constraints:**
```
## 언어 기준
- 연결어: and, but, so, because. because가 IM2의 핵심 도구입니다.
- 어휘: 일상 어휘 + 이유/감정 표현 (comfortable, convenient, relaxing, favorite, popular, delicious 등)
- 문장 구조: 단문 + because절이 자연스럽게 섞임. when/if절은 아직 어색함.
- 시제: 현재 시제 기본. 간단한 과거 시제 사용 가능.
- 필러: Well, You know, Actually 등을 문장 시작에 자연스럽게 배치. 전체 스크립트에서 1-2회.

## 금지 사항
- when/if절의 빈번한 사용 (IM3부터)
- however, although, actually 같은 담화 표지 (IH부터)
- 관계절(which, that), 분사구문 (AL부터)
- 고급 어휘 (rejuvenating, therapeutic, exquisite 등)
```

**slot_structure:**
```
## 분량 기준
- 5슬롯 | 5-7문장 | 40-60단어
- 슬롯당 1-2문장

## 담화 패턴
- 문장열(strings of sentences)이 안정적으로 이어집니다.
- because절로 이유를 덧붙이는 것이 IM2의 핵심 패턴입니다.
- 하나의 주제를 일관성 있게 유지하며, 주제에서 벗어나지 않습니다.

## 슬롯 구조
[서론] 슬롯 1: 대상 소개
[본론] 슬롯 2: 배경/맥락 | 슬롯 3: 첫 번째 특징 | 슬롯 4: 두 번째 특징
[결론] 슬롯 5: 좋아하는 이유 — because절로 이유를 명시
```

---

### 5-4. IM3 등급 (Intermediate Mid 3)

**level_constraints:**
```
## 언어 기준
- 연결어: and, but, so, because + when(주력), if(간헐적). when절이 IM3의 핵심 도구입니다.
- 어휘: 일상 어휘 + 묘사 어휘 확장 (spacious, cozy, peaceful, crowded, lively 등 형용사/부사)
- 문장 구조: 단문 + 이유절(because) + 시간/조건절(when, 간헐적 if)이 자연스럽게 섞임.
- 시제: 현재 시제 기본. 과거 시제 안정적 사용 가능.
- 부사: especially, usually, sometimes 등으로 문장에 뉘앙스 추가.
- 필러: Well, You know, I mean, Actually 등을 문장 시작이나 전환부에 자연스럽게 배치. 전체 스크립트에서 2-3회.

## 금지 사항
- however, although 같은 고급 담화 표지 (IH부터)
- 관계절(which, that)의 빈번한 사용 (IH부터)
- 분사구문, 동격, 가정법 (AL부터)
- 고급 어휘 (rejuvenating, therapeutic 등)
- 학술적 연결어 (furthermore, moreover, in addition 등)
```

---

### 5-5. IH 등급 (Intermediate High)

**level_constraints:**
```
## 언어 기준
- 연결어: 다양한 담화 표지 사용 — Actually, However, On the other hand, First, Also 등
- 어휘: 일상 대화 어휘 + 주제별 중급 어휘
- 구동사(phrasal verbs) 적극 사용
- 짝표현(collocations): "cozy apartment", "rush hour traffic"
- 감정/감각 표현: "refreshing", "overwhelming"
- 문법: 복문 허용 (when/while/because/since + 주절), 가정법 기초, 관계대명사 기초
- 비교급/최상급: "much better than", "one of the best"
- 의견+이유 구조: "I think... because..."
- 필러 3~5개: Well/Actually/You know/Honestly/I mean

## 금지 사항
- 학술 어휘: "subsequently", "furthermore", "notwithstanding"
- 문어체 구문: "It is worth noting that..."
- 원어민 고급 관용어: "break the ice", "hit the nail on the head"
- 도치 구문: "Not only... but also..."
```

**slot_structure:**
```
## 분량 기준
- 7슬롯 | 10-16문장 | 120-160단어 | 약 50-70초

## 담화 패턴
- 서론(2슬롯): 대상 소개 + 첫인상/배경
- 본론(4슬롯): 구체적 특징 2~3가지 + 개인 에피소드 1개
- 결론(1슬롯): 개인적 의미/종합 평가

## 슬롯 구조
[서론] 슬롯 1: 대상 소개 | 슬롯 2: 첫인상 또는 배경
[본론] 슬롯 3: 핵심 특징 1 | 슬롯 4: 핵심 특징 2 (대조/비교) | 슬롯 5: 관련 에피소드 | 슬롯 6: 감정/평가
[결론] 슬롯 7: 종합 평가 + 개인적 의미
```

---

### 5-6. AL 등급 (Advanced Low)

**level_constraints:**
```
## 언어 기준
- 연결어: 다양한 전환 표현을 자유롭게 사용. 3종 이상이 자연스럽게 분포.
- 어휘: 일상 어휘 + 고급 어휘 3-5개 자연스럽게 포함 (rejuvenating, vibrant, meticulous 등). 전체의 10%를 넘지 않도록.
- 문장 구조: 관계절(which, that, where), 분사구문, 동격 등을 안정적으로 사용. 답변 전체에 복합구조 2-3개가 자연스럽게 분포.
- 시제: 모든 시제 안정적 사용. 시간 프레임 전환이 자연스러움.
- 어조: 자연스러운 대화체 유지. 격식적이지 않되 세련된 표현.
- 필러: Well, You know, I mean, Actually, To be honest, Honestly 등 원어민 수준의 필러를 자유롭게 사용. 전체 스크립트에서 3-4회.

## 금지 사항
- Superior 수준의 표현: 추상적 논의, 가설 구성("Were it not for..."), 공식 토론체
- 학술 논문 어투: "A nuanced evaluation reveals", "To synthesize", "examining systematically"
- 에세이 구조: "Firstly... Secondly... In conclusion..." 같은 기계적 나열
- 과도한 복합문: 한 문장에 관계절+분사구문+동격이 모두 들어가는 것
- 과도한 고급 어휘: 전체의 10% 이상이 고급 어휘면 부자연스러움
```

**slot_structure:**
```
## 분량 기준
- 8슬롯 | 8-12문장 | 145-200단어
- 슬롯당 1-2문장

## 담화 패턴
- 서론-본론-결론이 명확하고, 문단 내 문장들이 유기적으로 연결됩니다.
- 전환 표현(moreover, what's more, in particular 등)이 3종 이상 자연스럽게 분포합니다.
- 개인적 관점과 통찰이 담화에 녹아 있습니다.
- 복합 구조가 안정적으로 유지(sustain)됩니다.

## 슬롯 구조
[서론] 슬롯 1: 대상 소개 | 슬롯 2: 선택 이유/맥락
[본론] 슬롯 3: 배경/맥락 | 슬롯 4: 첫 번째 특징 (감각적 상세) | 슬롯 5: 느낌/인상 | 슬롯 6: 두 번째 특징 (비교/대조) | 슬롯 7: 활용/일상 속 역할
[결론] 슬롯 8: 개인적 의미/종합 평가
```

## 6. 실제 생성 결과물

### 결과 A — IH 등급, 부동산 롤플레이 (134단어)

**질문**: You want to find a new house. Call a real estate agent and ask 3-4 questions.

**학습자 입력**: (없음 — Level 3 확장 정책 적용)

> Hello, I'd like to speak with a real estate agent about finding a new house. Actually, I'm moving to this area for work, so I need to find a place that is convenient and comfortable for my family. First, could you tell me what kinds of houses are available right now and their price range? Also, I'd like to know about nearby schools, because I have two children and their education is important to me. However, if I want a house with a big yard, are there any options available? Oh, and do you offer any help with the moving process? That would be really helpful. Well, thank you so much for your help. I really appreciate it.

**구조 분석:**
- Introduction (2슬롯): 인사+용건, 상황 맥락 설명
- Body (4슬롯): 질문 1~4 (기본정보, 학교, 조건, 추가)
- Conclusion (1슬롯): 감사/마무리
- 연결어: Actually, First, Also, However, Oh, Well
- 필러: Actually, Oh, Well

### 결과 B — AL 등급, 음악 묘사 (201단어)

**질문**: Describe your favorite music genres and artists.

**학습자 입력**: (없음 — Level 3 확장 정책 적용)

> Music has always been a significant part of my life, and my taste in genres is actually pretty eclectic. I find myself drawn to pop, indie rock, and sometimes even a bit of jazz, depending on my mood. The reason I gravitate toward these genres is that each offers something unique—pop is vibrant and catchy, whereas indie rock tends to feel a bit more authentic and raw. ## 7. 구체적 질문 사항

### System Prompt 관련

1. **ROLE 설정** ("ACTFL OPIc script coach")이 적절한가요? OPIc 스크립트 생성에 더 효과적인 페르소나가 있을까요?

2. **4가지 Core Principles** (Level Honesty, Spoken English, Story Preservation, Slot Fidelity)의 우선순위가 맞나요? 빠진 원칙이 있나요?

3. **"A perfect IM2 script is better than an accidentally IH script"** — 이 등급 준수 원칙이 실제 OPIc 학습에 적합한가요? 혹시 "살짝 위 등급의 표현을 섞는 것"이 실전에서 더 유리한 경우가 있나요?

4. **CONTENT EXPANSION POLICY** (Level 1~3) — 사용자 입력이 없을 때(Level 3) AI가 자체 생성하는 방식이 학습에 효과적인가요? "내 이야기가 아닌 스크립트"를 외우는 것이 실전에 도움이 되나요?

### 등급별 제약 관련

5. **등급별 허용/금지 기준** — 위 6개 등급 규격서(섹션 5)를 보시고, 등급 간 경계가 ACTFL 기준에 부합하는지 평가해 주세요. 특히 각 등급의 "핵심 도구"(IL: 단문, IM1: and/but/so, IM2: because, IM3: when/if, IH: 담화 표지, AL: 관계절/분사구문) 구분이 적절한가요?

6. **등급별 필러 개수** (예: IH는 3~5개) — 이 기준이 적절한가요? 실제 OPIc 고득점 답변에서 관찰되는 필러 사용 패턴은 어떤가요?

7. **등급별 단어 수 기준** (예: IH 묘사 120~160단어) — 이 범위가 현실적인가요? 실제 IH 답변의 평균 단어 수는 어느 정도인가요?

### 스크립트 품질 관련

8. **결과 A (IH 부동산 RP)** — 이 스크립트가 IH 등급에 적합한가요? 등급을 벗어나는 표현이 있나요? 부족한 점은?

9. **결과 B (AL 음악 묘사)** — 이 스크립트가 AL 등급에 적합한가요? AL답게 만드는 요소와 부족한 요소는?

10. 두 스크립트 모두 **학습자 입력 없이** 생성되었습니다. 이 경우 AI가 만든 내용(Adele, Maroon 5 등)이 **범용적이고 외우기 쉬운 수준**인가요? 개선할 점은?

### 구조 설계 관련

11. **슬롯 구조** (서론-본론-결론, 슬롯별 역할 지정) — 이 방식이 OPIc 답변 구성에 효과적인가요? 다른 구조화 방식을 추천하시나요?

12. **답변 유형(10가지)별 차별화** — 묘사/루틴/경험/비교/롤플레이 등 유형마다 구조를 다르게 하고 있습니다. 실제 OPIc 채점에서 유형별로 기대하는 담화 구조가 다른가요?

13. **연결어 사용 패턴** — 결과 A는 `First → Also → However → Oh`로 이어집니다. 이 연결어 선택과 배치가 IH에 적합한가요? 더 자연스러운 패턴이 있나요?

### 개선 방향 관련

14. 현재 프롬프트에서 **가장 먼저 개선해야 할 부분** 1~2가지는 무엇인가요?

15. OPIc 스크립트 생성에서 **AI가 가장 자주 실수하는 패턴** (실제 학습 현장에서 관찰되는)이 있다면 어떤 것인가요? 프롬프트로 방지할 수 있을까요?
