## 1. 전문가 핵심 결론

> **"평가 엔진은 세계 최고 수준이지만, 학습 피드백 UX는 아직 연구실 단계."**
> **"정보는 훌륭하지만 학습자가 실제로 행동하게 만드는 구조가 아니다."**

가장 중요한 3가지 문제:

| # | 문제 | 설명 |
|---|------|------|
| 1 | **핵심 피드백이 접혀 있음** | 심층 분석, 종합 피드백이 collapsed 기본값 — 가장 치명적 |
| 2 | **평가자 언어 사용** | 체크박스 코드, ACTFL 내부 용어 — 학습자 언어가 아님 |
| 3 | **데이터 중심 UI** | Action(행동)보다 Data(수치)가 먼저 나옴 |

## 2. 질문별 전문가 답변 (모의고사 관련만)

### Q1. 정보 표시 순서

**현재**: 통과율 → 답변 원문 → 발음 → 체크박스 → 교정 → 심층분석(접힘)

**전문가 권장 순서** (실제 OPIc 튜터링 기반):

```
① AI 코치 한줄 평가 — "내용은 좋았지만 관사와 동명사 오류가 반복됩니다."
② 핵심 교정 3개 — 가장 중요한 오류만
③ 개선된 문장 예시 — "이렇게 말하면 AL스럽습니다"
④ 전체 피드백 (짧은 버전)
⑤ 내 답변 보기
⑥ 발음/유창성 데이터
⑦ 상세 분석 (접힘)
```

> **핵심 원칙: 학습자는 먼저 "무엇을 고쳐야 하는지"를 알고 싶다. 데이터는 그 다음.**

### Q2. 심층 분석 5섹션

**현재 구조**: overall_assessment → linguistic_analysis → communicative_effectiveness → proficiency_gap → recommendation

**전문가 평가**: 언어학적으로 매우 좋지만, UX 관점에서 academic함.

**권장 구조** (학습자 친화):

| 현재 (연구 논문 구조) | 권장 (코치 구조) |
|---------------------|----------------|
| overall_assessment | **잘한 점** (What you did well) |
| linguistic_analysis | **고칠 점** (What needs improvement) |
| communicative_effectiveness | **원인** (Why this happens — L1 간섭) |
| proficiency_gap | **목표** (How AL speakers say it) |
| recommendation | **훈련** (What to practice) |

### Q3. 체크박스 코드 노출

> **"절대 그대로 보여주면 안 됩니다."**

**현재**: `INT-1-1`, `INT-4-F2` — 학습자에게 전혀 의미 없음

**권장 방식**:

```
유창성    5/5  ✔
발음      3/4
문법      1/2  ▲
문장 구조  1/1  ✔
```

또는:

```
✔ 문장으로 말하기
✔ 의미 전달 가능
✔ 발음 이해 가능
▲ 문법 정확성 개선 필요
```

> **코드는 백엔드 전용.**

### Q4. 통과율 수치

**현재**: "체크박스 통과율 78%" — 학습자가 의미를 모름

**권장 방식**:

```
기초 수행(INT): 안정  ✔
심화 수행(ADV): 부족  ▲
```

또는:

```
Intermediate Tasks: Strong
Advanced Tasks: Weak
```

> ACTFL 구조를 직접 활용하는 것이 더 직관적.

### Q5. 교정 표시 방식

**현재**: 오류 리스트 나열

**권장**: **Inline correction** (본문 위 수정 — 학원 첨삭 방식)

```
I'd like to talk about ~~internet~~ → the internet
```

또는:

```
about internet  ❌
about the internet  ✔
```

### Q6. 발음 점수 활용

**전문가 의견**: Azure 수치 자체는 OPIc 점수와 직접 상관 없음.

**ACTFL 기준 발음 영향도**:
- NH: 영향 큼
- IL: 영향 중간
- **IM 이상: 영향 매우 작음** — "이해 가능성"만 중요

**현재**: `55/100` — 의미 없는 수치

**권장**:

```
발음 이해도: 괜찮음 / 가끔 어려움 / 자주 어려움
```

> 점수 대신 **listener effort** 관점으로 전환.

### Q7. 전문가 진단서 톤

**현재**: ACTFL evaluator report — 너무 academic

**한국 학습자 선호**: **코치 스타일**

```
지금은 IM2 수준입니다.

일상 주제에서는 안정적으로 말하지만,
Advanced 질문에서는 이야기 구조가 약합니다.

특히 과거 경험을 이야기할 때 시제가 자주 흔들립니다.

이 부분만 개선하면 IM3 가능성이 높습니다.
```

> **evaluator tone → coach tone** 전환.

### Q8. FACT 점수 체계

**전문가 평가**: 좋은 아이디어지만, 학습자가 "FACT = 무엇?" 모름.

**권장 라벨 변경**:

| 현재 | 권장 |
|------|------|
| F (Fluency) | 말하기 흐름 |
| A (Accuracy) | 문법 정확성 |
| C (Content) | 내용 풍부도 |
| T (Task) | 질문 수행 |

### Q9. 반복 실수 패턴 분석

> **"이 기능은 엄청난 가치가 있다. 절대 제거하면 안 된다. 오히려 더 강조해야 한다."**

**한국 OPIc 학습자 실제 반복 실수 Top 6**:

| # | 패턴 |
|---|------|
| 1 | 관사 누락 |
| 2 | 과거 시제 오류 |
| 3 | 단수/복수 혼동 |
| 4 | 연결어 부족 |
| 5 | 문장 반복 |
| 6 | very 남용 |

→ **현재 EF에서 recurring_mistakes를 DB에 저장하지 않는 문제를 반드시 수정해야 함.**

### Q10. 학습 루프

**현재**: 모의고사 → 스크립트 → 쉐도잉 (효과적)

**빠진 단계**: **Response Rebuild** (답변 재구성)

```
내 답변 → 교정 → 다시 말하기
```

> 이 단계가 학습 효과의 핵심 메커니즘.

### Q11. 개별 vs 종합 평가 역할

| 구분 | 역할 | 초점 |
|------|------|------|
| **개별 평가** | micro feedback | 문장/오류/발음 (언어) |
| **종합 평가** | macro strategy | 왜 Advanced에서 무너지는가, 어떤 유형이 약한가, 어떤 전략이 필요한가 (전략) |

> **개별 = 언어, 종합 = 전략** — 역할이 명확히 구분되어야 중복 방지.

### Q12. 등급 간 차이 설명

**핵심**: 등급 차이를 **예시로** 보여줘야 함

```
IM2 → IM3 핵심 차이

IM2: 문장 나열
  I went there. It was fun. I ate food.

IM3: 문단 구조
  First I went there with my friends.
  The place was crowded, but it was exciting.
  After that we tried local food.
```

> **구조(structure)의 차이**를 실제 답변 예시로 비교.

### Q13. 피드백 분량

**현재**: 1,000~1,500 words — **너무 김**

**학습자 실제 읽는 양**: 200~300 words

**권장 구조**:
- 요약: 100 words (항상 표시)
- 심층 분석: 접힘 (관심 있는 사용자만)

### Q14. 시각적 표현

**최적 비율**: 텍스트 60% + 시각 40%

**권장 시각 요소**:
- Skill radar chart (유창성/정확성/내용/구조/발음)
- Level progress bar
- Weakness heatmap

### Q15. 가장 시급한 개선

| 순위 | 개선 |
|------|------|
| **1** | 핵심 피드백 접힘 제거 — 심층 분석이 숨겨져 있는 것이 가장 큰 UX 문제 |
| **2** | 체크박스 코드 제거 — 학습자에게 전혀 의미 없음 |

### Q16. 빠진 피드백 요소

**매우 중요한 것 3가지**:

| # | 요소 | 내용 |
|---|------|------|
| 1 | **구조 평가** | 도입 / 전개 / 마무리 구분 + 평가 |
| 2 | **시간 분배** | Intro 10% / Body 70% / Conclusion 20% |
| 3 | **스토리 밀도** | 예시 수, 구체성 평가 |

### Q17. 실제 OPIc 결과와의 괴리

**위험 존재**: AI에서 IM3인데 실제 시험 IM1 가능성

**해결 방법**: **Confidence Band**

```
예상 등급: IM2 (±1 level)
또는
IM2 — Confidence: Medium
```

## 3. 5-Layer 피드백 아키텍처 (전문가 제안)

현재의 Data-first UI를 **Insight-first UI**로 전환해야 함.

```
Layer 1 (3초)   — Immediate Insight    "You are IM2 because sentences, not paragraphs."
Layer 2 (10초)  — Key Corrections      Top 3 mistakes만
Layer 3 (30초)  — Answer Improvement   wrong → correct → better (3단계)
Layer 4 (2분)   — Skill Analysis       레이더 차트, 영역별 분석
Layer 5 (선택)  — Deep Report          현재의 linguistic_analysis 등
```

> **대부분 사용자는 Layer 1~2만 봄. Layer 3까지 보면 상위 사용자.**

### Layer 3: Answer Improvement (핵심)

대부분 서비스가 없는 단계. 학습의 핵심 메커니즘.

```
Your answer:
  "I spend a lot of time to watch YouTube."

Fix:
  "I spend a lot of time watching YouTube."

Better answer:
  "I spend a lot of time watching YouTube, especially when I want to relax."
```

> **wrong → correct → better** 3단계가 핵심.
> 대부분 서비스는 wrong → correct까지만. 학습은 correct → better에서 일어남.

## 4. 학습 효과 3배 구조 (전문가 제안)

### 현재 vs 이상적 구조

```
현재:  시험 → 평가 → 끝

이상:  시험 → 피드백 → 답변 재구성 → 말하기 연습 → 비교
```

### Learning Loop Architecture

```
Test (모의고사)
  ↓
Diagnosis (AI 평가)
  ↓
Fix mistakes (핵심 교정 3개)
  ↓
Rebuild answer (AI가 개선 답변 생성)
  ↓
Speak again (다시 말하기)
  ↓
Compare (내 답변 vs 개선 답변)
```

> 이 루프가 학습 효과를 3배 이상 높임.

### Spaced Reinforcement (반복 약점 훈련)

**Mistake Memory** — AI가 개인 약점을 계속 추적:

```
당신은 관사를 자주 빠뜨립니다.

연습 문장:
  the internet
  the restaurant
  the movie theater
```

### Weakness Heatmap

```
Grammar       ██████████ 10 errors
Articles      ████████████████ 16 errors
Past tense    ██████ 6 errors
```

> 이것이 학습 동기를 만듦.

## 5. AL 코치 시스템 설계 (전문가 제안)

### IM → AL 핵심 변화

```
sentence speaking → paragraph speaking → structured speaking
```

> 등급 상승의 핵심은 **담화 구조 능력**.

### AL 코치 4개 모듈

```
① Diagnosis     — linguistic / discourse / task 약점 분류
② Target Setting — current IM2 → target IM3, gap 정의
③ Answer Reconstruction — error correction이 아닌 answer rebuilding
④ Guided Practice — 구조 힌트와 함께 다시 말하기
```

### Answer Reconstruction (가장 중요)

대부분 시스템: error correction까지만
AL 코치 시스템: **answer rebuilding**

```
학생 답변:
  I like watching movies.
  I watch them on weekends.
  It is fun.

AI 코치:
  First, I usually watch movies on weekends.
  For example, last week I watched a comedy film with my friends.
  Because of that, I had a great time.
```

> 학습은 evaluation이 아니라 **imitation**에서 일어남.

### AL 코치 핵심 분석 지표

- **Paragraph detection** (문단 구조 감지)
- **Transition usage** (연결어 사용 빈도/다양성)
- **Example density** (구체적 예시 수)
- **Story length** (서사 길이)

## 8. 프롬프트 개선 방향 (전문가 의견 기반)

### 개별 평가 프롬프트

**현재 유지**: checkboxes, corrections, deep_analysis 구조 자체는 양호

**추가/변경 필요**:

| 항목 | 변경 내용 |
|------|----------|
| deep_analysis 섹션명 | overall_assessment → **잘한점_고칠점**, proficiency_gap → **목표_예시** |
| **answer_improvement 추가** | GPT가 학생 답변을 기반으로 "개선된 버전" 1개 생성 |
| **structure_evaluation 추가** | 도입/전개/마무리 비율 + 시간 분배 평가 |
| **one_line_insight 추가** | 한줄 핵심 피드백 (AI 코치 코멘트) |
| corrections 제한 | Top 3개만 (한 번에 3개 이상 고치면 학습 효과 하락) |

### 종합 평가 프롬프트

**현재 유지**: overall_comments_ko, comprehensive_feedback, training_recommendations

**추가/변경 필요**:

| 항목 | 변경 내용 |
|------|----------|
| overall_comments_ko 톤 | evaluator → **coach** 스타일 |
