## 참고: 사용자가 발견한 이슈 패턴 (학습)

사용자는 검증 중 다음 같은 패턴으로 이슈를 보고함:
- "이 페이지에 ~ 버튼이 없네?" (기능 누락)
- "이 팝업은 시스템팝업으로 뜨고 있는거 맞아?" (디자인 일관성)
- "이건 뭐야? 왜 필요해?" (불필요한 요소 식별)
- 본질적 의도 질문: "내가 이걸 만드는 이유가 뭘까?" (**중요한 비전 질문 — 잘 답해야 함**)

이슈 보고 시:
1. 즉시 원인 파악
2. 옵션 제시 (단순 vs 정교)
3. 사용자 결정 후 빠르게 수정
4. 항상 한국어로 답
5. 검증은 사용자가 함
