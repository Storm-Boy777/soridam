## 직전 세션에서 해결한 이슈 3건

### 1. ✅ 그룹 상세 페이지 빈 화면
**원인**: `study_group_members.user_id`가 `auth.users(id)` FK라 `profiles`로 직접 join 안 됨. `profiles:user_id(...)` PostgREST 자동 join 실패 → SA 에러 → `notFound()`.

**해결**: 멤버 + profiles **분리 쿼리** + 클라이언트 join.
- 수정 파일:
  - `lib/actions/admin/study-groups.ts` (`getStudyGroupDetail`)
  - `app/opic-study/lobby/[sessionId]/page.tsx`
  - `app/opic-study/session/[sessionId]/page.tsx`

### 2. ✅ /opic-study에서 그룹 0개 표시 (RLS 재귀)
**원인**: `study_group_members` SELECT 정책 안에서 자기 테이블을 EXISTS로 참조 → PostgreSQL 무한 재귀 감지 → false 반환 → 본인 멤버십도 못 봄.

**해결**: 마이그레이션 051 — `SECURITY DEFINER` 함수 `is_study_group_member()` 도입. 모든 정책 단순화 (재귀 제거).

### 3. ✅ Home "멤버 4명" 하드코딩
**원인**: 디자인 시안의 mock 4명을 그대로 표시.

**해결**: `Home` 컴포넌트에 `nextSessionMemberCount` + `nextSessionMembers` props 추가. 페이지에서 실제 그룹 멤버 데이터 fetch + 매핑.
