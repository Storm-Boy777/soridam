> **이 문서의 목적**: 협업 Claude가 본 문서 하나로 오픽 스터디 모듈이 **어떤 기능이고 어떻게 동작하는지** 빠르게 파악하기 위한 인계 문서.
>
> **상세 사양**: `docs/설계/오픽스터디.md` (1450줄) 참조.
>
> **작성일**: 2026-05-01

## 0. 한 줄 요약

> 한국인 OPIc 학습자 대상 학습 플랫폼 **소리담**의 신규 모듈. 멤버들이 함께 모여 실시간으로 OPIc 기출 콤보를 학습하는 **스터디 룸 시스템**.

---

## 1. 프로젝트 배경

### 소리담 (Soridam)
- 한국인 대상 AI 기반 OPIc(영어 말하기 시험) 학습 플랫폼
- 도메인: https://soridamhub.com
- 기존 모듈: 시험후기 / 스크립트 / 모의고사 / 튜터링 / 쉐도잉

### 오픽 스터디 모듈 (이번 작업 대상)
- 별도 페이지 (`/opic-study`)로 운영되는 **멤버 전용 스터디 룸**
- 한 달 단위 등급별 그룹 운영 — 예: "5월 오픽 AL 스터디"
- 멤버 4~5명이 함께 모여 실시간으로 OPIc 기출 콤보 학습
- AI 일타강사가 멤버별 답변에 코칭 제공
- 완전 무료 (시스템 운영 비용 부담)

### 기존 모듈과의 관계
- 시험후기 모듈의 데이터를 **공유 활용** (콤보 출제 정보)
- 사용자는 일반 학습은 기존 모듈에서, 그룹 스터디는 이 신규 모듈에서

---

## 2. 핵심 기능 원칙

### 1) 점수/숫자 비공개
- 평가가 아닌 **학습 코칭** 컨셉
- 발음 점수, 등급, 통과율 등 일체 노출 X
- 모든 피드백은 자연어 (강점 / 다듬을 부분 / 팁)

### 2) 협력적 학습
- 멤버 간 점수 비교 X, 순위 X
- 4명 답변 결과를 나란히 보여주되 **"서로 배우는 인사이트 공유"** 컨셉

### 3) 일타강사 코칭 톤
- AI 강사 페르소나는 친근 + 자신감 있는 일타강사
- "오! 도입 굿이에요", "다음에 한 번 더!" 같은 구어체

### 4) 실시간 멤버 동기화
- 멤버 4~5명이 **같은 화면을 동시에** 봄
- 한 멤버가 콤보 선택하면 즉시 모두에게 반영

---

## 3. 사용자 모델

### 사용자 유형
| 유형 | 권한 |
|------|------|
| 관리자 | 그룹 생성, 멤버 등록, 그룹 관리 |
| 그룹 멤버 | 스터디 룸 입장, 답변, 코칭 받기 |
| 비멤버 | 신청 안내 페이지만 노출 |

### 멤버십
- 관리자가 회원 풀에서 직접 멤버 등록 (자가 가입 X)
- 한 사용자가 여러 그룹에 속할 수 있음
- 그룹은 한 달 단위 운영 (시작일~종료일)

### 그룹 운영 예시
```
"5월 오픽 AL 스터디"
├─ 기간: 2026-05-01 ~ 2026-05-31
├─ 목표 등급: AL (Advanced Low)
└─ 멤버: Alice, Bob, Carol, Dan (관리자가 등록)
```

## 4. 사용자 흐름

### 관리자
```
관리자 페이지 → 그룹 생성 (이름/등급/기간 입력)
           → 회원 풀에서 멤버 검색 + 다중 선택
           → 그룹 활성화
```

### 멤버 (스터디 진행)
```
/opic-study 진입
  ↓
내가 속한 그룹 카드 표시
  ↓
"새 세션 시작" 클릭 → 세션 룸 입장 (immersive 레이아웃)
  ↓
[Step 1] 모드 선택 (온라인 / 오프라인)
[Step 2] 카테고리 선택 (일반 / 롤플레이 / 어드밴스)
[Step 3] 주제 선택 (콤보 빈도 높은 순)
[Step 4] 콤보 선택 (출제 빈도 높은 순)
[Step 5] AI 일타강사 가이드 (오늘 학습 인트로)
[Step 6] 질문 루프 (콤보 안 N개 질문 = 보통 3개)
   ├─ 6-1: 멤버 중 한 명이 "내가 답변" 자임
   ├─ 6-2: 그 멤버 녹음 / 다른 멤버 청취
   ├─ 6-3: AI 코칭 생성 중 (자동)
   ├─ 6-4: AI 코칭 카드 표시 (모든 멤버 동시)
   ├─ 6-5: 모든 멤버 답변까지 6-1~6-4 반복
   ├─ 6-6: 4명 코칭 동시 비교 + 토론
   └─ 6-7: 다음 질문으로 진행
[Step 7] 종료/요약
```

### 비멤버
```
/opic-study 진입 → "스터디 멤버 전용" 안내 + 신청 이메일
```

## 5. 두 가지 모드

| | 온라인 모드 | 오프라인 모드 |
|---|------------|------------|
| 참여 방식 | 각자 다른 위치, 각자 디바이스 | 한 자리에 모여 한 디바이스로 |
| 화면 동기화 | 실시간 동기화 | 동기화 없음 |
| 녹음 | 각자 디바이스 마이크 | 한 디바이스 마이크 (발화자 교대) |
| 인터넷 | 필수 | 필수 (AI 코칭 위해) |

→ 모드는 세션 시작 시 선택, **중간 전환 가능** (헤더 토글)

## 6. 화면 카탈로그 (총 18화면)

상세 와이어프레임은 [`docs/설계/오픽스터디.md`](설계/오픽스터디.md) "화면별 와이어프레임" 섹션 참조.

### A. 진입 페이지 4종 (`/opic-study`)
1. 멤버 + 활성 그룹 1개 (그룹 카드 + 새 세션 시작)
2. 멤버 + 진행 중 세션 있음 (합류 안내)
3. 멤버 + 활성 그룹 0개 (지난 이력 + 안내)
4. 비멤버 + 신청 안내

### B. 세션 룸 (`/opic-study/session/[sessionId]`, 10화면)
- Step 1: 모드 선택
- Step 2: 카테고리 선택
- Step 3: 주제 선택
- Step 4: 콤보 선택 (콤보 빈도 + 질문별 등장률 + "이미 학습" 표시)
- Step 5: AI 일타강사 가이드
- Step 6-1: 발화자 선정 (4명 카드)
- Step 6-2: 답변 녹음 (본인/타인 화면 분기)
- Step 6-3: AI 코칭 생성 중
- Step 6-4: AI 코칭 카드 (개별)
- Step 6-6: 4명 코칭 동시 비교 + 토론
- Step 7: 종료/요약

### C. 관리자 3페이지 (`/admin/study-groups`)
1. 그룹 목록 + 통계
2. 새 그룹 생성 (폼 + 멤버 등록)
3. 그룹 상세 (멤버 관리 + 세션 이력)

### D. 세션 이력 1페이지 (`/opic-study/history`)
- 그룹의 완료된 세션 시간역순 목록

### 공통 컴포넌트
- 세션 룸 헤더 (breadcrumb + 모드 토글 + 접속 멤버 표시 + 종료)
- 연결 끊김 배너
- 토스트/알림
- 모달/확인 다이얼로그
- AI 코칭 카드 (재사용)

## 7. 핵심 기능 동작

### 실시간 동기화 (온라인 모드)
- 멤버 중 누구나 컨트롤 가능 — 호스트 없음
- 먼저 누른 사람의 선택이 모두에게 반영 (first-write-wins)
- DB의 세션 row 한 개를 모든 멤버가 구독, 변경 시 즉시 푸시

### 발화권 (순차 답변)
- 한 질문에 대해 멤버들이 **한 명씩 차례로** 답변
- 답변자: 마이크 ON, 녹음 진행
- 다른 멤버: 마이크 OFF, 답변 청취
- 답변 완료 후 누구나 "다음 발화자" 클릭 가능

### AI 코칭 카드 (브랜드 핵심)
멤버 답변 → Whisper STT + Azure 발음 평가 → GPT 코칭 멘트 생성 → 카드 표시

```
🎤 Alice의 답변
"I'm really into hip-hop these days..."

🎓 일타강사 한마디
오! 도입 진짜 깔끔했어요. "I'm really into hip-hop these days"
같은 표현 자연스러워요. 근데 중간에 'go to concerts' 말한
다음에 시제가 살짝 흔들렸어요...

✓ 잘한 점
  - 자연스러운 도입 표현
  - 풍부한 어휘 ('genre' 같은)
  - 발음 또렷

⚠ 다듬을 부분
  - 현재 → 과거 넘어갈 때 시제 일관성
  - 'rhythm' 한 번 더 또박또박

💡 다음 답변 팁
  - "I usually..., and last month..." 패턴
```

→ **점수/숫자 일체 X**. 자연어 코칭 멘트만.

### 4명 동시 비교 (Step 6-6)
- 멤버 4명 카드 나란히 (각 카드: 강점 / 다듬을 부분 / 팁)
- "함께 짚어보는 시간 (5분)" 타이머 (자동 진행 X, 안내용)
- 카드 클릭 시 전체 코칭 모달

### 모드 토글
- 헤더에 상시 노출
- 누구나 클릭 → 즉시 모든 멤버 화면 분기 변경

### 콤보 정렬 (학습 우선순위)
- 토픽: 콤보 빈도 높은 순 (예: 음악 19콤보, 집 16콤보)
- 콤보: 같은 질문 세트 반복 출제 빈도 높은 순
- 학습 이력 표시: 다른 콤보에서 이미 학습한 질문 → "✓ 학습" 뱃지

## 8. 사용 환경

### 디바이스
- **모바일 우선** (소리담 사용자 모바일 비중 높음)
- PC도 지원 (스터디 호스트가 큰 화면 보고 진행하는 경우)

### 기술 스택 (호환성 참고)
- Next.js (App Router) + Tailwind CSS
- TypeScript
- Supabase Realtime (postgres_changes + presence)
- 마이크 녹음: 브라우저 MediaRecorder API

### 임머시브 레이아웃 주의
세션 룸은 모바일 풀스크린(`h-dvh`) 레이아웃. 스크롤 영역 페인팅 이슈가 있어 특정 패턴 사용 필요. 자세한 건 `CLAUDE.md` 참조.

## 9. 일타강사 페르소나 (페르소나 정의)

AI 강사가 가이드(Step 5)와 코칭(Step 6-4) 모두 담당. 한 캐릭터의 일관된 톤이 핵심.

### 캐릭터
- 5~10년차 OPIc 전문 일타강사
- 학생들이 가장 좋아하는 강사
- 솔직, 따뜻함, 자신감
- 격려 우선, 약점도 부드럽게
- "다음에 더 잘하는 법"에 집중

### 발화 예시 (구어체)
- "오늘 음악 콤보 AL로 잡고 갈게요"
- "오! 도입 진짜 깔끔했어요"
- "다음엔 시제 한 번 더 챙기면 훨씬 매끄럽게 나올 거예요"
- "자, 멤버들끼리 답변 끝나면 서로 짚어주세요. 시작합시다!"

## 10. 참조 데이터 (실제 콘텐츠 예시)

### 그룹
- "5월 오픽 AL 스터디" / 2026-05-01 ~ 2026-05-31 / 멤버 4명

### 콤보 예시 (음악 일반 콤보 A — 출제 53%)
- Q1 (묘사): "What kind of music do you listen to? Who are some of your favorite musicians or composers?"
- Q2 (비교): "When did you first become interested in music? Tell me how your interest in music developed from your childhood until today."
- Q3 (과거경험): "Could you think back to a particularly memorable time when you heard live music? Describe that experience in detail."

### 일타강사 가이드 멘트 예시
> 오늘 음악 콤보 AL로 잡고 갈게요. 이 콤보, 음악이 일반 자리에 나올 때 53% 확률로 등장하는 정형화된 패턴이에요.
>
> 핵심은 '추상적 묘사 → 구체적 비교 → 임팩트 있는 경험'으로 자연스럽게 흐르는 거예요.
>
> 자, 멤버들끼리 답변 끝나면 서로 짚어주세요. 시작합시다!

### 코칭 카드 예시
- 메인 멘트: "오! Alice, 도입 진짜 깔끔했어요. 'I'm really into hip-hop these days' 같은 표현 자연스러워요. 근데 중간에 시제가 살짝 흔들렸어요. 다음엔 시제 한 번 더 챙기면 훨씬 매끄럽게 나올 거예요."
- 강점: 자연스러운 도입 / 풍부한 어휘 / 발음 또렷
- 다듬을 부분: 시제 일관성 / 'rhythm' 한 번 더 또박
- 팁: "I usually..., and last month..." 패턴
