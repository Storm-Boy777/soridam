# 오픽 스터디 모듈 설계

> **문서 목적**: 스터디 그룹 멤버들이 실제 OPIc 기출 콤보를 함께 학습하는 실시간 동기화 스터디 시스템의 설계 기준을 정의한다.
> **작성일**: 2026-05-01
> **참조 데이터**: `submissions` + `submission_combos` + `submission_questions` (시험후기 모듈 SSOT 활용)

---

## 핵심 설계 결정 요약

| # | 항목 | 결정 |
|---|------|------|
| O-1 | 모듈 성격 | 별도 페이지 (`/opic-study`), 스터디 멤버 전용 |
| O-2 | 그룹 운영 | 월별 + 등급별 운영 ("5월 오픽 AL 스터디") |
| O-3 | 멤버십 | 관리자가 회원 풀에서 직접 등록 (자가 가입 X) |
| O-4 | 학습 단위 | 1세션 = 1콤보 (콤보 안의 N개 질문 모두 학습) |
| O-5 | 출제 데이터 | 승인된 시험후기 풀 (시험후기 모듈과 SSOT 공유) |
| O-6 | 콤보 정렬 | 토픽 — 콤보 빈도순 / 콤보 — 같은 질문 세트 반복 빈도순 |
| O-7 | 컨트롤 | 호스트 없음, 누구나 컨트롤 (first-write-wins) |
| O-8 | 녹음 방식 | 순차 (한 명씩, 다른 멤버 청취) + "다음 발화자" 수동 클릭 |
| O-9 | AI 가이드 | 콤보 선택 직후 1회 생성 (GPT-4.1 일타강사 톤) |
| O-10 | AI F/B | 멤버별 답변 끝나면 즉시 F/B 생성, 전 멤버 결과 화면 공유 |
| O-11 | F/B 입력 | Whisper(STT) + Azure(발음평가) + 콤보/등급 컨텍스트 종합 → GPT-4.1 |
| O-12 | F/B 톤 | 점수/체크박스 ❌ → 일타강사 코칭 멘트 (강점/개선/팁) |
| O-12-1 | **점수 비공개** | **발음/등급/숫자 점수는 사용자 UI에 노출 X** (내부 GPT 입력 전용). 시험 평가 X, 학습 코칭 O |
| O-13 | 비용 부담 | **스터디 완전 무료** — Whisper/Azure/GPT 모두 시스템 API 비용 |
| O-14 | 모드 | 온라인(실시간 동기화) / 오프라인(단일 디바이스) 토글 |
| O-15 | 비멤버 진입 | 신청서 안내 페이지 (단순 안내, 추후 정식 신청 폼 확장) |
| O-16 | 등급 적용 | 그룹 등급 자동 적용 (세션 중 등급 선택 단계 없음) |
| O-17 | 학습 이력 | 콤보/질문 단위 트래킹 (다른 콤보의 같은 질문 → "이미 학습" 표시) |
| O-18 | 데이터 자기강화 | 시험후기 누적 → 빈도 정확도 자동 상승 |

## 학습 흐름 큰 그림

### 사용자 여정

```
[관리자]
  └─ 스터디 그룹 생성 ("5월 오픽 AL 스터디", 등급=AL, 기간=5/1~5/31)
      └─ 회원 풀에서 멤버 등록 (5명)

[비멤버]
  /opic-study 진입 → "스터디 멤버가 아닙니다" + 신청 안내

[멤버]
  /opic-study 진입
    └─ 내가 속한 그룹 카드 표시
        └─ "스터디 시작" 클릭
            ├─ 새 세션 만들기 / 진행 중 세션 합류
            └─ 세션 룸 (immersive)
                ├─ Step 1: 모드 선택 (온라인/오프라인)
                ├─ Step 2: 카테고리 선택   ┐
                ├─ Step 3: 주제 선택        ├─ first-write-wins
                ├─ Step 4: 콤보 선택        ┘
                ├─ Step 5: AI 일타강사 가이드 (자동 생성, 1회)
                ├─ Step 6: 질문 루프 (콤보 안 N개 질문)
                │    ┌─ 6-1: 발화자 선정 ("내가 답변할게요")
                │    ├─ 6-2: 답변 녹음 (해당 발화자만 마이크 ON, 나머지 청취)
                │    ├─ 6-3: 즉시 AI F/B 생성 (Whisper STT + Azure 발음 + GPT-4.1)
                │    ├─ 6-4: 일타강사 F/B 카드 표시 (모든 멤버에게 공유)
                │    ├─ 6-5: 모든 멤버 답변 완료까지 6-1~6-4 반복
                │    ├─ 6-6: 4명 F/B 카드 동시 비교 + 인사이트 토론 (타이머)
                │    └─ 6-7: "다음 질문" 또는 "다음 발화자" 클릭
                └─ Step 7: 종료 (학습 이력 저장)
```

### 한 콤보 학습 = N명 × M질문

예: 멤버 4명 × 음악 일반 콤보 A (3 질문)
- Q1 묘사 → Alice 답변 → 평가 → Bob 답변 → 평가 → Carol 답변 → 평가 → Dan 답변 → 평가 → 4명 결과 동시 공유 → F/B 토론 → 다음 질문
- Q2 비교 → 같은 흐름
- Q3 과거경험 → 같은 흐름
- 종료

## 모드 비교

| | 온라인 모드 | 오프라인 모드 |
|---|------------|------------|
| **참여 방식** | 각자 다른 위치, 각자 디바이스 | 한 자리에 모여 한 디바이스로 |
| **화면 동기화** | Supabase Realtime (전 멤버 실시간) | 동기화 없음 |
| **녹음** | 각자 디바이스 마이크 | 한 디바이스 마이크 (발화자 교대) |
| **AI 평가** | 멤버별 개별 평가 | 멤버별 개별 평가 (한 디바이스에서 순차) |
| **인터넷** | 필수 | 필수 (AI 평가 위해) |
| **진행 흐름** | 동일 | 동일 |

> **핵심**: 화면 동기화만 다르고 학습 흐름은 동일. 모드는 세션 시작 시 선택, 중간 전환 가능.

## 데이터 모델

### 신규 테이블 4개

> **FK 참조 표준**: 본 프로젝트는 모든 user 참조에 `auth.users(id)` 사용 (profiles 테이블은 metadata 확장용, FK 대상 X).

#### 1. `study_groups` — 스터디 그룹 (월별+등급별)

```sql
CREATE TABLE study_groups (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name            text NOT NULL,                              -- "5월 오픽 AL 스터디"
  target_level    text NOT NULL,                              -- 'IM2'|'IH'|'AL' 등
  start_date      date NOT NULL,
  end_date        date NOT NULL,
  status          text NOT NULL DEFAULT 'active',             -- 'active'|'closed'
  description     text,
  created_by      uuid NOT NULL REFERENCES auth.users(id),    -- 관리자
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now()
);

CREATE INDEX idx_study_groups_status ON study_groups(status);
CREATE INDEX idx_study_groups_dates ON study_groups(start_date, end_date);
```

#### 2. `study_group_members` — 멤버십

```sql
CREATE TABLE study_group_members (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  group_id        uuid NOT NULL REFERENCES study_groups(id) ON DELETE CASCADE,
  user_id         uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  display_name    text,                                       -- 그룹 내 표시명 (선택)
  joined_at       timestamptz NOT NULL DEFAULT now(),
  UNIQUE(group_id, user_id)
);

CREATE INDEX idx_sgm_user ON study_group_members(user_id);
CREATE INDEX idx_sgm_group ON study_group_members(group_id);
```

#### 3. `opic_study_sessions` — 한 번의 스터디 세션 (= 룸)

```sql
CREATE TABLE opic_study_sessions (
  id                       uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  group_id                 uuid NOT NULL REFERENCES study_groups(id) ON DELETE CASCADE,

  -- 모드
  online_mode              boolean NOT NULL DEFAULT true,

  -- 진행 단계 (실시간 동기화 대상)
  step                     text NOT NULL DEFAULT 'mode_select',
  -- 'mode_select' | 'category_select' | 'topic_select' | 'combo_select'
  -- | 'guide' | 'recording' | 'feedback_share' | 'discussion' | 'completed'

  -- 학습 콘텐츠 선택 상태
  selected_category        text,                                -- 'general' | 'roleplay' | 'advance'
  selected_topic           text,
  selected_combo_sig       text,                                -- 'qid1|qid2|qid3'
  selected_question_ids    text[] NOT NULL DEFAULT '{}',        -- 콤보 안 질문 ID 순서

  -- 진행 상태
  current_question_idx     integer NOT NULL DEFAULT 0,          -- 콤보 안 N번째 질문
  current_speaker_user_id  uuid REFERENCES auth.users(id),      -- 지금 답변 중인 멤버

  -- AI 가이드 (1회 생성, 캐싱)
  ai_guide_text            text,
  ai_guide_key_points      jsonb,                               -- ["...", "...", "..."] 3개. F/B 입력에 일관성 위해 사용
  ai_guide_generated_at    timestamptz,

  -- 메타
  status                   text NOT NULL DEFAULT 'active',      -- 'active' | 'completed' | 'abandoned'
  started_at               timestamptz NOT NULL DEFAULT now(),
  ended_at                 timestamptz,
  created_by               uuid NOT NULL REFERENCES auth.users(id),
  updated_at               timestamptz NOT NULL DEFAULT now()   -- 트리거로 자동 갱신
);

CREATE INDEX idx_oss_group ON opic_study_sessions(group_id);
CREATE INDEX idx_oss_status ON opic_study_sessions(status);
```

#### 4. `opic_study_answers` — 멤버별 답변 + 평가 결과

```sql
CREATE TABLE opic_study_answers (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id          uuid NOT NULL REFERENCES opic_study_sessions(id) ON DELETE CASCADE,
  user_id             uuid NOT NULL REFERENCES auth.users(id),
  question_id         text NOT NULL REFERENCES questions(id),  -- 'SEL_MUS_N_C2_01'
  question_idx        integer NOT NULL,                        -- 콤보 안 0/1/2 (순서)

  -- 음성/STT
  audio_url           text,
  transcript          text,

  -- 발음 평가 (Azure) — 내부 GPT 입력 전용, UI 노출 X
  pronunciation_score jsonb,                                   -- { accuracy, fluency, prosody, words[] }

  -- AI F/B (일타강사 코칭 — 점수/숫자 X, 질적 코칭 멘트만)
  feedback_result     jsonb,                                   -- { feedback_text, strengths[], improvements[], tips[] }
  feedback_generated_at timestamptz,

  -- 메타
  created_at          timestamptz NOT NULL DEFAULT now(),
  UNIQUE(session_id, user_id, question_idx)                    -- 한 멤버는 한 질문에 한 번만 답변
);

CREATE INDEX idx_osa_session ON opic_study_answers(session_id);
CREATE INDEX idx_osa_user ON opic_study_answers(user_id);
CREATE INDEX idx_osa_qidx ON opic_study_answers(session_id, question_idx);
```

### 학습 이력 — 별도 테이블 vs 뷰

학습 이력 (콤보 학습 완료, 다른 콤보의 같은 질문 = "이미 학습")은 **`opic_study_answers`에서 직접 집계**.
별도 테이블 불필요.

```sql
-- 한 멤버가 학습한 모든 question_id 집합
SELECT DISTINCT a.question_id
FROM opic_study_answers a
WHERE a.user_id = $1
  AND a.feedback_result IS NOT NULL;

-- 한 그룹에서 학습 완료한 콤보 시그니처
SELECT DISTINCT s.selected_combo_sig
FROM opic_study_sessions s
WHERE s.group_id = $1
  AND s.status = 'completed';
```

## RLS 정책 (큰 그림)

| 테이블 | SELECT | INSERT/UPDATE/DELETE |
|--------|--------|---------------------|
| `study_groups` | 관리자 전체 / 멤버 본인 그룹 | 관리자만 |
| `study_group_members` | 관리자 / 본인 그룹 멤버 | 관리자만 |
| `opic_study_sessions` | 그룹 멤버 (자기 그룹 세션만) | 그룹 멤버 (Realtime 동기화 위해 UPDATE 허용) |
| `opic_study_answers` | 같은 세션 모든 멤버 (답변 공유) | 본인 답변만 INSERT, 본인만 UPDATE |

> **핵심**: 세션 룸 안에서는 **모든 멤버가 같은 데이터를 읽고, 자기 답변만 쓴다**. 세션 row UPDATE는 누구나 가능 (first-write-wins).

## Realtime 동기화 구조

### 구독 대상

```typescript
// 세션 룸 입장 시
const channel = supabase
  .channel(`opic-study-session:${sessionId}`)
  .on('postgres_changes', {
    event: 'UPDATE',
    schema: 'public',
    table: 'opic_study_sessions',

  }, handleSessionUpdate)
  .on('postgres_changes', {
    event: 'INSERT',
    schema: 'public',

  .on('postgres_changes', {

```

## 동시성 (first-write-wins) 처리

### 시나리오: 두 멤버가 동시에 콤보 선택

```sql
-- SA 함수: selectCombo()
UPDATE opic_study_sessions
SET selected_combo_sig = $1,
    selected_question_ids = $2,
    step = 'guide'
WHERE id = $sessionId
  AND step = 'combo_select'           -- 단계 가드
  AND selected_combo_sig IS NULL      -- 아직 선택되지 않은 경우만
RETURNING *;

-- 결과:
--   row 반환됨 → 내가 선택 성공, 모든 멤버에게 Realtime 푸시
--   row 없음 → 다른 멤버가 먼저 선택. 클라이언트에서 갱신된 상태로 동기화
```

같은 패턴이 카테고리/주제 선택, "다음 질문" 진행에도 적용.

### 시나리오: 발화자 자임 ("내가 답변할게요")

```sql
UPDATE opic_study_sessions
SET current_speaker_user_id = $userId
WHERE id = $sessionId
  AND step = 'recording'
  AND current_speaker_user_id IS NULL  -- 다른 사람이 자임 안 한 경우만
RETURNING *;
```

## 모듈 외부 연동

### 시험후기 모듈 (출제 데이터 SSOT)
- `submissions` (status='complete' AND exam_approved='approved')
- `submission_combos`, `submission_questions`
- → 콤보 빈도 집계 SQL은 SA에서 실시간 또는 캐싱

### questions 마스터 (471행)
- 질문 텍스트, 영문/한글, question_type 표시

### 크레딧 시스템 — 완전 무료
- **사용자 크레딧 차감 없음** (잔액 체크/차감 코드 불필요)
- Whisper STT + Azure 발음 평가 + GPT-4.1 F/B 모두 **시스템 API 비용 부담**
- `api_usage_logs`에는 기록 (운영자 비용 모니터링용)
- 운영 정책: 스터디는 학습 커뮤니티 도구, 무료 정책으로 사용자 진입 장벽 제거

### 모의고사 EF 재사용 가능성
- `mock-test-process` (Whisper STT + Azure 발음): **로직 재사용 가능** (별도 EF로 추출 또는 호출)
- `mock-test-eval` (체크박스 74개 평가): **재사용 X** — 스터디용 신규 EF `opic-study-feedback` 작성

## AI F/B 생성 파이프라인 (신규 EF: `opic-study-feedback`)

```
멤버 답변 (마이크 녹음, webm)
    │
    ▼
[Storage 업로드]  Supabase Storage 'opic-study-recordings' 버킷
    │
    │ (audio_url 발급)
    │
    ├──────────────────────────┬──────────────────────────┐
    ▼                          ▼
[Whisper API]               [Azure Speech SDK]
영어 STT                    발음 평가
  ↓ transcript              ↓ pronunciation_score
                              - accuracy / fluency / prosody
                              - 단어/음운별 점수
    │                          │
    └──────────────┬───────────┘
                   │
                   ▼
        ┌──────────────────────────────────────┐
        │  GPT-4.1 일타강사 F/B 생성             │
        │                                       │
        │  입력 (CO-STAR 프롬프트):              │
        │  - 콤보 정보 (질문, 카테고리, 등급)    │
        │  - AI 가이드 (일관성)                 │
        │  - transcript (어휘/문장/흐름)         │
        │  - pronunciation_score (발음)         │
        │                                       │
        │  출력 (JSON):                         │
        │  {                                    │
        │    feedback_text: "오! 도입 깔끔...", │
        │    strengths: [...],                  │
        │    improvements: [...],               │
        │    tips: [...]                        │
        │  }                                    │
        └──────────────────────────────────────┘
                   │
                   ▼
[opic_study_answers UPDATE]
  - audio_url, transcript
  - pronunciation_score (Azure 결과 그대로)
  - feedback_result (GPT 결과)
  - feedback_generated_at
                   │
                   ▼
[Supabase Realtime] 모든 멤버 화면에 F/B 카드 푸시
```

### F/B 결과 JSON 구조 (`feedback_result`)

```typescript
interface FeedbackResult {
  // 메인 코칭 멘트 (1~2문단, 일타강사 톤)
  feedback_text: string;

  // 강점 (1~3개, 구체적)

  improvements: string[];       // ["시제 일관성 보강", "rhythm 발음 또렷이"]

  tips: string[];               // ["I usually..., and last month... 패턴 적용"]
```

### `pronunciation_score` JSON 구조 (Azure 원본)

```typescript
interface PronunciationScore {
  accuracy: number;             // 0~100
  fluency: number;
  prosody: number;
  completeness: number;
  pron_score: number;           // 종합

  // 단어별 디테일 (선택)
  words?: Array<{
    word: string;
    accuracy: number;
    error_type?: string;        // 'mispronunciation' | 'omission' | ...
  }>;
}
```

> **이 두 컬럼이 분리된 이유**: Azure 원본은 GPT 입력용으로 보존하되 **사용자 UI에 노출 X**. GPT는 점수를 보고 "rhythm 발음 또렷이"처럼 자연어로 풀어서 코칭. 사용자는 숫자가 아닌 코칭 멘트만 봄.

## 엣지 케이스 + 인터랙션 처리

### 1. 멤버 이탈 (presence offline)

| 상황 | 처리 |
|------|------|
| 답변 도중 이탈 (Alice 답변 중 창 닫음) | `current_speaker_user_id = NULL` 자동 복원, "Alice 연결 끊김 — 다음 발화자 선정" 안내 |
| 대기 중 이탈 (Bob이 발화권 안 가진 상태에서 이탈) | 답변 완료 카운트에서 제외, 남은 멤버로 진행 |
| 모든 멤버 이탈 | 세션 자동 'abandoned' 처리 (10분 무활동 기준) |

### 2. 답변 도중 재녹음

- **녹음 중 [다시] 버튼**: MediaRecorder 정지 후 처음부터 재시작 (Storage 업로드 전이라 DB 변경 X)
- **F/B 받은 후 재녹음**: 불가 (한 멤버는 한 질문에 한 번만 답변 — UNIQUE 제약). 다음 콤보에서 같은 질문 만나면 다시 도전 가능

### 3. F/B 생성 실패

| 실패 지점 | 복구 방식 |
|---------|----------|
| Storage 업로드 실패 | 클라이언트 재시도 (3회) + 사용자에게 "다시 답변" 안내 |
| Whisper STT 실패 | EF에서 자동 재시도 (1회) → 실패 시 transcript 비워두고 GPT는 발음 데이터만으로 코칭 |
| Azure 발음 실패 | EF에서 자동 재시도 (1회) → 실패 시 transcript만으로 GPT 코칭 |
| GPT-4.1 실패 | EF에서 자동 재시도 (2회) → 최종 실패 시 "잠시 후 다시" 토스트 + 세션 진행 가능 ([재시도] 버튼) |

### 4. 답변 패스 / 미참여

- **명시적 패스 기능 없음** (모든 멤버 답변이 핵심 흐름)
- 단, 멤버 이탈로 답변 못 한 경우 → 남은 멤버로 진행, 종료 화면에 "Alice 미참여" 표시

### 5. 모드 중간 전환

```
[헤더 우측] 🌐 온라인 ⇄ 🏠 오프라인 (토글)
```
- 누구나 토글 가능 → `online_mode` UPDATE → 모든 멤버 화면 분기 변경
- **온라인 → 오프라인 전환 시**: 모든 멤버에게 "한 디바이스 모드로 전환됨" 안내. 마이크 권한이 한 디바이스에만 활성화됨
- **오프라인 → 온라인 전환 시**: "각자 디바이스로 진행" 안내

### 6. 인터넷 끊김

- Realtime 구독 끊김 감지 → "연결이 불안정해요. 재연결 중..." 토스트
- 재연결 시 `opic_study_sessions` 최신 상태 다시 fetch → 화면 동기화
- 답변 도중 끊김 → 녹음 데이터는 클라이언트 보존, 재연결 후 업로드 재시도

### 7. 동시 클릭 충돌

| 시나리오 | 처리 |
|---------|------|
| 두 멤버 동시 카테고리 선택 | UPDATE WHERE 가드 → 한 명만 성공, 다른 멤버는 갱신된 상태로 자동 동기화 |
| 두 멤버 동시 "내가 답변" 클릭 | 위와 동일. 발화권은 한 명에게만 |
| 두 멤버 동시 "다음 질문" 클릭 | 멱등성 — `current_question_idx` 증가는 한 번만 (현재 질문에서 +1만 허용) |

### 8. 토론 시간 타이머

- Step 6-6의 5분 타이머는 **자동 진행 X, 안내용**
- 시간 끝나도 자동 다음 질문으로 안 넘어감 — 누군가 "다음 질문" 눌러야 진행
- 토론 더 길게 하고 싶으면 그냥 안 누르고 계속 토론

## Step 전환 로직 (인터랙션 명확화)

### 단계 자동 전환 vs 수동 진행

| 사용자 액션 | 결과 (UPDATE) | 다음 step |
|----------|--------------|----------|
| 모드 선택 | `online_mode + step` | `category_select` |
| 카테고리 선택 | `selected_category + step` | `topic_select` |
| 주제 선택 | `selected_topic + step` | `combo_select` |
| 콤보 선택 | `selected_combo_sig + selected_question_ids + step` | `guide` (가이드 생성 EF 트리거) |
| 가이드 "시작하기" | `step` | `recording` |
| "내가 답변" 클릭 | `current_speaker_user_id` | `recording` 유지 (substep만 변경) |
| 답변 완료 → F/B 도착 | EF가 `opic_study_answers.feedback_result` UPDATE | `recording` 유지 |
| "다음 발화자" 클릭 | `current_speaker_user_id = NULL` | `recording` 유지 |
| **모든 멤버 답변 완료** | (자동 감지 — SA에서 체크) | `feedback_share` |
| "다음 질문" 클릭 | `current_question_idx + 1`, `current_speaker_user_id = NULL` | `recording` (다음 질문) 또는 `discussion`/`completed` |

### "모든 멤버 답변 완료" 감지

```typescript
// SA: submitAnswer 처리 후
const session = await getSession(sessionId);
const memberCount = await countActiveMembers(session.group_id);  // 이탈자 제외
const answeredCount = await countAnswers(sessionId, currentQuestionIdx);

if (answeredCount >= memberCount) {
  await supabase
    .from('opic_study_sessions')
    .update({ step: 'feedback_share' })
    .eq('id', sessionId)
    .eq('current_question_idx', currentQuestionIdx)
    .eq('step', 'recording');
}
```

> **이탈자 제외**: presence channel에서 SUBSCRIBED 멤버만 카운트. 또는 직전 30초간 활동(presence track) 있는 멤버만.

### 콤보 끝 (마지막 질문 답변 완료) 감지

```typescript
// "다음 질문" 클릭 시
const isLastQuestion = currentQuestionIdx + 1 >= selected_question_ids.length;
const nextStep = isLastQuestion ? 'completed' : 'recording';

await supabase.from('opic_study_sessions').update({
  current_question_idx: currentQuestionIdx + 1,
  current_speaker_user_id: null,
  step: nextStep,
  ended_at: isLastQuestion ? new Date().toISOString() : null,
  status: isLastQuestion ? 'completed' : 'active',
}).eq('id', sessionId);
```

## AI 일타강사 페르소나 (공통)

가이드와 F/B 모두 **하나의 일관된 페르소나**가 처음부터 끝까지 코칭한다는 인상이 핵심.

### 캐릭터 설정
```
이름: (UI 노출 X — 그냥 "일타강사")
역할: 5~10년차 OPIc 전문 강사. 학생들이 가장 좋아하는 일타.
성격:
  - 솔직하고 직설적이지만 따뜻함
  - 학생의 강점을 먼저 짚어주고 격려
  - 약점도 부드럽게, 다음 답변 팁으로 자연스럽게 연결
  - 점수보다 "다음에 더 잘하는 법"에 집중
  - 시험 이야기보다 "이 표현 자연스럽다", "이 흐름이 좋다" 식 코칭
```

### 톤 규칙
| 해야 할 것 | 하지 말 것 |
|----------|-----------|
| ~예요/~거든요 어미 (격식 낮춤) | ~합니다/~입니다 (너무 격식) |
| "오!", "굿!", "좋아요!" 감탄사 | 무미건조한 분석 톤 |
| "~해보면 어떨까요" 제안형 | "~하세요/마세요" 명령형 |
| "다음에 한 번 더!" 동기부여 | "틀렸어요/부족해요" 부정 |
| 구체적 표현 인용 ("I usually...") | 추상적 평가 ("문법이 약해요") |

### 절대 금지
- 숫자 점수 (88점, 75점 등) 언급
- 등급 평가 ("IH 수준이에요" 같은 직접적 등급 표시)
- 체크박스/항목별 통과/실패 표현
- 다른 멤버와 직접 비교 ("Bob보다 잘했네요")
- 시험관 톤 ("규칙에 따르면...", "기준 미달")

| 비어있음 (무응답) | "이번엔 답변 못 했네요. 다음 질문에서 한 문장이라도 시작해보세요. 시작이 어려운 건 자연스러워요." |
| 5초 이내, 한 문장 이하 | "조금 더 길게 풀어볼 여지가 있어요. 'because~' 한 마디 붙이면 답변이 살아나요." 위주 |
| 정상 분량 | 위 출력 예시처럼 강점+개선+팁 |

→ 프롬프트 안에 분기 조건 명시. GPT가 transcript 길이 보고 알아서 톤 조정.

## 데이터 쿼리 + 캐싱 전략

### 쿼리 카탈로그 (화면별 필요 데이터)

| 화면 | 필요 데이터 | 변경 빈도 | 캐싱 |
|------|-----------|----------|------|
| 진입 페이지 | 사용자의 활성 그룹 | 한 달 1회 | `staleTime: 5분` |
| 진입 페이지 | 그룹의 세션 이력 | 세션 종료마다 | `staleTime: 1분` |
| Step 2 카테고리 | 카테고리별 콤보/토픽 수 | 후기 승인 시 (~주 1회) | `staleTime: 30분` |
| Step 3 토픽 | 카테고리별 토픽 빈도 + 그룹 학습 이력 | 후기 + 세션 종료 | `staleTime: 5분` |
| Step 4 콤보 | 토픽별 콤보 빈도 + 질문 등장률 + 학습 이력 | 후기 + 세션 종료 | `staleTime: 5분` |

> **데이터 양 기준**: 시험후기 37건 / 콤보 192개 / 질문 471개. 인덱스 정상 작동 시 모든 쿼리 50ms 이내. **Materialized View 불필요**, 실시간 집계 + 클라이언트 캐싱이면 충분.

### 쿼리 설계 원칙

**원칙 1: SQL은 raw 데이터까지만, 가공은 SA에서**
- 콤보 시그니처 매칭, 빈도 % 계산 같은 로직은 Server Action에서 처리
- SQL 단순 유지 → 디버깅/최적화 ↑

**원칙 2: 시험후기 모듈 SSOT 직접 참조**
- 별도 캐시 테이블 X
- `submissions` + `submission_combos` + `submission_questions` 직접 집계
- 시험후기 누적 → 다음 쿼리부터 자동 반영

**원칙 3: 학습 이력은 `opic_study_*`에서 집계**
- 별도 테이블 X
- `opic_study_sessions.selected_combo_sig` (그룹 학습 이력)
- `opic_study_answers.question_id` (사용자 학습 이력)

### 핵심 SQL — 토픽 목록 (Step 3 데이터)

```sql
SELECT
  c.topic,
  COUNT(*) AS combo_count,
  COUNT(DISTINCT q_first.question_type_eng) AS type_diversity
FROM submission_combos c
JOIN submissions s ON s.id = c.submission_id
LEFT JOIN LATERAL (
  SELECT question_type_eng FROM questions q
  JOIN submission_questions sq ON sq.question_id = q.id
  WHERE sq.submission_id = c.submission_id AND sq.combo_type = c.combo_type
  LIMIT 1
) q_first ON true
WHERE s.status = 'complete' AND s.exam_approved = 'approved'
  AND CASE WHEN c.combo_type LIKE 'general%' THEN 'general'
           ELSE c.combo_type END = $1
GROUP BY c.topic
ORDER BY combo_count DESC;
```

### 콤보 시그니처 알고리즘 (TS)

```typescript
function buildComboSignature(questionIds: string[]): string {
  // 정렬된 시그니처 (순서 무관 그루핑용)
  return [...questionIds].sort().join('|');
}

function buildOrderedSignature(questionIds: string[]): string {
  // 출제 순서 그대로 (DB 저장용)
  return questionIds.join('|');
}

// 콤보 그루핑
function groupCombosBySignature(rawCombos: RawCombo[]): GroupedCombo[] {
  const map = new Map<string, RawCombo[]>();
  for (const c of rawCombos) {
    const sig = buildComboSignature(c.question_ids);
    const list = map.get(sig) ?? [];
    list.push(c);
    map.set(sig, list);
  }

  return Array.from(map.entries()).map(([sig, combos]) => ({
    sig,
    representative: combos[0],          // 첫 번째 출제 순서를 대표로
    frequency: combos.length,
    examples: combos,                   // 시험일/등급 등 메타
  })).sort((a, b) => b.frequency - a.frequency);
}
```

> **핵심 결정**: 정렬된 시그니처로 그루핑(순서 무관) + 대표 콤보의 출제 순서 보존. 음악 일반 데이터에서 본 것처럼 같은 질문 조합인데 순서만 다른 케이스를 같은 콤보로 통합.

```typescript
function buildUserPrompt(template: string, vars: Record<string, any>): string {
  return template.replace(/\{(\w+)\}/g, (_, k) => {
    const v = vars[k];
    if (Array.isArray(v)) return v.map((x, i) => `${i+1}. ${JSON.stringify(x)}`).join('\n');
    return String(v ?? '');
  });
}
```
