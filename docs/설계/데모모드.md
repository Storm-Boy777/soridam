# 체험판 모드 (가이드 투어) 설계

> **목적**: 로그인한 무료 사용자가 크레딧 없이 핵심 기능의 플로우를 체험하고, 유료 전환(결제) 또는 후기 제출(크레딧 획득)로 이어지도록 유도
> **핵심 원칙**: 기존 UI 100% 재사용, 별도 데모 페이지 없음, 가이드 레이어만 오버레이, 크레딧 소모 없음

---

## 0. 무료 플랜 정책

| 항목 | 무료 플랜 | 비고 |
|------|----------|------|
| 가입 시 크레딧 | **0개** (기본 지급 없음) | — |
| 스크립트 크레딧 | 후기 제출 시 **1개** 지급 (최대 2개) | 25일 룰 유지 |
| 모의고사 | **체험판만** 제공 (실전 0회) | 유료 전환 필요 |
| 쉐도잉 훈련 | 보유 스크립트 기반 **훈련 무제한** | 듣기/따라읽기/혼자말하기 무료 |
| 쉐도잉 평가 | **스크립트 크레딧 1개 소모** | Step 4 실전 녹음 AI 평가 |
| 빈도 분석 | **어드밴스만** 공개 (일반/롤플레이는 유료) | 비로그인도 어드밴스만 조회 가능 |

### 체험판 vs 실전 구분

| | 체험판 | 실전 |
|---|--------|------|
| 크레딧 소모 | **없음** | 1회 차감 |
| 데이터 | 하드코딩 샘플 | 실제 API 호출 |
| 결과 저장 | **안 됨** | DB 저장 |
| 반복 가능 | **무제한** | 크레딧 한도 |
| 가이드 투어 | 첫 체험 시 표시 | 없음 |

## 1. 아키텍처 개요

### 1.1 진입 방식

**로그인 필수** — 비로그인 사용자는 가입/로그인 후 접근.

#### 진입 경로 A: 크레딧 부족 시 자동 안내

**스크립트/쉐도잉**: TrialCreditGate 모달로 안내
```
스크립트 "생성" 클릭 → 크레딧 0개 감지
  ┌────────────────────────────────┐
  │ 스크립트 크레딧이 없습니다.      │
  │                                │
  │ [체험판으로 미리 보기]  ← 1순위  │
  │ [후기 제출하고 크레딧 받기]       │
  │ [요금제 보기]                    │
  └────────────────────────────────┘
```

**모의고사**: 모달 없음 — 모드 선택 UI에 항상 3가지 표시, 크레딧에 따라 활성/잠금

```
[크레딧 0개]                              [크레딧 1개+]
┌──────────────────────────┐             ┌──────────────────────────┐
│                          │             │                          │
│  ✅ 체험판 모드            │             │  훈련 모드                │
│     (15문항 체험)          │             │  실전 모드                │
│                          │             │                          │
│  🔒 훈련 모드  ← 이용권 필요│             └──────────────────────────┘
│  🔒 실전 모드  ← 이용권 필요│
│                          │
│        [요금제 보기]       │
└──────────────────────────┘

- 크레딧 0개: 체험판 활성 + 훈련/실전 잠금(비활성) → "결제하면 이걸 쓸 수 있구나" 인지
- 크레딧 1개+: 훈련/실전만 표시 (체험판 미노출)
- 잠금 모드 클릭 시 → "요금제 보기" 안내
```

#### 진입 경로 B: 모듈 탭 내 상시 버튼

```
스크립트 탭 상단
  ┌──────────────────────────────────────┐
  │ 💡 체험판으로 스크립트 생성 과정을     │
  │    미리 체험해보세요  [체험판 시작]     │
  └──────────────────────────────────────┘
```

#### 진입 경로 C: 쿼리 파라미터

```
/scripts/create?mode=trial     (스크립트 체험판)
/mock-exam/session?mode=trial  (모의고사 체험판)
/scripts?tab=shadowing&mode=trial  (쉐도잉 체험판)
```

- `?mode=trial` 쿼리 파라미터로 체험판 모드 진입
- 로그인 상태에서만 유효 (미로그인 시 로그인 페이지로 리다이렉트)

### 1.2 체험판 반복 정책

| 상황 | 가이드 투어 | 샘플 데이터 |
|------|-----------|-----------|
| **첫 체험** | 전체 가이드 (단계별 말풍선) | O |
| **재체험** | 가이드 없이 바로 진행 | O |
| **가이드 다시 보기** | 상단 배너 "가이드 다시 보기" 토글 | O |

- 첫 체험 여부: `localStorage`에 `trial_completed_{module}` 플래그 저장
- 체험판 자체는 **무제한 반복 가능** (크레딧 소모 없음)

### 1.3 분기 아키텍처

```
┌──────────────────────────────────────────────────┐
│  TrialProvider (Context)                         │
│  ┌─────────────────────────────────────────────┐ │
│  │ isTrialMode: boolean                        │ │
│  │ guideStep: number                           │ │
│  │ guideConfig: GuideStepConfig[]              │ │
│  │ isFirstTrial: boolean (localStorage 기반)    │ │
│  │ advanceGuide() / resetGuide()               │ │
│  │ showGuide: boolean (재체험 시 토글 가능)      │ │
│  └─────────────────────────────────────────────┘ │
│                                                  │
│  ┌──────────────┐  ┌─────────────────────────┐  │
│  │ 기존 컴포넌트 │  │ TrialOverlay (가이드)    │  │
│  │ (100% 동일)   │  │ - 하이라이트 마스크       │  │
│  │               │  │ - 말풍선 (툴팁)           │  │
│  │ API 호출 지점  │  │ - 타이핑 애니메이션       │  │
│  │ → useTrialData│  │ - 전환 CTA 버튼          │  │
│  │   분기 훅     │  │ - 상단 체험판 배너        │  │
│  └──────────────┘  └─────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

### 1.4 핵심 설계 원칙

1. **로그인 필수** — 비로그인 사용자는 체험판 접근 불가 (가입 유도)
2. **크레딧 소모 없음** — 체험판은 샘플 데이터, DB 쓰기 없음
3. **기존 코드 수정 최소화** — 분기 훅(`useTrialData`)으로 API 호출만 가로챔
4. **가이드 레이어 분리** — `TrialOverlay` 컴포넌트가 기존 UI 위에 오버레이
5. **사용자 주도 진행** — 자동 진행 없음, 실제 버튼 클릭으로 단계 이동
6. **입력은 자동화** — textarea/input에만 타이핑 애니메이션 적용
7. **무제한 반복** — 체험판은 횟수 제한 없음, 가이드는 첫 체험 시만 자동 표시

### 2.2 TrialProvider (trial-provider.tsx)

```tsx
interface TrialContextValue {
  isTrialMode: boolean;
  currentGuideStep: number;          // 현재 가이드 단계
  totalGuideSteps: number;           // 전체 가이드 단계 수
  guideConfig: GuideStepConfig[];    // 각 단계별 설정
  isFirstTrial: boolean;             // 첫 체험 여부 (localStorage)
  showGuide: boolean;                // 가이드 표시 여부 (재체험 시 토글)
  advanceGuide: () => void;          // 다음 가이드 단계로
  goToGuideStep: (step: number) => void;
  resetGuide: () => void;
  toggleGuide: () => void;           // 가이드 보기/숨기기 토글
}

interface GuideStepConfig {
  targetSelector: string;            // 하이라이트 대상 CSS 셀렉터
  title: string;                     // 말풍선 제목
  description: string;               // 말풍선 설명
  position: "top" | "bottom" | "left" | "right"; // 말풍선 위치
  action?: "click" | "type" | "wait"; // 사용자 필요 액션
  typeText?: string;                 // action="type" 시 자동 입력 텍스트
  highlightPadding?: number;         // 하이라이트 패딩
  showCTA?: boolean;                 // "직접 해보기" CTA 표시
  onEnter?: () => void;              // 단계 진입 시 콜백
}
```

### 2.3 TrialOverlay (trial-overlay.tsx)

전체 화면을 반투명 마스크로 덮고, 현재 가이드 단계의 대상 요소만 잘라내어 하이라이트.

```
┌────────────────────────────────────────┐
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
│  ░░░░░┌──────────────────┐░░░░░░░░░░░░│
│  ░░░░░│  하이라이트 영역   │░░░░░░░░░░░░│
│  ░░░░░│  (기존 UI 요소)    │░░░░░░░░░░░░│
│  ░░░░░└──────────────────┘░░░░░░░░░░░░│
│  ░░░░░  ┌─────────────────┐ ░░░░░░░░░░│
│  ░░░░░  │ 📌 말풍선 안내     │ ░░░░░░░░░░│
│  ░░░░░  │ 이 버튼을 눌러     │ ░░░░░░░░░░│
│  ░░░░░  │ 주제를 선택하세요  │ ░░░░░░░░░░│
│  ░░░░░  │                   │ ░░░░░░░░░░│
│  ░░░░░  │ [직접 해보기 →]    │ ░░░░░░░░░░│
│  ░░░░░  └─────────────────┘ ░░░░░░░░░░│
│  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
└────────────────────────────────────────┘
```

**구현 방식**:
- `position: fixed` + `z-index: 9999`로 전체 화면 오버레이
- CSS `clip-path` 또는 SVG `<mask>`로 하이라이트 영역 잘라내기
- `ResizeObserver` + `IntersectionObserver`로 대상 요소 위치 추적
- 하이라이트 영역은 `pointer-events: auto`, 나머지는 마스크 클릭 시 "다음 단계 안내"

### 2.5 TrialTyping (trial-typing.tsx)

```tsx
function useTrialTyping(options: {
```

- 실제 `textarea`의 value를 한 글자씩 업데이트

- 타이핑 중 사용자 입력 차단 (`pointer-events: none` on textarea)

## 3. 샘플 데이터 구조

> **중요**: 아래 데이터 구조는 설계 의도를 보여주는 **개략적 예시**입니다.
> 구현 시 반드시 실제 타입 정의를 참조하여 작성해야 합니다.
> - 스크립트: `lib/types/scripts.ts` → `ScriptDetail`, `ScriptOutput`, `ScriptParagraph`, `KeyExpression` 등
> - 모의고사: `lib/types/mock-exam.ts` → `MockTestSession`, `MockExamMode`
> - 쉐도잉: `lib/stores/shadowing.ts` → `TimestampItem`, `ShadowingEvaluation`

### 3.1 스크립트 체험판 데이터 (script-trial-data.ts)

```tsx
// 주제: 경험 (past_special 등) — 구현 시 실제 생성 결과로 교체
export const TRIAL_TOPIC = { id: "...", name: "경험", category: "..." };
export const TRIAL_QUESTION = { /* 실제 questions 테이블 행 구조 */ };
export const TRIAL_USER_STORY = "퇴근 후 집에 오면 항상...";

// ⚠️ ScriptDetail 타입에 맞춰야 함

// key_expressions → KeyExpression[] = { en, ko, tip }
// reusable_patterns → ReusablePattern[] = { template, description_ko, example }
// discourse_markers → DiscourseMarker[] = { en, ko, function, usage }
// key_sentences → KeySentence[] = { english, reason }
// structure_summary → StructureSummaryItem[] = { tag, description }
export const TRIAL_SCRIPT_RESULT: ScriptDetail = {
  /* 구현 시 실제 생성 결과를 그대로 복사 */
};
```

### 3.2 모의고사 체험판 데이터 (mock-exam-trial-data.ts)

**고정 기출 1세트 하드코딩** — 서버 조회 없음.

```tsx
// ⚠️ mode는 "training" | "test"만 허용 (MockExamMode 타입)
// 체험판 여부는 별도 is_trial 플래그로 관리
export const TRIAL_MOCK_SESSION = {
  session_id: "trial_mock",
  mode: "training" as const,  // "trial"은 타입에 없음 → "training" 사용
  status: "active" as const,
  current_question: 1,
  total_questions: 15,
  is_trial: true,  // 체험판 식별용 (MockTestSession 타입 외 별도 관리)
  // ⚠️ 실제 MockTestSession에는 user_id, submission_id, holistic_status,
  //    expires_at 등 추가 필드 필요 — 체험판에서는 더미값 사용
```

## 4. 모듈별 데모 플로우 상세

### 4.1 스크립트 + 쉐도잉 통합 체험판

> **핵심**: 스크립트 생성 → 패키지 생성 → 쉐도잉 훈련이 **하나의 연결된 체험 플로우**.
> 스크립트/패키지는 "구경" (자동 가이드), 쉐도잉에서 "직접 체험" (전환 동기 극대화).

#### 사전 준비 (개발 시 1회)

개발 시점에 **실제로 스크립트 생성 → 패키지 생성**을 실행하고, 그 결과물을 하드코딩 데이터로 저장.

```
1. 실제 스크립트 생성 API 호출
   → 주제: 경험 / 질문: 루틴
   → 스토리: "퇴근하면 항상 이어폰을 꽂고 음악을 들어요..."
   → 결과: 영문 스크립트 + 한글 번역 + 핵심 표현 4종

2. 실제 패키지 생성 API 호출
   → TTS WAV 파일 + 문장별 타임스탬프 JSON

3. 결과물을 정적 파일로 저장
   → components/trial/trial-data/script-trial-data.ts (스크립트 JSON)
   → public/trial/script-package.wav (TTS 음성)
   → public/trial/script-package-timestamps.json (타임스탬프)
```

**데이터는 실제로 생성된 진짜 결과물** — 체험판에서는 API를 호출하지 않고 이 데이터를 보여줌.

#### Phase A: 스크립트 생성 가이드 (자동 진행, 사용자는 "다음"만)

**진입**: `/scripts/create?mode=trial`

### 4.2 모의고사 체험판 (15문항 전체 진행)

**진입**: `/mock-exam/session?mode=trial`

#### 실전과의 차이

| 항목 | 실전 | 체험판 |
|------|------|--------|
| 문항 수 | 15문항 | **15문항 (동일)** |
| 질문 소스 | 기출 풀 (사용자 선택) | **고정 기출 1세트** (하드코딩, 서버 조회 없음) |

```
  │    → question_ids: 고정 15개

  │
  │ 4. 15문항 순차 진행
  │    → 질문 오디오 재생 (실제 TTS — audio_url 하드코딩)
  │    → 녹음 (실제 MediaRecorder 사용, Storage 미업로드)
```

### 6.1 진입 경로 (로그인 후)

체험판은 **로그인 필수**. 비로그인 사용자는 가입/로그인 후 접근.

```
로그인 상태에서:

경로 A: 크레딧 상태에 따른 자동 안내
├── 스크립트 "생성" 클릭 → 크레딧 0개 → TrialCreditGate 모달
├── 모의고사 → 모드 선택 UI에 항상 3가지 표시 (체험판+훈련+실전), 크레딧에 따라 활성/잠금
└── 쉐도잉 "평가" 클릭 → 크레딧 0개 → TrialCreditGate 모달

경로 B: 모듈 탭 내 상시 배너
└── "체험판으로 미리 체험해보세요 [체험판 시작]"

경로 C: URL 직접 접근
├── /scripts/create?mode=trial
├── /mock-exam/session?mode=trial
└── /scripts/shadowing?mode=trial
```

### 6.2 체험판 → 전환 유도

```
체험판 진행 중/완료 시:
├── 스크립트 완료 → "후기 제출하고 크레딧 받기" / "요금제 보기"
```

### 6.3 middleware

**수정 불필요** — 체험판은 로그인 필수이므로 기존 인증 로직 그대로 유지.

## 8. 기술 상세

### 8.1 하이라이트 오버레이 구현

```tsx
// TrialOverlay 핵심 로직
function TrialOverlay({ steps }: { steps: GuideStepConfig[] }) {
  const { currentGuideStep, isGuideVisible } = useTrialContext();
  const [targetRect, setTargetRect] = useState<DOMRect | null>(null);
  const step = steps[currentGuideStep];

  // 대상 요소 위치 추적
  useEffect(() => {
    const el = document.querySelector(step.targetSelector);
    if (!el) return;

    const updateRect = () => setTargetRect(el.getBoundingClientRect());
    updateRect();

    const observer = new ResizeObserver(updateRect);
    observer.observe(el);
    window.addEventListener("scroll", updateRect, { passive: true });

    return () => {
      observer.disconnect();
      window.removeEventListener("scroll", updateRect);
    };
  }, [step.targetSelector]);

  if (!isGuideVisible || !targetRect) return null;

  const pad = step.highlightPadding ?? 8;

  return (
    <div className="fixed inset-0 z-[9999]">
      {/* SVG 마스크: 하이라이트 영역만 투명 */}
      <svg className="absolute inset-0 h-full w-full">
        <defs>
          <mask id="trial-mask">
            <rect width="100%" height="100%" fill="white" />
            <rect
              x={targetRect.left - pad}
              y={targetRect.top - pad}
              width={targetRect.width + pad * 2}
              height={targetRect.height + pad * 2}
              rx="12"
              fill="black"
            />
          </mask>
        </defs>
        <rect
          width="100%" height="100%"
          fill="rgba(0,0,0,0.5)"
          mask="url(#trial-mask)"
        />
      </svg>

      {/* 말풍선 */}
      <TrialTooltip
        targetRect={targetRect}
        position={step.position}
        title={step.title}
        description={step.description}
        showCTA={step.showCTA}
        stepNumber={currentGuideStep + 1}
        totalSteps={steps.length}
      />
    </div>
  );
}
```

### 8.2 타이핑 애니메이션 구현

```tsx
function useTrialTyping({
  text,
  targetRef,
  speed = 50,
  delay = 500,
  onComplete,
}: TrialTypingOptions) {
  const [isTyping, setIsTyping] = useState(false);

  const start = useCallback(() => {
    if (!targetRef.current) return;
    setIsTyping(true);

    let i = 0;
    const timer = setTimeout(() => {
      const interval = setInterval(() => {
        if (i >= text.length) {
          clearInterval(interval);
          setIsTyping(false);
          onComplete?.();
          return;
        }
        i++;
        const el = targetRef.current;
        if (!el) return;

        // React state와 동기화를 위해 nativeInputValueSetter 사용
        const nativeSetter = Object.getOwnPropertyDescriptor(
          HTMLTextAreaElement.prototype, "value"
        )?.set;
        nativeSetter?.call(el, text.slice(0, i));
        el.dispatchEvent(new Event("input", { bubbles: true }));
      }, speed);
    }, delay);

    return () => clearTimeout(timer);
  }, [text, targetRef, speed, delay, onComplete]);

  return { isTyping, start };
}
```

### 8.3 모바일 대응

- 말풍선 위치: 화면 하단 영역에 대상이 있으면 `top` 방향, 상단이면 `bottom` 방향 자동 전환
- 하이라이트 패딩: 모바일에서는 터치 영역 확보를 위해 `pad * 1.5`
- 데모 배너: 모바일에서 한 줄로 축소 ("체험 모드" + 가입 버튼만)
- `TrialOverlay` 마스크 클릭 시: 모바일에서는 스와이프 힌트 표시

## 9. 주의사항

### 9.1 보안
- 체험판 모드에서는 실제 DB 쓰기 작업 절대 불가
- `mode=trial`이 있더라도 API 엔드포인트에 직접 요청 시 크레딧 확인 (기존 RLS + 크레딧 검증 유지)
- 체험판 데이터는 클라이언트 하드코딩 — 서버 자원 소비 없음
- 로그인 필수 — 비로그인 시 체험판 접근 불가

### 9.2 UX
- 첫 체험 진입 시 "샘플 데이터로 기능을 미리 체험해보세요!" 안내 (1회)
- 가이드를 닫으면 자유 탐색 가능 (API 호출은 여전히 체험판 데이터 반환)
- 뒤로 가기 → 해당 모듈 탭으로 이동 (체험판 상태 초기화)
- 체험 완료 시 전환 CTA: "후기 제출하고 크레딧 받기" / "요금제 보기"
- 재체험 시 가이드 없이 바로 시작 (배너에 "가이드 다시 보기" 토글)

### 9.3 성능
- `TrialOverlay`는 가이드가 활성일 때만 마운트 (`isGuideVisible`)
- `ResizeObserver` 정리 (`useEffect` cleanup)
- 데모 데이터는 정적 import — 번들에 포함되지만 용량 미미 (< 5KB)
- 쉐도잉 샘플 WAV는 `public/trial/`에 배치 (CDN 캐싱)
