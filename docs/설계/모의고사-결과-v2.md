## 3. ACTFL 공식 기준 참조

> **출처**: 공식 세부진단서 7건 (IM2×2, IM3×2, IH×2, AL×1) + ACTFL Proficiency Guidelines 2024 + OPIc Familiarization Guide

### FACT 기준 (ACTFL 2024 공식)
ACTFL이 정의한 말하기 능력의 4가지 평가 축:
- **F** (Functions & tasks): 수행할 수 있는 과제 유형
- **A** (Accuracy): 언어적 정확성 (문법, 어휘, 발음, 사회문화적 지식)
- **C** (Context & content): 대화 가능한 상황과 주제 범위
- **T** (Text type): 발화의 길이와 복잡도 (단어 → 문장 → 단락 → 확장 담화)

### 공식 진단서 구조 (6페이지)
| 페이지 | 내용 | 개인화 |
|--------|------|--------|
| 1. Proficiency Report | 등급 인증서 + ACTFL Guidelines 전문 + Subscore | ❌ 고정 |
| 2. Score Report | Functional Highlights + SPEAKERS 테이블 + TIPS | ❌ 고정 |
| 3. ACTFL Proficiency Guidelines | 등급 상세 설명 (2~3문단) | ❌ 고정 |
| 4. Diagnostic Form: Advanced | 5행 × (Performance + 체크리스트) | ✅ 개인화 |
| 5. Diagnostic Form: Intermediate | 4행 × (Performance + 체크리스트) | ✅ 개인화 |
| 6. Overall Comments | 평가자 종합 서술 1문단 | ✅ 개인화 |

### Advanced Functions 5축 (세부진단표 핵심)
| # | 영문 | 한글 | 평가 대상 |
|---|------|------|----------|
| 1 | Can narrate and describe in all time frames | 시제 통제 / 서술 정밀도 | 시제별 서술·묘사 + 논리적 순서 + 동사 형태 + 세부 묘사 |
| 2 | Can consistently produce connected discourse of paragraph length | 담화 구조 | 발화 단위(단어→문장→단락) + 어순 + 응집 장치 |
| 3 | Can speak about a variety of topics, including current events | 주제 다양성 / 어휘 폭 | 어휘 범위 + 타 언어 개입 + 오인지 어휘 |
| 4 | Can deal effectively with an unanticipated complication | 돌발 대처 | 해결 성공 여부 + 의사소통 장치 |
| 5 | Can be readily understood by speakers not accustomed to non-native speakers | 이해도 / 유창성 / 발음 | 유창성(속도·끊김·연결) + 문법 + 화용 능력 + 발음(조음·강세·억양) |

### Performance 판정 등급 (공식 진단서 기준)
| 판정 | 의미 | 주로 나타나는 등급 |
|------|------|-----------------|
| **Meets Criteria Fully** | 완전 충족 | 해당 레벨의 기능을 안정적으로 수행 |
| **Meets Criteria Minimally** | 최소 충족 | 기능 수행은 하나 일관성 부족 |
| **Developing Ability** | 발전 중 | IH에서 Advanced 기능에 대해 주로 |
| **Emerging Performance** | 출현 수준 | IM에서 Advanced 기능에 대해 주로 |

|------|----------|------|
| Communication Tasks | **F** | 수행 가능한 과제 유형 |
| Contexts / Content | **C** | 대화 가능한 상황과 주제 |
| Discourse Type | **T** | 발화 단위 (문장/단락) |

### 등급별 ACTFL Guidelines (고정 텍스트 소스)

**Intermediate Mid**:
> Speakers at the IM sublevel are able to handle successfully a variety of uncomplicated communication tasks in straightforward social and transactional situations. They can engage effectively in predictable and concrete exchanges that involve personal information related to self, family, home, daily activities, interests, and personal preferences. Their speech largely consists of strings of sentences; it may contain pauses, reformulations, and self-corrections.

**Intermediate High**:
> Speakers at the IH sublevel are able to converse with ease and confidence when dealing with routine tasks and social situations. They can handle a substantial number of tasks associated with the Advanced level, but they are unable to sustain performance of all of these tasks all of the time. They often can narrate and describe in all major time frames using connected discourse of paragraph length, but their speech exhibits one or more features of breakdown.

**Advanced Low**:
> Speakers at the AL sublevel are able to handle a variety of communication tasks. They can participate in most informal and some formal conversations on topics related to school, home, and leisure activities. They demonstrate the ability to narrate and describe in the major time frames with some control of aspect. They combine and link sentences into connected discourse of paragraph length.

### TIPS FOR IMPROVING PROFICIENCY (등급별 공식, Score Report)

**IM 등급 공식 TIPS**:
1. Practice giving detailed descriptions (묘사·서술 연습)
2. Speak in all time frames (모든 시제로 말하기)
3. Connect speech into cohesive discourse (응집력 있는 담화 연결)
4. Broaden the topics (주제 확장)
5. Increase fluency and improve delivery (유창성·전달력 향상)

**IH 등급 공식 TIPS**:
1. Focus on speaking with more accuracy in all major time frames (시제 정확성)
2. Practice telling stories (스토리텔링 연습)
3. Deal with the unexpected (돌발 상황 대처 연습)
4. Speak in cohesive paragraphs (응집적 단락 발화)
5. Increase fluency and improve delivery (유창성·전달력 향상)

### 탭 2: 세부진단표 ✅ 확정
> 레퍼런스 Page 3~4에 해당
> 공식 DIAGNOSTIC FORM 체크박스 = 평가엔진 `checkbox-definitions.ts`와 1:1 대응 확인 완료

**목표**: "되는 것/안 되는 것" 한눈에 — 공식 OPIc 세부진단서를 한글화

**핵심 결정**:
- 평가엔진이 이미 체크하고 있는 체크박스 결과(INT 20 + ADV 42 + AL 12)를 한글 라벨 + 체크박스 UI로 표시
- 공식 DIAGNOSTIC FORM 양식을 충실히 재현 (3컬럼: 기능 | 판정 | 세부 체크)
- **AL 체크박스(12개)는 IH/AL 등급에서만 표시** — "이걸 충족 못해서 IH", "이걸 충족해서 AL" 목적

**등급별 표시 범위**:
| 등급 | INT (20개) | ADV (42개) | AL (12개) |
|------|-----------|-----------|----------|
| NL~IM3 | ✅ 표시 | ✅ 표시 | ❌ 숨김 |
| IH | ✅ 표시 | ✅ 표시 | ✅ 표시 ("이것을 충족 못해서 IH") |
| AL | ✅ 표시 | ✅ 표시 | ✅ 표시 ("이것까지 충족해서 AL") |

**체크박스 의미 방향** (확정):
| 항목 | 실제 OPIc 진단서 | 우리 v2 UI |
|------|----------------|-----------|
| 체크 의미 | ■ = **미충족 (개선 필요)** | ✅ = **충족** |
| 빈칸 의미 | ☐ = 문제 없음 | ☐ = 미충족 |
| 관점 | 평가관 → "이것을 개선하라" | 학습자 → "이것은 됐다/안 됐다" |
| 카운트 | 없음 | "6/11 충족" 식으로 표시 |

- **결정 근거**: 실제 진단서는 평가관이 개선점을 표시하는 도구이고, 우리 서비스는 학습자가 자기 수준을 파악하는 도구 → 관점이 다르므로 표현도 달라야 함
- ✅ = 잘했다, ☐ = 못했다는 직관적이고, 카운트·뱃지 색상과도 방향이 일관됨
- 실제 진단서의 "체크=못한 것" 방식은 한 번 더 사고 전환이 필요해 사용자에게 혼란

**실제 OPIc 진단서와의 차이점**:
| 항목 | 실제 진단서 | 우리 v2 UI | 이유 |
|------|-----------|-----------|------|
| 최상위 제목 | `DIAGNOSTIC FORM` 하나 | 섹션별 별도 카드 (INT/ADV/AL) | 가독성, 스크롤 구간 구분 |
| 섹션 구분 | 테이블 헤더로만 구분 | 별도 카드 + 섹션 제목 | 모바일 대응, 시각적 명확성 |
| 3번째 컬럼 | "기준 미충족 시 개선 필요 항목" | "세부 항목" + 충족 카운트 | 충족 방향으로 전환했으므로 |
| 체크 방향 | 미충족=체크 | 충족=체크 | 학습자 관점 직관성 |
| Performance 표기 | 영문 (Meets CriteriaFully 등) | 한글 (완전 충족 등) | 한글 서비스 |

### 탭 3: 문항별 평가 ✅ 확정
> 공식 OPIc 진단서에는 없는 콘텐츠 — 우리 서비스의 핵심 차별점
> v2 전용 프롬프트로 생성 (v1 코칭 구조와 완전 분리)

**목표**: 목표 등급 기준으로 Q2~Q15 개별 문항 평가 — "전문가 평가관의 관찰 + 방향 제시"

**핵심 결정**:
- **평가 기준 = 목표 등급** (결과 등급이 아님) — 응시자가 서비스를 이용하는 이유가 목표 등급 취득이므로
- **평가 톤 = 평가관 관찰** — "~이 관찰되었다", "~에 도달하지 못하였다" (v1의 "~하세요" 코칭 톤 아님)
- **방향 제시까지만** — "~할 필요가 있다" 수준. 구체적 교정본/연습법은 튜터링 영역
- **튜터링 직접 연계** — 문항별 평가 결과가 튜터링 처방엔진의 입력 데이터가 됨
- **Q1 제외** — 자기소개 고정 문항이므로 평가 대상 아님 (Q2~Q15, 14개 문항)

**목표 등급 설정 방식** (추후 결정):
- 옵션 A: 사용자 프로필에서 목표 등급 설정
- 옵션 B: 모의고사 시작 전 선택
- 옵션 C: 결과 등급 +1 자동 설정 (IM3 → IH)

**튜터링 연계 데이터 구조**:
```typescript
// 문항별 평가 결과 (v2) — 튜터링 처방엔진이 소비
interface QuestionEvalV2 {
  question_number: number;          // 2~15
  question_type: string;            // "description" | "routine" | "past_special" | ...
  target_grade: string;             // 목표 등급 ("IH" 등)
  fulfillment: "fulfilled" | "partial" | "unfulfilled" | "skipped";
  task_checklist: Array<{           // 과제 체크리스트 (목표 등급 기준)
    item: string;
    pass: boolean;
  }>;
  observation: string;              // 평가관 관찰 소견 (3~5문장, GPT 생성)
  directions: string[];             // 개선 방향 1~2개 (GPT 생성)
  weak_points: string[];            // 약점 태그 (튜터링 처방엔진 입력)
  recommended_drills: string[];     // 추천 훈련 태그 (튜터링 연계)
  speech_meta: {                    // 발화 메타
    duration_sec: number;
    wpm: number;
    pronunciation_score: number;
  };
}
```

> 2회차 이상부터 활성 | ✅ 목업 완료

**목표**: 이전 시험 대비 변화를 평가관 관찰 톤으로 분석. 사용자가 직관적으로 이해할 수 있는 구체적 관찰 문장 중심.

- **구체적 관찰 문장만** — GPT가 이전↔현재 세션의 평가 데이터를 비교하여 관찰 문장 생성
- **목표 등급 기준 도달도** — 문항 유형별로 목표 등급 체크리스트 통과 현황을 ●○ 도트로 표시
- **FACT 용어 비노출** — FACT 약어, 점수 직접 노출 금지

### 5.1 프롬프트 구조

**2-메시지 구성**: System Prompt (평가 기준 + 출력 형식) + User Prompt (문항별 데이터)

#### System Prompt (전문)

```
당신은 OPIc 시험의 공인 평가관입니다. 응시자의 답변을 목표 등급({TARGET_GRADE}) 기준으로 개별 평가합니다.

## 평가 원칙
1. **평가 기준**: 목표 등급({TARGET_GRADE})의 ACTFL 기준에 도달했는지를 판정합니다.
2. **평가 톤**: 평가관의 관찰 보고서 톤으로 작성합니다.
   - "~이 관찰되었다", "~에 도달하지 못하였다", "~이 확인되었다"
   - 코칭 톤("~하세요", "~해 보세요") 절대 사용 금지
3. **방향만 제시, 교정 금지**: 개선 방향만 제시하고, 구체적 예문이나 연습 방법은 튜터링 영역입니다.
4. **근거 기반**: 모든 판정은 답변 텍스트에서 구체적 근거를 인용합니다.

## 등급 기준 요약
- **IH (Intermediate High)**: 일상 과제를 편안하게 수행. Advanced 과제(주요 시제 서술·묘사, 단락 수준)를 상당 부분 처리하나 약점 존재. 문장 연결+때때로 단락.
- **IM3 (Intermediate Mid 3)**: Intermediate 과제에서 양과 질 모두 우수. 연결 문장 생산. 가끔 주저함/자기 교정.

## 질문 유형별 핵심 과제
- **묘사(description)**: 대상의 핵심 특징 2개+, 현재시제 안정, 개인화(감상/경험)
- **루틴(routine)**: 활동 나열+빈도 표현, 순서 연결어, 현재시제 일관
- **비교·변화(comparison/adv_14)**: 두 축 명시(과거vs현재), 비교 표현(compared to, unlike, while), 비교 관점 2개+, 사회적 맥락(adv_14)
- **경험-어린시절(past_childhood)**: 과거시제 일관, 시간순서, 감정·반응 구체적, 의미/교훈 마무리
- **경험-특별한(past_special)**: 과거시제 일관, 사건 전개(전-중-후), 구체적 세부사항, 감상 마무리
- **롤플레이-질문(rp_11)**: 인사·도입, WH-질문 3개+, 정중 표현(Could you~), 마무리
- **롤플레이-대안제시(rp_12)**: 상황 인식, 사과·공감, 대안 2개+, 협상·설득 표현
- **사회적이슈(adv_15)**: 사회적 주제 명시, 다관점(2+), 근거 기반 논증, 구조적 전개

## 출력 JSON 형식 (각 문항별)
{
  "question_number": 2,
  "fulfillment": "fulfilled" | "partial" | "unfulfilled",
  "task_checklist": [
    { "item": "과제 항목", "pass": true/false, "evidence": "근거 인용" }
  ],
  "observation": "3~5문장의 평가관 관찰 소견. 구체적 근거 포함.",
  "directions": ["개선 방향 1~2개. 방향만 제시, 코칭 금지."],
  "weak_points": ["약점 키워드"],
  "recommended_drills": ["drill_tag"]
}

## fulfillment 판정 기준
- **fulfilled**: 필수 과제 전부 충족 + 심화 과제 50%+ 충족
- **partial**: 필수 과제 일부 미충족 또는 심화 과제 50% 미만
- **unfulfilled**: 필수 과제 대부분 미충족

## drill_tag 목록
tense_control, narrative_structure, comparison_frame, connector_diversity, paragraph_organization, emotional_expression, paragraph_closure, filler_reduction, roleplay_questions, complication_handling, negotiation_expressions, social_perspective, detail_expansion, description_depth
```

### 5.2 프롬프트 설계 근거

| 설계 요소 | 결정 | 근거 |
|----------|------|------|
| **평가 기준** | 목표 등급(IH) | 결과 등급(IM3) 기준이면 대부분 "충족"으로 변별력 없음 |
| **평가 톤** | 관찰 보고서 | v1 코칭 톤과 차별화, 전문가 진단서 컨셉 |
| **체크리스트 evidence** | 답변 텍스트 인용 | 판정 근거를 투명하게 제시하여 신뢰도 확보 |
| **방향만 제시** | 교정본/연습법 제외 | 진단서 역할 한정, 처방은 튜터링 영역 |
| **유형별 과제 분리** | 10개 유형별 핵심 과제 명시 | question_type에 따라 평가 기준이 달라야 함 |
| **drill_tag** | 14개 태그 | 튜터링 처방엔진과 연계 (weak_points → 처방 자동화) |
| **fulfillment 3단계** | fulfilled/partial/unfulfilled | skipped는 3축 무응답 감지로 별도 판정 (프롬프트 외부) |

### 5.3 프롬프트 한계점 & 개선 방향

| 한계 | 상세 | 개선 방향 |
|------|------|----------|
| **등급 기준 간소화** | IH/IM3 요약만 포함, ACTFL 전체 기준 미반영 | 프로덕션에서는 `evaluation_prompts_v2` 테이블에 등급별 상세 기준 저장 |
| **유형별 과제 불균형** | 10개 유형 각 1~2줄로 기술, 세부 기준 부족 | `checkbox-definitions.ts`의 필수/심화 항목을 유형별로 프롬프트에 반영 |
| **체크리스트 개수 미지정** | GPT가 3~5개 자유 생성 | 유형별 필수 N개 + 심화 M개 고정 구조로 변경 |
| **무응답 처리 없음** | skipped 판정 로직 미포함 | 3축 무응답 감지(내용부족/전달붕괴/발화질붕괴)를 프롬프트 또는 전처리 단계에서 구현 |
| **등급별 프롬프트 분리** | IH 고정 | 목표 등급에 따라 기준 동적 변경 필요 |
| **temperature** | 0.3 고정 | 관찰 톤 일관성 vs 다양성 균형 조정 필요 |
| **evidence 품질** | GPT 재량으로 인용 | 인용 규칙(원문 그대로, 최소 구절 단위 등) 명시 필요 |
