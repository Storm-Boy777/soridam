### 5단계 가이드 (`activeGuideStep`)

현재 상태에 따라 자동 계산:

```typescript
// mock-exam-session.tsx:351-370
const activeGuideStep: GuideStep = (() => {
  if (questionPlayer.isPlaying) return "listen";
  if (questionPlayer.canReplay && !questionPlayer.hasReplayed) return "replay";
  if (recorder.state === "recording") return "record";
  if (uploadState === "uploading" || "retrying" || "submitted") return "next";
  if (hasPendingEval) return "eval";
  return "listen";
})();
```

| 스텝 | 조건 | 아이콘 (활성) | 라벨 |
|------|------|-------------|------|
| listen | `isPlaying = true` | Volume2 (흰색) | 질문 듣기 |
| replay | `canReplay && !hasReplayed` | Clock (흰색) | {n}초 (카운트다운) |
| record | `recorder.state = "recording"` | Mic (흰색) | 답변 녹음 |
| next | `uploadState ∈ {uploading, retrying, submitted}` | ArrowRight/Loader2 | 다음 문제 |
| eval | 평가 pending 있음 | Bot (흰색) | 답변 평가 |

`useQuestionPlayer.play(audioUrl)` 내부 (`use-question-player.ts:73-117`):

```
1. 기존 Audio 객체 정리
2. new Audio(audioUrl) 생성
3. audio.onplay → isPlaying = true
4. audio.ontimeupdate → playbackProgress = (currentTime / duration) * 100
5. audio.onended:
   - playbackProgress = 100
   - isPlaying = false
   - hasPlayed = true
   - 리플레이 윈도우 시작 (첫 재생 후에만)
   - onPlaybackEndedRef.current?.() 콜백 호출
```

#### Step 2: 리플레이 윈도우 (5초)

```typescript
// use-question-player.ts:56-70 — startReplayWindow
setCanReplay(true);
setReplayCountdown(replayWindowSeconds);  // 5

countdownTimerRef.current = setInterval(() => {
  setReplayCountdown(prev => {
    if (prev <= 1) {
      clearCountdown();
      setCanReplay(false);
      return 0;
    }
    return prev - 1;
  });
}, 1000);
```

- 5초 카운트다운 동안 "다시 듣기 (n초)" 버튼 활성화
- 카운트다운 = 0 → `canReplay = false` → 버튼 비활성화
- **1회만 가능**: `hasReplayed` 플래그로 제어

#### Step 3: 자동 녹음 시작 (`onPlaybackEnded`)

```typescript
// mock-exam-session.tsx:127-130
const autoStartRecording = useCallback(() => {
  if (recorder.state !== "idle" || uploadState !== "idle") return;
  recorder.startRecording();
}, [recorder, uploadState]);

// useQuestionPlayer에 콜백 연결
const questionPlayer = useQuestionPlayer({
  replayWindowSeconds: 5,
  onPlaybackEnded: autoStartRecording,  // ref 패턴으로 최신 참조

```

`onPlaybackEnded` 호출 시점:
- **첫 재생 완료** → `autoStartRecording()` → 녹음 자동 시작
- **리플레이 완료** → `autoStartRecording()` → 녹음 다시 자동 시작

리플레이 시 현재 녹음을 먼저 리셋:

## Phase 10: Edge Function 체인 (백그라운드)

답변이 `submitAnswer`로 제출될 때마다 fire-and-forget으로 시작:

### Stage A: `mock-test-process` (~70초/문항)

```
1. Storage에서 오디오 다운로드
2. Whisper STT (음성 → 텍스트)
3. 스킵 판정 (_shared/skip-detector.ts):
   - audio_duration ≤ 15초
   - transcript ≤ 15자
   - Whisper 환청 (반복 패턴)
4. Azure 발음 평가 (_shared/azure-pronunciation.ts):
   - WebSocket 기반 (60초 타임아웃)
   - 5영역: accuracy, fluency, completeness, prosody, pronunciation
   - 단어별 분석 (ErrorType: None/Omission/Insertion/Mispronunciation)
5. 메타데이터 계산:
   - WPM (Words Per Minute)
   - 필러 단어 15패턴 (um, uh, hmm, like, you know...)
   - Long Pause 감지 (WPM 기반 휴리스틱)
6. eval_status = "stt_completed" 업데이트
7. fire-and-forget → Stage B
```

재시도 3회 (exponential backoff 1→2→4초). 실패 시:
- `retry_count < 3` → `eval_status = "pending"` (재시도 가능)
- `retry_count ≥ 3` → `eval_status = "failed"`

Azure 실패 시에도 GPT 평가 계속 진행 (graceful degradation)

### Stage B-1: `mock-test-eval-judge` (~60초/문항)

```
1. question_type → prompt key 매핑:
   - description → eval_description
   - routine → eval_routine
   - comparison → eval_comparison
   - experience_* (3종) → eval_past_experience (공유)
   - comparison_change → eval_comparison_change
   - social_issue → eval_social_issue
   - rp_12 → eval_roleplay

2. question_type → checkbox_type 매핑:
   - INT: 묘사, 루틴, 질문하기
   - ADV: 비교, 경험, 대안제시
   - AL: 비교변화, 사회적이슈

3. evaluation_prompts DB 조회 → 변수 치환:
   - {transcript}, {wpm}, {accuracy_score}, {fluency_score}...
   - {checkbox_ids}, {checkbox_labels}

4. GPT-4.1-mini 호출 (eval_settings 기반):
   - 체크박스 pass/fail 판정
   - task_fulfillment 상태 판정
   - feedback_branch 결정

5. 기대 체크박스 ID 기준 fail-closed 검증
   - 누락 항목은 fail로 보정
   - 예상 밖 ID는 저장하지 않음

6. mock_test_evaluations INSERT
7. eval_status = "judge_completed"
8. fire-and-forget → Stage B-2
```

### Stage B-2: `mock-test-eval-coach` (~40초/문항)

```
1. B-1 결과(checkboxes_summary, task_fulfillment, feedback_branch) 로드
2. feedback_branch별 코칭 경로 분기
   - failed: rule-based 구제 메시지
   - partial/fulfilled: GPT-4.1 코칭 6-Layer
3. priority_prescription 생성
4. mock_test_evaluations UPDATE (coaching_feedback, priority_prescription)
5. eval_status = "completed"
6. 전체 평가 완료 감지 → fire-and-forget → Stage C
```

### Stage C: `mock-test-report` (~30초)

```
1. 전체 evaluations 로드
2. 평가엔진 7-Step (_shared/rule-engine.ts):
   Step 1: 체크박스 집계 + 누적 로직 (INT-2, ADV-2-SP)
   Step 2: Floor 판정 (INT pass rate → NH/IL/IM1/IM2/IM3)
   Step 3: Ceiling 판정 (ADV pass rate → BROKE_DOWN/RESPOND/SUSTAIN)
   Step 4: Sympathetic Listener (발음 + ADV-5 복합 점수)
   Step 5: Q12 게이트키퍼 (ADV-4 pass rate)
   Step 6: AL 판정 (AL 게이트키퍼 + 통과율)
   Step 7: 최종 등급 결정 (NH ~ AL)

3. FACT 점수 계산 (_shared/checkbox-definitions.ts):
   F(16개 체크박스) + A(31개) + C(5개) + T(INT2+SP+CD)
   각 영역 × 2.5 = 100점 만점

4. 2단계 저장:
   ① 평가엔진 + FACT → mock_test_reports UPSERT
   ② GPT 종합 리포트 (eval_comprehensive 프롬프트) → UPDATE

5. sessions.holistic_status = "completed", status = "completed"
```
