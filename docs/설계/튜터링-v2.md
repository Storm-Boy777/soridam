## 1. 핵심 철학: 일타 강사

### 일타 강사가 학원 강사와 다른 점

**진단의 정확성** + **처방의 구체성**

| | 학원 강사 (v1) | 일타 강사 (v2) |
|---|----------|----------|
| 진단 | "문법이 약하시네요" | "15문항 중 8개에서 연결어가 and, so 2종뿐이야" |
| 처방 | "연결어를 다양하게 쓰세요" | "however, moreover, on the other hand 3개 외워서, 30초 답변에 3개 넣어" |
| 완료 | "3번 연습했으니 다음으로" | "3회 중 2회에서 4종 이상 나왔으니 다음으로" |
| 톤 | "~를 연습하세요" (시스템) | "이것만 고치면 IH 간다" (인격) |
| 방법 | 약점별 개별 교정 | **틀 기반 학습** — 만능 틀 장착 → 어휘·연결어 업그레이드 |

### v1의 문제점

| 항목 | v1 현실 | v2 목표 |
|------|---------|---------|
| **진단 근거** | coaching_report.top3_priorities (GPT 주관 요약 3줄) | 15문항 × weak_points 빈도·심각도 통계 분석 |
| **처방 생성** | DRILL_TAG 18개 고정 매핑 → 최대 6개 처방 | 병목 스코어링 → 티어별 맞춤 드릴 매칭 |
| **등급 분화** | 3구간(IL_IM, IM_IH, IH_AL) 단순 분기 | **4티어 체계** — 등급 간 결정적 차이 기반 훈련 분화 |
| **프로토콜 선택** | 모든 약점 → 동일 5프로토콜 순차 | 약점 유형 + 티어별 맞춤 프로토콜 조합 |
| **완료 판정** | 3회 반복 = 기계적 완료 | 성공 기준 충족률 기반 (드릴별 측정 가능한 기준) |
| **교수법** | 약점 교정 중심 | **강지완 메소드** — 틀 장착 → 바꿔 끼기 → 채점표 역공학 |
| **증거 인용** | 없음 | "Q3에서 'and then' 3번 반복한 거 봤지?" 학생 발화 직접 인용 |

### 3모듈 역할 유지

| 모듈 | 역할 | 비유 |
|------|------|------|
| 모의고사 v2 개별평가 | 진찰 — 문항별 증거 수집 | 의사의 검사 |
| 모의고사 v2 종합평가 | 진단서 — 등급 판정 + 근거 | 의사의 소견서 |
| **튜터링 v2** | **치료** — 병목 집어서 고침 | **일타 강사의 과외** |

### "다음 한 칸" 원칙

> **지금 등급에서 바로 다음 등급으로 가는 데 필요한 것만 처방한다.**
> AL 목표라도 현재 IM2면 IM3에 필요한 것만 먼저. IH→AL은 IM3→IH를 달성한 후에.

이유: OPIc은 누적 역량 시험이다. 문단 구성(IH)을 못하면서 어휘 다양성(AL)을 훈련하는 것은 무의미하다. ACTFL 기준도 "하위 등급의 기능을 안정적으로 수행"해야 상위 등급을 부여한다.

## 2. 4-Tier 등급 갭 시스템

### 설계 근거

ACTFL Proficiency Guidelines 2024 + 강지완 OPIc AL 강의 22강 분석에서 도출한 **등급 간 결정적 차이**에 기반한다. AM(Advanced Mid 이상)은 OPIc 범위를 넘어서므로 제외한다.

### Tier 정의

| Tier | 목표 구간 | 현재 등급 | 핵심 전환 과제 | ACTFL 키워드 |
|------|-----------|-----------|---------------|-------------|
| **Tier 1** | → IL (Intermediate Low) | NL, NH | 문장 만들기, 기본 거래 수행 | "create with language", "sentences" |
| **Tier 2** | → IM1~IM2 | IL | 일관성, 발화량, 망설임 감소 | "strings of sentences", "some hesitation" |
| **Tier 3** | → IM3~IH | IM1, IM2 | **Skeleton Paragraph**, 시제 시도, 단락 수준 발화 | "paragraph-length discourse", "connected sentences" |
| **Tier 4** | → AL (Advanced Low) | IM3, IH | 지속적 문단 유지, 어휘 업그레이드, 돌발 대처 | "sustained paragraph", "narration/description in all time frames" |

### Tier 1: → IL (문장 만들기)

**현재 상태**: 단어 나열, 짧은 구, 암기 표현에 의존
**목표**: 주어+동사+목적어로 완전한 문장을 만들 수 있다

**ACTFL 근거**: Intermediate Low는 "creating with language"가 핵심. Novice와의 결정적 차이는 **외운 표현이 아닌 자기 문장을 만드는 능력**.

**훈련 초점**:
| # | 훈련 항목 | 설명 | 성공 기준 |
|---|-----------|------|-----------|
| 1 | 기본 문장 생성 | SVO 구조로 완전한 문장 만들기 | 30초 답변 내 완전한 문장 3개+ |
| 2 | 기본 거래 수행 | 질문 이해 → 관련 대답 | 과제 충족률 50%+ |
| 3 | 필수 표현 장착 | I like ~, I usually ~, There is ~ | 만능 패턴 5개 사용 |

**강지완 메소드 적용**: 이 티어에서는 강지완 강의의 "만능 템플릿"을 단순화한 **Basic Frame**(기본 틀)을 장착한다.
```
Basic Frame:
  I [동사] [목적어].
  I usually [동사] [빈도].
  I like it because [이유].
```

### Tier 2: → IM1~IM2 (일관성 + 발화량)

**현재 상태**: 문장은 만들지만 짧고, 주제 이탈이 잦고, 긴 침묵이 있다
**목표**: 연속 문장 5개+ 말할 수 있고, 주제를 벗어나지 않는다

**ACTFL 근거**: IM1~IM2는 "strings of sentences"가 핵심. IL과의 결정적 차이는 **여러 문장을 이어서 말하는 지속력**과 **주제 유지**.

**훈련 초점**:
| # | 훈련 항목 | 설명 | 성공 기준 |
|---|-----------|------|-----------|
| 1 | 발화량 확보 | 30초 이상 끊기지 않고 말하기 | 40초+ 연속 발화 (WPM 80+) |
| 2 | 주제 유지 | 질문에서 벗어나지 않기 | 관련 없는 문장 1개 이하 |
| 3 | 기본 연결어 | and, so, but, because 자연스러운 사용 | 연결어 3종+ 사용 |
| 4 | 만능 틀 장착 | 강지완식 4줄 틀 (도입-이유-예시-마무리) | 4줄 구조 완성 |

**강지완 메소드 적용**: "4줄 만능 틀" — 어떤 질문이든 4줄로 답할 수 있는 기본 구조.
```
4줄 만능 틀:
  ① 핵심 답변: I [동사] [주제].
  ② 이유/배경: Because [이유]. / The reason is [배경].
  ③ 구체적 예시: For example, [경험/사례].
  ④ 마무리: That's why I [재확인]. / So, [요약].
```

### Tier 3: → IM3~IH (Skeleton Paragraph + 시제)

**현재 상태**: 문장은 이어지지만 문단 구성이 없고, 과거/현재/미래 시제 전환이 불안정
**목표**: Topic Sentence → Supporting ×3 → Concluding의 **Skeleton Paragraph** 구조로 말할 수 있다

**ACTFL 근거**: IH는 "paragraph-length discourse"가 핵심. IM과의 결정적 차이는 **문장 나열이 아닌 문단 수준 발화**와 **시제 시도**. 강지완: "IM3는 문단이 보이기 시작하는 등급, IH는 안정적으로 문단을 유지하는 등급."

**훈련 초점**:
| # | 훈련 항목 | 설명 | 성공 기준 |
|---|-----------|------|-----------|
| 1 | **Skeleton Paragraph** | 도입-본론×3-마무리 구조 | 답변의 60%+에서 구조 확인 |
| 2 | 연결어 다양성 | however, moreover, on the other hand 등 | 답변당 연결어 4종+ |
| 3 | 시제 시도 | 과거/현재/미래 중 2개+ 사용 | 시제 전환 2회+ (오류 허용) |
| 4 | 마무리 문장 | 답변을 명확히 끝내기 | 마무리 있는 답변 80%+ |
| 5 | 디테일 추가 | 5감, 감정, 빈도 등 구체화 | 본론당 디테일 2개+ |

**강지완 메소드 적용**: **Skeleton Paragraph + 9대 연결어** — 이것이 IM→IH의 핵심 무기.
```
Skeleton Paragraph:
  ① Topic Sentence: "There are a few things I want to tell you about [주제]."
  ② Support 1: "First of all, [내용]. Moreover, [디테일]."
  ③ Support 2: "On top of that, [내용]. As a result, [결과]."
  ④ Support 3: "Last but not least, [내용]."
  ⑤ Concluding: "That is pretty much everything about [주제]."

9대 연결어 (강지완):
  나열: First of all / On top of that / Last but not least
  대조: However / On the other hand
  결과: As a result / Therefore
  추가: Moreover / In addition
```

**채점표 역공학 (강지완)**: IH 채점표에서 확인하는 것 = 문단 구조의 존재 여부. "문장이 아무리 좋아도 문단이 안 보이면 IM."

### Tier 4: → AL (문단 유지 + 어휘 + 돌발)

**현재 상태**: 문단 구조는 있지만 길어지면 붕괴, 어휘가 제한적, 돌발 상황에서 무너짐
**목표**: 2분 내내 문단을 유지하고, 돌발 상황을 성공적으로 처리한다

**ACTFL 근거**: AL은 "sustained paragraph-length discourse in all major time frames"가 핵심. IH와의 결정적 차이는 **문단의 지속성**, **시제 정확성**, **돌발 대처 능력**. 강지완: "IH는 문단을 시작할 수 있고, AL은 문단을 끝까지 유지할 수 있다."

**IH→AL의 3대 붕괴 유형 (ACTFL + 강지완)**:
| 붕괴 유형 | 설명 | 채점 영향 |
|-----------|------|-----------|
| **시제 붕괴** | 과거 이야기 중 현재형으로 전환 (went → go) | "시제를 시도하지만 유지하지 못함" → IH 판정 |
| **문단 붕괴** | 2분 답변 후반부에서 문장 나열로 퇴행 | "문단을 시작하지만 지속하지 못함" → IH 판정 |
| **어휘 축소** | 후반부로 갈수록 good, nice, fun만 반복 | "어휘 범위가 제한적" → IH 판정 |

**훈련 초점**:
| # | 훈련 항목 | 설명 | 성공 기준 |
|---|-----------|------|-----------|
| 1 | 시제 정확성 | went/go 혼동 제거, 일관된 시제 유지 | 시제 오류 2개 이하/2분 답변 |
| 2 | 문단 지속 | 2분 전체를 문단 구조로 유지 | 후반부 문단 붕괴 0회 |
| 3 | 어휘 업그레이드 | good→excellent, like→prefer, a lot→significantly | 반복 어휘 비율 15% 이하 |
| 4 | 돌발 대처 | 롤플레이 문제 해결 + 대안 제시 | 해결책+대안+후속조치 3요소 |
| 5 | 사회적 관점 | 찬반 양립 + 근거 제시 (비교변화/사회이슈) | 양측 관점 + 자기 입장 |

**강지완 메소드 적용**:
- **어휘 치환법**: 기본 어휘 → 고급 어휘 1:1 치환표 (좋다 → magnificent, excellent, outstanding)
- **시제 고정 연습**: 하나의 에피소드를 과거→현재→미래 3회 반복 (시제만 바꿔 말하기)
- **AL 4대 감점 항목** (강지완): Agreement(주어-동사 일치), Preposition(전치사 오류), Syntax(어순 오류), Breadth of Vocabulary(어휘 폭)
- **돌발 대처 3단 공식**: ① 상황 인식 + 사과 → ② 해결책 제시 → ③ 대안 + 후속 조치

## 3. 아키텍처: 3-Layer 시스템

```
Layer 1: 증거 수집 (모의고사 v2가 이미 완료)
  15문항 × { fulfillment, weak_points[{code,severity,reason,evidence}], transcript, speech_meta }
       │
       ▼
Layer 2: 병목 판별 (튜터링 v2 핵심 — 새로 구현)
  빈도 집계 → 심각도 가중 → 티어 필터 → 병목 스코어링
  출력: 병목 Top 1~3 (순위 + 점수 + 증거 문항번호 + 증거 발화)
       │
       ▼
Layer 3: 드릴 매칭 (하이브리드 — 카탈로그 + GPT 맞춤화)
  티어별 드릴 카탈로그 (뼈대, 하드코딩) + GPT (학생별 맞춤화)
  출력: 처방 1~3개 (타겟스킬 + 훈련방식 + 성공기준 + 증거인용)
```

### 불변 원칙

- ⛔ 기존 v1 코드 일절 수정하지 않음 (EF, 컴포넌트, SA, 타입, DB 테이블)
- ✅ v1 프로덕션은 그대로 운영
- ✅ 모의고사 v1 세션/음성/답변 데이터는 v2에서 읽기 가능
- ✅ v2 전용 파일/테이블 신규 생성 (v1과 완전 분리)

### 작업 순서

1. **weak_point 코드 사전 확정** (현재 단계) — 초안 작성 완료, GPT-5.4 자문으로 검증
2. **모의고사 v2 출력 규격 반영** — 코드 사전 기반으로 `모의고사-결과-v2.md` 프롬프트 설계에 반영
3. **드릴 카탈로그 상세 설계** — GPT-5.4 OPIc 전문가 자문으로 티어별 드릴 확정
4. **목 데이터 작성** — 확정된 구조를 하드코딩 JSON으로 (mt_8099015f 기반)
5. **UI 구현** — v2 컴포넌트, 라우트, 페이지 (진단서 스타일)
6. **병목 판별 알고리즘 구현** — Server Action 또는 유틸 함수
7. **EF 구현** — v2 전용 Edge Functions (드릴 생성, 훈련 평가)
8. **SA/타입 구현** — v2 전용 Server Actions, 타입
9. **실데이터 테스트** — 기존 세션으로 v1 vs v2 비교
10. **교체** — 만족 시 v2 → v1 대체

## 4. Layer 2: 병목 판별 알고리즘

### 입력 데이터 (모의고사 v2 평가 결과)

> **인터페이스 계약**: §12.6 `QuestionEvalV2ForTutoring` 참조
> **코드 사전**: §12.2 weak_point 코드 사전 v2 (36개) 참조

```typescript
// 15문항의 v2 개별 평가 데이터
interface QuestionEvalV2 {
  question_number: number;
  question_type: string;           // "description" | "routine" | "comparison" | ...
  fulfillment: FulfillmentStatus;  // "fulfilled" | "partial" | "unfulfilled" | "skipped"

  // v2: 표준화된 코드 + severity + evidence
  weak_points: Array<{
    code: string;                  // "WP_S03" (36개 코드 사전의 enum)
    severity: "severe" | "moderate" | "mild";
    reason: string;                // 병목 진단 표시용
    evidence: string;              // 학생 발화 인용
  }>;

  // ❌ recommended_drills 제거 (드릴 매칭은 병목 엔진이 담당)
  transcript: string;              // 실제 발화 텍스트
  speech_meta: {
    wpm: number;
    duration_sec: number;
    word_count: number;
    accuracy_score: number | null;
    fluency_score: number | null;
    prosody_score: number | null;
    pause_count_3s_plus: number;
    filler_ratio: number;
  };
}
```

### Step 1. 현재 티어 결정

```
session_grade + target_grade → 티어 자동 결정

매핑:
  현재 NL/NH  → Tier 1 (→IL)
  현재 IL     → Tier 2 (→IM1~IM2)
  현재 IM1/IM2 → Tier 3 (→IM3~IH)
  현재 IM3/IH → Tier 4 (→AL)

"다음 한 칸" 원칙:
  사용자 목표가 AL이어도, 현재 IM2면 Tier 3(→IM3~IH)부터 시작.
  Tier 3 목표 달성 후 → Tier 4로 진급.
```

### Step 2. 빈도 집계

```
15문항의 weak_points를 모두 펼쳐서 코드별 출현 횟수를 센다.
v2에서는 primary + secondary 모두 집계하되, secondary는 0.5 가중치.

예시 (mt_8099015f, 현재 IM3 → 목표 IH → Tier 3 적용):
  WP_S03 connector_poverty:      8회 (Q1,Q2,Q3,Q5,Q7,Q10,Q12,Q14)
  WP_S06 closure_missing:        3회 (Q5,Q7,Q14)
  WP_A08 timeframe_control_failed: 3회 (Q4,Q9,Q12)
  WP_C04 viewpoint_support_weak: 3회 (Q10,Q13,Q15)
  WP_C01 detail_shallow:         2회 (Q1,Q3)
  WP_T03 comparison_absent:      2회 (Q10,Q13)
  ...
```

### Step 3. 심각도 가중치

```
v2: weak_point 자체의 severity 필드를 사용한다.
(GPT가 문항 맥락을 보고 직접 판정 — 문항 fulfillment 기반보다 정확)

  severity = "severe"    → × 3.0  (해당 기능이 사실상 수행되지 않음)
  severity = "moderate"  → × 1.5  (시도는 있으나 불완전/불안정)
  severity = "mild"      → × 0.5  (약한 흔적이 일부 존재)
  skipped 문항           → 제외

이유: GPT가 transcript를 직접 보고 판정한 severity가
문항 fulfillment 기반 간접 추정보다 정확하다.
```

### Step 4. 티어 관련도 필터

```
§12.4 티어 관련도 매트릭스의 가중치를 적용한다.

각 weak_point 코드는 현재 티어에서의 가중치(0~1.0)가 정해져 있다.
  가중치 1.0 = 해당 티어의 핵심 병목 (필수)
  가중치 0.5 = 중요한 보조 병목
  가중치 0.2 = 참고 수준
  가중치 0   = 사실상 무관 → 스코어에서 제외

예시 (Tier 3 기준, 가중치 1.0인 코드들):
  WP_S03 connector_poverty, WP_S04 paragraph_absent,
  WP_S08 narrative_structure_weak, WP_S09 coherence_weak,
  WP_A02 tense_limited, WP_A08 timeframe_control_failed,
  WP_C01 detail_shallow

가중치 0인 비관련 약점은 스코어에서 제외하되, 기록은 유지한다.
```

### Step 5. 병목 스코어 계산

```
bottleneck_score = Σ(severity_weight × tier_relevance)  // 해당 코드가 나온 전 문항 합산

  severity_weight: severe=3.0, moderate=1.5, mild=0.5
  tier_relevance: §12.4 매트릭스에서 현재 티어의 가중치 (0~1.0)

예시 결과 (mt_8099015f, Tier 3):
  1위: WP_S03 connector_poverty — score 24.0 (8회 출현 × 높은 severity × Tier 3 가중치 1.0)
  2위: WP_S06 closure_missing  — score  7.5
  3위: WP_A08 timeframe_control_failed — score  6.0

출력: BottleneckResult[]
```

> **🆕 `thought_progression`**: WP_S09(coherence_weak) 전용. `connector_diversity`가 연결어 **종류 수**를 늘리는 드릴이라면, 이 드릴은 문장 간 **논리적 흐름**(thought flow)을 만드는 훈련. 연결어가 4종 이상이어도 논리 관계가 부자연스러우면 S09에 해당.
> - **훈련법**: Skeleton Paragraph 틀 위에 전환 사인포스트(First of all / On top of that / As a result) 배치 → 각 Support 사이 인과·대비·추가 관계 명시적으로 연결
> - **강지완**: "그냥 나열하지 마. First, 다음에 Moreover, 다음에 As a result — 이 순서가 이유가 있어야 해."

> **🆕 `timeframe_sustain`**: WP_A08(timeframe_control_failed) 전용. `time_frame_variety`가 "2개+ 시제를 시도하라"는 드릴이라면, 이 드릴은 "시작한 time frame을 끝까지 유지하라"는 훈련. A02(시제 미시도)와 정반대 — A08은 시도했으나 유지 실패.
> - **훈련법**: 하나의 에피소드를 과거로 시작했으면 끝까지 과거 유지 → 다음 에피소드에서 현재로 전환 → time frame 전환점을 의식적으로 표시
> - **강지완**: "시제 바꿔 끼기의 다음 단계. 바꿨으면 유지해야 해. 중간에 흔들리면 채점관이 '이 사람 control이 안 된다'고 판단해."

> **🆕 `reason_chain`**: WP_C07(reason_support_weak) 전용. `description_depth`가 감각·감정 디테일을 추가하는 드릴이라면, 이 드릴은 **왜(why)**에 대한 논증력을 키우는 훈련. adv_14(비교변화), adv_15(사회이슈)에서 특히 중요.
> - **훈련법**: "왜-왜-예시" 3단 공식 — ① 주장 ② 이유(because...) ③ 근거(for example...) → 순서대로 말하는 연습 3회
> - **강지완**: "채점관이 'why'를 물으면 'because + for example'이 무조건 나와야 해. 이유 없이 주장만 하면 content 점수 안 나와."

**Tier 3 핵심 교수법 (강지완)**:
1. **틀 먼저, 내용 나중에**: Skeleton Paragraph 구조를 먼저 체화한 후, 연결어·디테일을 채움
2. **바꿔 끼기**: 같은 틀에 다른 주제·연결어·시제를 끼워넣는 반복 훈련
3. **채점표 역공학**: "IH 채점관은 문단 구조를 본다" → 구조가 보이면 IH

> **🆕 `solution_proposal`**: WP_T08(solution_absent) 전용. `complication_handling`이 "돌발 상황 전체 대처"(상황인식→해결→대안)라면, 이 드릴은 **해결책 제시 자체**에 포커싱. RP12에서 문제는 잘 설명하지만 해결/대안이 빠지는 패턴에 특화.
> - **훈련법**: 문제 상황 제시 → "I think the best solution would be..." 시작 → 해결책 1개 + "If that doesn't work," + 대안 1개 + "After that, I would..." + 후속조치 — 3요소 반복
> - **강지완**: "문제 설명만 하고 끝내면 안 돼. 채점관은 '그래서 어쩌라고?'라고 생각해. 무조건 솔루션을 3개 말해."

**Tier 4 핵심 교수법 (강지완)**:
1. **3대 붕괴 방지**: 시제 붕괴·문단 붕괴·어휘 축소를 각각 집중 훈련
2. **어휘 치환법**: good→excellent, like→prefer, a lot→significantly (1:1 치환표)
3. **AL 4대 감점 인식**: Agreement, Preposition, Syntax, Breadth of Vocabulary

### 드릴 → 훈련 접근법 매핑 원리

약점의 성격에 따라 **가장 효과적인 접근법이 다르다**:

| 약점 성격 | 최적 접근법 | 이유 | 예시 드릴 |
|-----------|-----------|------|-----------|
| **구조 부재** | frame_install (틀 장착) | 틀 자체가 없어서 설치해야 함 | skeleton_paragraph, thought_progression, solution_proposal |
| **표현 부족** | swap_drill (바꿔 끼기) | 틀은 있는데 안에 넣을 표현이 부족 | connector_diversity, reason_chain, timeframe_sustain |
| **오류 습관** | self_correction (자기 교정) | 틀린 부분을 인지하고 고치는 반복 | tense_accuracy, agreement_accuracy |
| **속도/압박** | timed_pressure (시간 압박) | 여유 시 가능하지만 압박 시 무너짐 | paragraph_sustain, speech_volume |
| **패턴 암기** | pattern_drill (패턴 반복) | 특정 표현/공식을 외워야 함 | multi_part_checklist, negotiation_expressions |

**v1과의 차이**: v1은 모든 약점에 EPP→Var→Timed→Repair 순차 적용. v2는 **약점 유형에 맞는 접근법 + 티어에 맞는 난이도**로 조합한다.

## 7. ACTFL 평가 기준 통합

### 등급별 결정적 차이 (v2 병목 판별에 활용)

| 전환 | ACTFL 핵심 기준 | 채점관이 보는 것 | v2 매핑 |
|------|---------------|----------------|---------|
| NL/NH→IL | create with language | 외운 표현 vs 자기 문장 | Tier 1: sentence_formation |
| IL→IM | strings of sentences | 문장 연결 + 지속력 | Tier 2: speech_volume, basic_connectors |
| IM→IH | paragraph-length discourse | **문단 구조의 존재** | Tier 3: skeleton_paragraph |
| IH→AL | sustained paragraph in all time frames | **문단 지속 + 시제 정확 + 어휘 폭** | Tier 4: paragraph_sustain, tense_accuracy, vocabulary_upgrade |

### ACTFL FACT 4축과 티어 매핑

| FACT 축 | 설명 | Tier 1 | Tier 2 | Tier 3 | Tier 4 |
|---------|------|--------|--------|--------|--------|
| **Functions** | 언어 기능 (묘사/서술/비교) | 기본 거래 | 단순 묘사 | 묘사+서술 | 서술+비교+돌발 |
| **Accuracy** | 정확성 (문법/발음) | 의미 전달 | 기본 문법 | 시제 시도 | 시제 정확+AL 4대 감점 |
| **Content** | 내용 (주제 범위/깊이) | 개인 영역 | 일상 주제 | 확장 주제 | 사회적 이슈 |
| **Text Type** | 텍스트 유형 (단어→문장→문단) | 문장 | 문장 연결 | **문단** | **지속 문단** |

### 무응답/스킵 처리

모의고사 v2에서 이미 3축 무응답 감지를 수행 (`fulfillment: "skipped" | "unfulfilled"`).
튜터링 v2에서는:
- `skipped` 문항: 병목 분석에서 제외 (데이터 없음)
- `unfulfilled` 문항: 심각도 가중치 × 3.0 (이 약점이 답변을 망침)
- 전체 unfulfilled 비율 > 60%: Tier를 한 단계 낮춤 (과대평가 방지)

  Screen 2: 연습 3라운드 (9분) — 3개 주제로 Skeleton 말하기
  Screen 3: 성공 기준 체크 — "3회 중 2회에서 5줄 구조?"
  Screen 4: Recap — 결과 + 다음 처방 안내

처방 2 (connector_diversity) → swap_drill (바꿔 끼기):
  Screen 0: Brief — "오늘 목표: 연결어 4종+"
  Screen 1: 시범 — 9대 연결어 듣기 + 쉐도잉
  Screen 2: 연습 3라운드 — 같은 Skeleton에 연결어 바꿔 끼기
  Screen 3: 성공 기준 체크 — "3회 중 2회에서 4종 이상?"
  Screen 4: Recap — 결과

처방 3 (tense_accuracy, Tier 4) → self_correction (자기 교정):
  Screen 0: Brief — "오늘 목표: 시제 오류 2개 이하"
  Screen 1: 시범 — 같은 이야기 과거/현재/미래 3버전 듣기

  Screen 3: 성공 기준 체크 — "시제 오류 2개 이하?"
  Screen 4: Recap — 결과

## 12. 선행 과제: 모의고사 v2 인터페이스 계약

> **핵심**: 튜터링 v2(소비자)가 필요로 하는 데이터를 모의고사 v2(공급자)가 정확히 출력해야 한다.
> 모의고사 v2 EF/프롬프트 설계 전에 이 계약을 먼저 확정해야 한다.
> **상태**: ✅ GPT-5.4 OPIc 전문가 자문 완료 → v2 코드 사전 확정
> **태깅 프롬프트**: `docs/설계/weak-point-tagging-prompt.md` (별도 파일)

### 12.1. 현재 목 데이터의 문제점

현재 `eval-v2-result.json` (mt_8099015f)의 실제 출력을 분석한 결과:

| # | 문제 | 현재 상태 | 필요 상태 | 영향 |
|---|------|----------|----------|------|
| 1 | **weak_points 코드 혼재** | 영문 코드(`connector_diversity`)와 한글 서술(`시제 일관성 부족`)이 혼재 | **표준화된 코드만** 출력 | 병목 엔진 빈도 집계 불가 |
| 2 | **severity 필드 부재** | weak_points가 flat list (가중치 없음) | **약점별 심각도** (severe/moderate/mild) | 병목 스코어 = 빈도 × **심각도** × 등급관련도 |
| 3 | **grade_relevance 매핑 없음** | 약점이 어떤 등급에 영향 주는지 정보 없음 | **코드별 티어 관련도** 테이블 | "다음 한 칸"에 맞는 처방 불가 |
| 4 | **speech_meta 불완전** | Q2만 발음 점수 있고, Q3~Q15는 null | **전 문항 발음 평가** | Tier 4 발음/유창성 진단 불가 |
| 5 | **checklist 라벨 부재** | pass/fail boolean만 (항목명 없음) | **항목명 + pass/fail** | 증거 인용 ("시제 일관성 체크 미통과") 불가 |
| 6 | **drills ≈ weak_points 복사** | `recommended_drills`가 `weak_points`와 거의 동일 | **증상 코드 ↔ 드릴 코드 분리** | 증상(뭐가 부족) ≠ 처방(뭘 훈련) |

### 12.2. weak_point 코드 사전 v2 ✅ 확정

> **GPT-5.4 OPIc 전문가 자문 완료** (2026-03-20)
> v1 초안(31개) → 자문 피드백 반영 → **v2 확정(36개: S9+A8+C7+T8+D4)**
>
> **핵심 보정 원칙** (GPT-5.4 자문):
> - OPIc은 문법 체크 시험이 아니라, spontaneous speech를 대상으로 holistic rating
> - IM→IH 경계: "문장 나열" → "문단 길이의 connected discourse"

> - **병목 엔진은 micro-grammar보다 level ceiling 요인에 더 민감해야 함**

**설계 원칙**:
- 74개 체크박스를 그대로 쓰지 않음 (너무 세분화 → 빈도 집계 무의미)

- 각 코드에 **FACT 축 매핑 + 티어별 관련도** 부여
- 모의고사 v2 GPT 프롬프트에서 이 코드만 출력하도록 강제 (Structured Outputs)

### 12.6. 모의고사 v2 출력 규격 v2 ✅ 확정

모의고사 v2 EF가 문항별로 출력해야 할 데이터:

```typescript
// 모의고사 v2 → 튜터링 v2 인터페이스 계약 (v2 확정)
interface QuestionEvalV2ForTutoring {
  question_number: number;        // 2~15
  question_type: string;          // "description" | "routine" | ...
  fulfillment: FulfillmentStatus; // "fulfilled" | "partial" | "unfulfilled" | "skipped"

  // ✅ v2: 표준화된 코드 + severity + evidence (GPT Structured Outputs로 강제)
  weak_points: Array<{
    code: string;                 // "WP_S03" (36개 코드 사전의 코드만 허용, enum으로 강제)
    severity: "severe" | "moderate" | "mild";  // GPT가 판정 (병목 엔진 가중치에 사용)
    reason: string;               // "답변 전체에서 and, so 2종만 사용" (병목 진단 표시용)
    evidence: string;             // "and then... and then... 3회 반복" (학생 발화 인용)
  }>;

  // ✅ v2: 체크리스트 항목명 포함
  task_checklist: Array<{
    item: string;                 // "시제 전환 시도" (항목명)
    pass: boolean;
  }>;

  // ✅ v2: 발화 메타 (전 문항 필수)
  speech_meta: {
    duration_sec: number;
    wpm: number;
    word_count: number;
    accuracy_score: number | null;  // Azure 발음 평가
    fluency_score: number | null;
    prosody_score: number | null;
    pause_count_3s_plus: number;    // 3초+ 침묵 횟수
    filler_ratio: number;           // 필러 비율 (0.0~1.0)
  };

  transcript: string;             // STT 원문
  observation: string;            // 평가관 관찰 소견
  directions: string[];           // 개선 방향

  // ❌ recommended_drills 제거 — 드릴 매칭은 튜터링 v2 병목 엔진이 담당
}
```

**v1 인터페이스와의 차이**:

| 필드 | v1 (현재) | v2 (확정) | 변경 이유 |
|------|----------|----------|----------|
| `weak_points` | `string[]` (코드/한글 혼재) | `Array<{code, severity, reason, evidence}>` | 병목 엔진 입력 표준화 |
| `severity` | 없음 | `"severe" \| "moderate" \| "mild"` | 병목 스코어 = 빈도 × **심각도** × 티어관련도 |
| `task_checklist` | `Array<{pass}>` (항목명 없음) | `Array<{item, pass}>` | 증거 인용 가능 |
| `speech_meta` | 대부분 null | 전 문항 필수 + pause_count, filler_ratio 추가 | Delivery 태깅 근거 |
| `recommended_drills` | weak_points 복사 | **제거** | 증상(코드)과 처방(드릴) 분리 |

### 12.7. 병목 알고리즘 입력 구조 v2

v2 코드 사전에 맞게 병목 판별 알고리즘(§4)의 입력도 변경:

```typescript
// 기존 (v1)
weak_points: string[];  // ["connector_diversity", "tense_control"]

// 변경 (v2)
weak_points: Array<{
  code: string;       // "WP_S03"
  severity: string;   // "severe" | "moderate" | "mild"
  reason: string;
  evidence: string;
}>;
```

**심각도 가중치 변경**:
```
// 기존: 문항 fulfillment 기반
unfulfilled 문항에서 나온 약점 → × 3.0
partial 문항에서 나온 약점     → × 1.5
fulfilled 문항에서 나온 약점   → × 0.5

// 변경: weak_point 자체의 severity 사용 (GPT가 문항 맥락을 보고 직접 판정)
severity = "severe"    → × 3.0
severity = "moderate"  → × 1.5
severity = "mild"      → × 0.5
```

**병목 스코어 공식 (변경 없음, 입력만 변경)**:
```
bottleneck_score = Σ(severity_weight × tier_relevance)  // 전 문항 합산
```
