## E. 평가 트리거

### E-1. admin-trigger-eval 호출

```bash
# <SESSION_ID>를 Step D에서 출력된 세션 ID로 교체
curl -s -X POST "https://rwdsyqnrrpwkureqfxwb.supabase.co/functions/v1/admin-trigger-eval" \
  -H "Content-Type: application/json" \
  -d '{"session_id":"<SESSION_ID>","admin_key":"opictalk-eval-2026"}'
```

**정상 응답**:
```json
{"session_id":"mt_xxx","triggered":14,"message":"14개 답변에 대해 eval-judge fire-and-forget 호출 완료..."}
```

> `triggered`가 14보다 적으면 일부 답변의 `eval_status`가 이미 변경된 것.

### E-2. 작동 원리

```
admin-trigger-eval
  → stt_completed 상태 answers 조회
  → 각각 eval-judge fire-and-forget 호출 (EF→EF 내부 호출)
    → eval-judge 완료 → eval-coach 자동 체인 (fire-and-forget)
      → eval-coach 완료 → 모든 답변 완료 확인 → mock-test-report 자동 체인
        → report 완료 → holistic_status = 'completed'
```

전체 소요 시간: **1~3분** (GPT API 호출 14회 × 2단계 + 리포트 1회)

## 파이프라인 구조 참조

```
평가 파이프라인 (4-Stage):
  Stage A: mock-test-process   → Whisper STT + Azure 발음 (이관 시 스킵 — 소리담 데이터 재활용)
  Stage B-1: mock-test-eval-judge → GPT-4.1-mini 체크박스 평가 + 과제충족
  Stage B-2: mock-test-eval-coach → GPT-4.1 코칭 피드백
  Stage C: mock-test-report    → 규칙엔진 + FACT 점수 + GPT 종합 리포트

eval_status 흐름:
  pending → processing → stt_completed → evaluating → judge_completed → completed
  (Stage A)              ↑ 이관 시 여기서 시작    (B-1)         (B-2)
```
