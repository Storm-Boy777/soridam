## 0. 핵심 원칙 (개별평가 · 종합평가 · 튜터링 공통 참조)

> ⚠️ **이 원칙은 V2 개별평가, 종합평가, 튜터링 구현 시 반드시 참조해야 한다.**

### 평가의 목적

```
evaluation_criteria_v2 (60행 기준표)의 역할:

"목표 등급 + 유형 기준으로, 이 응시자의 답변이 목표에 도달했는가?"

→ 도달:   "충족. 어떤 점이 잘 되었는가" (소견)
→ 미달:   "미충족. 어디가 부족한가" (소견) + "뭘 해야 하는가" (방향) + "핵심 병목" (약점)
```

### 3단계 흐름: 평가 → 종합 → 튜터링

| 단계 | 모듈 | 역할 | 비유 |
|------|------|------|------|
| **1. 개별평가** | `diagnose-v2` + `consult-v2` | 목표 등급 기준으로 현재 답변의 수준을 진단 | 의사 진찰 |
| **2. 종합평가** | `report-v2` | 전체 문항의 진단 결과를 종합하여 등급 판정 + 종합 소견 | 진단서 |
| **3. 튜터링** | 튜터링 모듈 | 진단 결과를 바탕으로 목표 달성을 위한 훈련 제공 | 처방 + 치료 |

### 핵심 구분

- **평가(consult-v2)** = "목표 등급 기준으로 현재 수준이 어떤가?" → **진단**
- **튜터링** = "그 진단 결과를 바탕으로 목표 달성을 위해 뭘 해야 하는가?" → **처방 + 훈련**
- 평가는 **잣대(기준표)**로 판정하고, 튜터링은 **판정 결과**를 입력으로 받아 훈련을 설계한다
- 평가와 튜터링은 역할이 명확히 분리되어야 한다

### `evaluation_criteria_v2` 기준표의 위치

```
consult-v2 (개별평가):  기준표를 잣대로 → 소견/방향/약점 생성
report-v2 (종합평가):   개별평가 결과 집계 → 등급 판정 + 종합 소견
튜터링 (처방+훈련):     개별평가의 약점 + 종합평가의 등급 → 훈련 과제 설계
```

```
                         gpt-4.1                    gpt-4.1
```

## 6. 프롬프트 & 평가 기준 체계 (V2 확정)

### 6.1 프롬프트 테이블 (`evaluation_prompts_v2`)

| key | 용도 | 동적 주입 변수 |
|-----|------|---------------|

| `consult_v2` | B-2 소견 + 방향 + 약점 | `{criteria}`, `{diagnose_result}`, `{transcript}`, `{scores}` |

| `report_growth_v2` | C 성장 분석 | `{current_session}`, `{previous_session}` |

### 6.2 평가 기준 테이블 (`evaluation_criteria_v2`, 60행)

`script_specs`와 동일한 패턴. **등급 × 유형 조합으로 정밀한 평가 기준 제공.**
severity 판정 기준 포함 (같은 WP 코드라도 등급별로 severity가 달라짐).

### 6.3 WP 우선순위 테이블 (`wp_type_priority`, 10행)

### 6.4 consult-v2 실행 흐름

```
1. consult_v2 공통 프롬프트 로드 (evaluation_prompts_v2)
2. evaluation_criteria_v2에서 (target_level, question_type) 1행 로드
3. wp_type_priority에서 (question_type) 1행 로드
4. diagnose-v2 결과 로드 (체크박스 pass/fail + 과제충족)
5. 변수 치환:
   - {criteria}         → 해당 등급×유형 기준표 (severity 판정 기준 포함)
   - {wp_check_guide}   → 해당 유형의 WP 3-Tier 체크 목록 + 게이트키퍼 + 과잉 금지 규칙
   - {diagnose_result}  → 체크박스+과제충족 결과
   - {transcript}       → STT 텍스트
   - {scores}           → 발음/유창성 점수
6. GPT-4.1 호출
7. 출력: observation + directions[] + weak_points[]
```

### 6.5 등급 × 유형이 분리 불가능한 이유

같은 "연결어 부족" 현상도 등급과 유형에 따라 평가가 완전히 달라짐:

| 등급 + 유형 | 평가 |
|------------|------|
| IM2 + description | mild — "이 등급에서 연결어 다양성은 필수 아님" |
| IH + description | moderate — "단락 구성에 연결어 필요" |
| IM2 + roleplay_11 | 해당 없음 — "질문 유형이라 연결어보다 의문문 구조가 핵심" |
| IH + advanced_15 | severe — "논증 구조에서 연결어는 필수" |

→ **60행 조합 테이블이 최고 품질을 보장하는 유일한 방법**
