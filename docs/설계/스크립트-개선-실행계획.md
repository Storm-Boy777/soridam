## Phase 1: Pass 1 개선 (DB 데이터만 수정, 코드 변경 없음)

### Step 1-1. System Prompt 수정

**대상**: `ai_prompt_templates` WHERE `template_id = 'script_system'` — 1행

**변경 내용**:

| 구분 | 현재 | 개선 |
|------|------|------|
| ROLE | `certified ACTFL OPIc script coach (10+ years)` | `ACTFL-certified OPIc rater and speaking coach, focusing on spontaneous spoken fluency` |
| 원칙 | 4개 | 6개 (+TEXT TYPE CONTROL, +ANTI-MEMORIZATION TONE) |
| SELF-VALIDATION | 5개 | 7개 (+Tone check, +Text type check) |

**개선 System Prompt 전문**:
```
## ROLE

You are an ACTFL-certified OPIc rater and speaking coach,
focusing on spontaneous spoken fluency.
Your job is to generate responses that would be rated exactly
at the target proficiency level according to ACTFL guidelines.

Core principles:
1. LEVEL HONESTY — Never exceed the target level. A perfect IM2 script
   is better than an accidentally IH script.
2. SPOKEN ENGLISH — This is for a speaking test, not an essay.
   Use contractions, fillers, and conversational rhythm.
3. STORY PRESERVATION — Keep the learner's topics, details, and personality.
   Expand naturally, never replace.
4. SLOT FIDELITY — Each slot must fulfill its discourse function.
5. TEXT TYPE CONTROL — Each level must follow its ACTFL text type:
   IL: isolated words and simple sentences
   IM: strings of sentences with emerging elaboration
   IH: connected paragraphs with sustained narration
   AL: extended discourse with elaboration and abstraction
6. ANTI-MEMORIZATION TONE — Never sound rehearsed or essay-like.
   Include natural hesitation, self-correction (IH+), and
   conversational rhythm. Avoid mechanical connectors
   (First/Second/However/Therefore).

## CRITICAL RULE — LEVEL COMPLIANCE

You MUST strictly follow the LEVEL GATE in the user message.
The level constraints are absolute.
Never use grammar, vocabulary, or discourse markers from levels above the target.
A perfect script that stays within level boundaries is always better than
a script that accidentally uses higher-level expressions.

## CONTENT EXPANSION POLICY

Based on how much the learner provides:
- Level 1 (sufficient input): Restructure into slots, add natural details within their topic scope.
- Level 2 (partial input — keywords/fragments): Use their keywords as anchors, build slot content around them.
- Level 3 (no input): Generate a natural, relatable example using the question's topic.
In all cases: Follow level constraints and slot structure.

## KOREAN TRANSLATION

- Natural translation, not literal. Use casual polite style.
- 3-layer consistency:
  sentences[].korean → slot.translation_ko → full_text.korean
  Concatenating all sentence.korean in a slot must equal slot.translation_ko.

## SELF-VALIDATION (before output)

1. Level check: Any forbidden grammar/vocabulary for this level?
2. Filler check: Are fillers placed naturally (not just at sentence starts)?
3. Korean check: sentence.korean concatenation = slot.translation_ko?
4. Text check: All sentences concatenated with spaces = slot.text?
5. Structure check: Each slot fulfills its designated discourse function?
6. Tone check: Does this sound like spoken conversation, not a written essay?
7. Text type check: Does the output match the ACTFL text type for this level?
```

**SQL**: UPDATE 1건

### Step 1-2. 필러 지시 변경

**대상**: `script_specs` 50행 (IL 제외 전체) — `level_constraints` 컬럼

**변경 패턴** (각 등급별):

| 등급 | 현재 | 개선 |
|------|------|------|
| IM1 | `Well, Oh, And 등 기본 필러... 전체 스크립트에서 1-2회` | `Well, Oh 등을 슬롯 시작부에 자연스럽게 배치. 개수 제한 없이 자연스러운 호흡으로.` |
| IM2 | `... 전체 스크립트에서 1-2회` | `... 전환부에 자연스럽게 배치. 개수 제한 없이 대화 리듬에 맞게.` |
| IM3 | `... 전체 스크립트에서 2-3회` | `... 문장 시작과 전환부에 자연스럽게 배치. 개수 제한 없음.` |
| IH | `... 전체 스크립트에서 2-3회` / `3-5회` | `... 문장 시작뿐 아니라 문장 중간의 호흡에서도 자연스럽게 사용. 개수 제한 없음.` |
| AL | `... 전체 스크립트에서 3-4회` | `... 원어민처럼 문장 중간, 생각 전환, 강조 앞에서 자유롭게 사용. 개수 제한 없음.` |

**SQL**: 50행 UPDATE — 각 행의 level_constraints에서 필러 관련 문장만 REPLACE

### Step 1-3. 연결어 구어체화 + self-correction 추가

**대상**: `script_specs` WHERE `target_level IN ('IH', 'AL')` — 20행 `level_constraints`

**IH 추가 내용**:
```
## 금지 사항 추가
- 에세이형 연결어의 기계적 사용: First → Second → Also → However → Therefore
- In conclusion, To sum up, Overall 등 마무리 표현

## 허용 대체 표현
- To start off, Another thing is, Oh and, But you know what, The thing is
- 롤플레이: I was wondering if... / Oh, another thing... / By the way...

## 자연스러운 발화 장치
- Self-correction 1회 포함: "It was huge, I mean, completely massive"
- 감정/반응 표현 필수: 각 슬롯에 화자의 감정이나 태도 반영
```

**AL 추가 내용**: 위와 동일 (이미 에세이 구조 금지가 있으므로 대체 표현 + self-correction만 추가)

**SQL**: 20행 UPDATE

### Step 2-3. scripts 테이블 데이터 저장 방식

**현재 저장 구조**:
- `key_expressions` JSONB 컬럼: `string[]` (영어만)
- `paragraphs` JSONB 컬럼: Pass 1 결과 전체 + `key_expressions`, `reusable_patterns`, `connectors`, `fillers` 병합 저장

**개선 방안** (2가지 중 택 1):

**방안 A: paragraphs JSONB에 통합 저장 (기존 패턴 유지)**
- `paragraphs.learning_analysis` 하위에 7가지 콘텐츠 저장
- `key_expressions` 별도 컬럼은 하위 호환용으로 유지 (en 값만 추출하여 저장)
- DB 스키마 변경 없음

**방안 B: learning_analysis JSONB 컬럼 신설**
- `scripts` 테이블에 `learning_analysis JSONB` 컬럼 추가
- Pass 2 결과를 독립 저장
- 관심사 분리가 깔끔하나 마이그레이션 필요

**채택: 방안 A** — DB 스키마 변경 없이 진행. `paragraphs` JSONB 안에 저장.

**저장 형태**:
```json
{
  "paragraphs": [...],
  "full_text": {...},
  "word_count": 134,
  "structure_summary": [...],
  "key_sentences": [...],
  "key_expressions": [...],     // 새 형태: {en, ko, tip}[]
  "discourse_markers": [...],   // 연결어+필러 통합
  "reusable_patterns": [...],   // 기존 유지
  "similar_questions": [...],
  "expansion_ideas": [...]

```

**하위 호환**:
- `scripts.key_expressions` JSONB 컬럼: `en` 값만 추출하여 `string[]`로 저장 (기존 코드 호환)
- `paragraphs.connectors` / `paragraphs.fillers`: 삭제 (discourse_markers로 통합)

### Step 2-4. 프론트엔드 — ScriptSummaryView 전면 재설계

**대상**: `components/scripts/create/script-renderer.tsx`의 `ScriptSummaryView`

**현재 구조**: 3카테고리 하이라이트 (expression/connector/filler) + 만능 패턴 리스트

**개선 구조**: 7가지 콘텐츠를 섹션/아코디언으로 표시

```
1. 핵심 문장 (key_sentences)
   → 영어 문장 + "왜 중요한지" 한국어 설명
   → 클릭 시 스크립트에서 하이라이트

2. 뼈대 구조 (structure_summary)
   → tag + description 세로 리스트
   → 흐름 화살표로 시각화

3. 핵심 표현 (key_expressions)
   → en + ko + tip
   → 클릭 시 스크립트에서 하이라이트 (기존 기능 유지)

4. 담화 장치 (discourse_markers)
   → en + ko + function태그 + usage
   → 기능별 그룹핑 (나열/대조/시간벌기 등)
   → 클릭 시 스크립트에서 하이라이트

5. 만능 패턴 (reusable_patterns)
   → 기존 유지 (template + 설명 + 예문)

6. 유사 질문 (similar_questions)
   → question + reuse_hint

7. 확장 아이디어 (expansion_ideas) — IM3 이상만 표시
   → 한국어 아이디어 리스트
```

**하이라이트 시스템 변경**:
- 현재: `expression` / `connector` / `filler` 3카테고리
- 개선: `key_sentence` / `key_expression` / `discourse_marker` 3카테고리
- 하이라이트 로직 자체(`buildHighlightedSegments`)는 재사용 가능
