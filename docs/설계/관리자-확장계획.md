## 추가 필요 기능 목록

### 🔴 우선순위: 높음 (운영 필수)

#### A-1. 사용자 활동 로깅
- **로그인/로그아웃 이력 테이블** (`user_activity_log`)
  - user_id, action (login/logout/session_refresh), ip_address, user_agent, created_at
  - Supabase Auth 웹훅 또는 미들웨어에서 기록
- **관리자 조회 UI**: 사용자 상세 페이지에 "접속 이력" 탭 추가
- **대시보드**: 일별 접속자 추이 차트, 평균 세션 시간
- **구현 방식**: 미들웨어에서 세션 갱신 시 기록 (Auth 웹훅보다 간단)

#### A-2. 비활성 사용자 감지
- **기준**: 7일/14일/30일 미접속 사용자 자동 분류
- **대시보드 위젯**: 비활성 사용자 수 + 목록
- **알림**: 플랜 만료 임박 사용자 + 장기 미접속 사용자 목록
- **데이터 소스**: `last_sign_in_at` (Supabase Auth) + `user_activity_log`

### 🟡 우선순위: 중간

#### B-1. 공지사항 관리
- **DB 테이블**: `announcements` (title, content, type, target_audience, start_at, end_at, is_active)
- **관리자 페이지**: CRUD + 공개/비공개 토글 + 스케줄링
- **사용자 UI**: 대시보드 상단 배너 또는 모달 (최초 1회만 표시)
- **유형**: 서비스 공지, 점검 안내, 이벤트, 업데이트

#### B-2. 사용자 행동 퍼널 분석
- **모듈 전환 퍼널**: 가입 → 후기 → 스크립트 → 모의고사 → 튜터링
- **단계별 이탈률**: 각 모듈에서 다음 모듈로 넘어간 비율
- **코호트 분석**: 가입 월별 리텐션 (1주/2주/1달/2달)
- **데이터 소스**: 기존 테이블 집계 (신규 테이블 불필요)

#### B-3. WAU/MAU 지표
- **현재**: DAU만 구현 (`get_dau_count` RPC)
- **추가**: WAU (7일 활성), MAU (30일 활성) + 추이 차트
- **DAU/MAU 비율** (Stickiness): 높을수록 서비스 중독성 높음

### 🟢 우선순위: 낮음 (향후 고려)

#### C-1. 고객 지원 / 문의 관리
- 문의 접수 폼 (사용자) + 답변 관리 (관리자)
- FAQ CRUD
- 대안: 외부 서비스 (Zendesk, Channeltalk 등) 연동

#### C-2. 페이지 체류 시간 분석
- Google Analytics / Mixpanel로 대체 가능
- 자체 구현 시: `page_view_log` 테이블 + beforeunload 이벤트
- ROI 낮음 — 외부 도구 권장

#### C-3. 보안 강화
- 관리자 IP 화이트리스트
- 관리자 2FA (TOTP)
- 비정상 로그인 감지 (동시 접속, 해외 IP 등)
- 대안: Supabase Auth 자체 보안 설정으로 부분 대체

#### C-4. 마케팅 도구
- 프로모션 코드 / 쿠폰 관리
- UTM 파라미터 추적
- 리퍼럴 프로그램
- 이메일 캠페인 연동

#### C-5. 사용자 피드백
- NPS 설문 (분기별)
- 모듈별 만족도 평가
- 기능 요청 게시판
- 대안: Typeform, Google Forms 연동

## DB 스키마 설계 (Phase 1)

### user_activity_log 테이블
```sql
CREATE TABLE user_activity_log (
  id BIGSERIAL PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  action TEXT NOT NULL, -- 'login' | 'logout' | 'page_view' | 'module_use'
  metadata JSONB DEFAULT '{}', -- { page, module, ip, user_agent, ... }
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스
CREATE INDEX idx_activity_log_user ON user_activity_log(user_id, created_at DESC);
CREATE INDEX idx_activity_log_action ON user_activity_log(action, created_at DESC);

-- RLS
ALTER TABLE user_activity_log ENABLE ROW LEVEL SECURITY;
CREATE POLICY "admin_read" ON user_activity_log FOR SELECT USING (
  (SELECT role FROM auth.users WHERE id = auth.uid()) = 'admin'
  OR auth.uid() = user_id
);
```

### announcements 테이블 (Phase 2)
```sql
CREATE TABLE announcements (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  type TEXT NOT NULL DEFAULT 'notice', -- 'notice' | 'maintenance' | 'event' | 'update'
  target_audience TEXT DEFAULT 'all', -- 'all' | 'free' | 'paid'
  is_active BOOLEAN DEFAULT TRUE,
  start_at TIMESTAMPTZ DEFAULT NOW(),
  end_at TIMESTAMPTZ,
  created_by UUID REFERENCES auth.users(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
