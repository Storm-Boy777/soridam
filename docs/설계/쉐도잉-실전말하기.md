# 쉐도잉 실전 말하기 — 1문항 미니 모의고사 설계

> **목적**: 쉐도잉 Step 4(실전 말하기)를 모의고사와 동일한 UX로 재구현.
> 사용자가 훈련한 스크립트의 질문을 실제 시험처럼 응답하고, AI 평가를 받는다.

---

## 1. 현재 vs 목표

| 항목 | 현재 (step-speak) | 목표 (미니 모의고사) |
|------|------------------|-------------------|
| UI | 질문 카드 + 녹음 버튼 나열 | 모의고사 세션 UI (AVA + 볼륨 미터 + 타이머) |
| 질문 재생 | 수동 버튼 클릭 | 자동 재생 → 녹음 자동 시작 |
| 타이머 | 경과 시간 표시 | 카운트다운 (2분) |
| 평가 | scripts EF (evaluate) | mock-test 파이프라인 (process → consult) |
| 결과 UI | EvaluationResult (쉐도잉 전용) | TrainingEvalPanel (모의고사 동일) |
| 체크박스 | 없음 | 없음 (단일 문항이라 불필요) |

## 2. 아키텍처

### 2.1 컴포넌트 구조

```
StepSpeak (리뉴얼)
├── Phase: ready → playing → recording → uploading → evaluating → result
│
├── [ready]     질문 미리보기 + "시작" 버튼
├── [playing]   AVA 아바타 + 질문 오디오 자동 재생
├── [recording] 볼륨 미터 + 2분 카운트다운 + 정지 버튼
├── [uploading] 업로드 프로그레스
├── [evaluating] 평가 대기 (폴링)
└── [result]    TrainingEvalPanel 재사용
```

### 2.2 재사용 컴포넌트

| 컴포넌트 | 출처 | 수정 필요 |
|---------|------|---------|
| `AvaAvatar` | mock-exam/session/ | 없음 (그대로 재사용) |
| `TrainingEvalPanel` | mock-exam/session/ | props 조정 (sessionId → evaluationData) |
| `useRecorder` | lib/hooks/ | 없음 |
| `useQuestionPlayer` | lib/hooks/ | 없음 |

### 2.3 재사용하지 않는 것

| 항목 | 이유 |
|------|------|
| MockExamSession 전체 | 15문항 루프, 세션 관리 등 과도한 복잡성 |
| SessionTimer | 40분 전체 타이머 불필요 |
| QuestionGrid | 15문항 그리드 불필요 |
| mock-test-eval EF | 체크박스 74개 판정 — 단일 문항에 불필요 |
| mock-test-report EF | 종합 리포트 — 단일 문항에 불필요 |

## 3. 평가 파이프라인

### 3.1 플로우

```
[녹음 완료]
  → Supabase Storage 업로드 (shadowing-recordings)
  → Stage A: mock-test-process (Whisper STT + Azure 발음)
  → Stage B: 쉐도잉 전용 consult EF (체크박스 스킵, 소견/방향/WP만)
  → DB 저장 (shadowing_evaluations 확장 또는 별도 테이블)
  → 클라이언트 폴링 → 결과 표시
```

### 3.2 EF 옵션 비교

**옵션 A: mock-test-process + mock-test-consult 직접 호출**
- 장점: 기존 EF 재사용, STT + 발음 분석 그대로
- 단점: mock-test-consult는 체크박스 결과 의존 → eval 스킵 시 consult에 체크박스 데이터 없음
- 결론: **consult EF를 수정하거나 별도 EF 필요**

**옵션 B: 새 EF `shadowing-evaluate` 생성**
- mock-test-process(Stage A)로 STT + 발음 → 결과를 받아서
- 새 EF에서 consult 프롬프트만 실행 (체크박스 없이 소견/방향/WP)
- 장점: 기존 EF 수정 없음, 깔끔한 분리
- 단점: 프롬프트 중복 가능

**옵션 C: 기존 scripts EF의 evaluate 핸들러를 개선**
- 현재도 STT → GPT 평가하는 파이프라인 있음
- consult 스타일 출력 형식으로 변경
- 장점: 가장 적은 변경
- 단점: scripts EF가 이미 복잡

### 3.3 추천: 옵션 B (새 EF)

```
shadowing-speak-eval EF
├── 1. Whisper STT (재사용: mock-test-process의 로직)
├── 2. Azure 발음 평가 (재사용)
├── 3. GPT consult (evaluation_criteria 기준표 + transcript)
│   └── 출력: fulfillment, observation, directions[], weak_points[]
└── 4. DB 저장 (shadowing_evaluations 확장)
```

## 4. DB 변경

### 4.1 shadowing_evaluations 테이블 확장

현재 컬럼:
```sql
id, session_id, overall_score, fluency_score, pronunciation_score, 
intonation_score, vocabulary_score, opic_level, feedback, created_at
```

추가 컬럼 (consult 결과 저장):
```sql
transcript TEXT,                    -- STT 결과
wpm INTEGER,                        -- 분당 단어 수
pronunciation_assessment JSONB,     -- Azure 발음 상세
fulfillment TEXT,                   -- 과제 충족도 (완전/부분/미충족)
observation TEXT,                   -- 소견 관찰
directions TEXT[],                  -- 개선 방향 배열
weak_points JSONB,                  -- [{code, severity, reason, evidence}]
eval_status TEXT DEFAULT 'pending'  -- pending/processing/completed/failed
```

## 5. 클라이언트 플로우 상세

### 5.1 Phase 전환

```
ready → (시작 클릭) → playing → (오디오 끝) → recording → (정지/2분) → uploading → evaluating → result
```

### 5.2 각 Phase UI

**ready**:
- 질문 영/한 표시
- 크레딧 잔액 표시
- "실전처럼 답변하기" CTA 버튼

**playing**:
- AVA 아바타 (isSpeaking=true)
- 질문 오디오 자동 재생
- 준비 안내 텍스트

**recording**:
- AVA 아바타 (isListening=true)
- 볼륨 미터 (useRecorder의 volumeLevel)
- 2분 카운트다운 타이머
- 정지 버튼

**uploading**:
- 업로드 프로그레스
- "AI가 분석 중입니다" 안내

**evaluating**:
- 폴링 (5초 간격, eval_status 확인)
- 로딩 애니메이션

**result**:
- TrainingEvalPanel 스타일 결과 표시
- "다시 도전" / "쉐도잉으로 돌아가기" 버튼

## 6. 구현 순서

| Phase | 작업 | 예상 |
|-------|------|------|
| **P1** | DB 마이그레이션 (shadowing_evaluations 확장) | 소 |
| **P2** | EF: shadowing-speak-eval (STT + 발음 + consult) | 대 |
| **P3** | SA: 업로드 + 세션 생성 + 평가 트리거 + 폴링 | 중 |
| **P4** | step-speak.tsx 리뉴얼 (Phase UI + 재사용 컴포넌트) | 대 |
| **P5** | 테스트 + 프롬프트 조정 | 중 |

## 7. 결정 사항 (확정)

| 항목 | 결정 | 이유 |
|------|------|------|
| 크레딧 차감 | **녹음 제출 시** | 평가 실패해도 STT/GPT 비용 발생 |
| 평가 기준표 | **모의고사 evaluation_criteria 그대로** | 60행(6등급×10유형) 이미 구축됨 |
| 결과 저장 | **shadowing_evaluations 확장** | 이력 통합 관리 |
| STT/발음 로직 | **mock-test-process 로직 복사** | 기존 EF 수정 없이 독립 동작 |
| 평가 EF | **옵션 B: 새 EF `shadowing-speak-eval`** | 깔끔한 분리 |
| 체크박스 | **불필요** | 단일 문항이라 pass/fail 체크 의미 없음 |
