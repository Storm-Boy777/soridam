# AI 튜터링 커리큘럼 설계서

> **문서 목적**: 강지완 OPIc AL 강사 분석에서 얻은 보강 포인트 정리 — 기존 튜터링 구현(C→D→QSE→E→L1→F)의 **보조 참고 자료**
> **근거 자료**: `docs/설계/강지완-일타강사-수업분석.md` (v3)
> **관계**: 기존 `docs/설계/튜터링.md`의 **보강 포인트 문서**. 아키텍처 대체가 아님.
> **핵심 결론**: 현재 구현(병목 기반 개인화 파이프라인)의 방향이 맞다. 강사 분석의 가치는 이 아키텍처 안에서 **판정 범위 확장 + 등급별 난이도 조절 + 어휘 시스템 추가**에 있다.
> **작성일**: 2026-03-30

## 1. 설계 철학

### 1-1. 핵심 원리

> **"내용을 외우지 말고, 틀을 외워서 체점받도록 하라. 그리고 그 틀 안에서 어휘만 업그레이드하라."**

이 강사의 교수법을 AI로 옮기면 다음 세 가지로 요약된다:

| 원리 | 설명 | AI 구현 방향 |
|------|------|-------------|
| **구조 우선** | 채점은 내용이 아닌 구조(문단/연결/질문 패턴)로 등급을 나눈다 | 구조 감지 → 구조 부재 시 구조 교수 우선 |
| **Template + Slot** | 만능 템플릿에 주제별 키워드만 교체하면 모든 유형에 대응 | 템플릿 제시 → slot 채우기 → 전체 답변 → 판정 |
| **어휘 교체** | 같은 내용을 고급 어휘로 1~2개만 교체하면 등급이 오른다 | 답변 분석 → IH→AL 어휘 치환 제안 → 재발화 |

### 1-2. 인간 강사와 AI의 차이점 및 대응

| 인간 강사가 잘하는 것 | AI가 대신할 수 있는 방법 |
|--------------------|----------------------|
| 답변을 듣고 즉시 미세 오류 포착 | STT + 규칙 엔진(문법/구조 판정) + LLM(맥락적 판정) |
| 비유로 개념 설명 (힙합, 군대 구령) | 사전 정의된 교수 메시지 + 예시 음성 |
| 실시간 애드립 교정 | LLM 기반 교정문 생성 + 대안 표현 제시 |
| 학생 수준에 맞춘 즉각 난이도 조절 | Phase/Level 기반 자동 난이도 조절 |
| 억양/리듬 교정 | Azure Speech 발음 평가 + 강세 패턴 분석 (제한적) |
| 감정적 동기부여 ("파이팅!") | 격려 메시지 + 진행률 시각화 |

### 1-3. AI가 인간 강사보다 나을 수 있는 점

| 영역 | AI 장점 |
|------|---------|
| **일관성** | 매 세션 동일한 판정 기준 적용 |
| **반복 연습** | 24시간 무제한 재발화 연습 |
| **어휘 교정** | 즉시 AL 수준 교정본 생성 + diff 비교 |
| **진도 추적** | 오류 패턴 누적 분석, 취약점 자동 식별 |
| **비용** | 1:1 과외 대비 압도적 가성비 |

## 2. OPIc 채점 구조 해석

### 2-1. 등급 결정 메커니즘

강사의 교수법 + 하루오픽 평가 엔진 7-Step + OPIc 세부진단서(F1~F5)를 통합하면:

```
[Step 2: Floor] 모든 답변에서 문단 수준 일관 유지 (F2-05 Skeletal paragraphs)
  ├─ INT 통과율 < 65% → IL/IM1 (문장 나열)
  ├─ INT 통과율 75~95% → IM2 (연결 문장)
  └─ INT 통과율 ≥ 95% → IM3 (문단 구조) → IH 심사 진입
      │
[Step 5: Q12 게이트] roleplay_12 대안제시 (F4 Complication = OPIc 공식 "돌발")
  ├─ ADV-4 통과율 < 50% → 최대 IM3 (IH 불가)
  └─ ADV-4 통과율 ≥ 50% → IH 심사 자격
      │
[Step 3: Ceiling] 시제+어휘+문법의 ADV 체크박스 일관 유지 (F1 Time Frames + F3 Topic Variety)
  ├─ ADV 통과율 < 70% → BROKE_DOWN → IM 유지
  ├─ ADV 70~90% → 조건부 IH 가능
  └─ ADV ≥ 90% → IH 확정 + AL 심사 진입
      │
[Step 4: Sympathetic] 발음+이해용이성 (F5 Comprehensibility)
  ├─ 50%+ 성공 → IH 유지
  └─ 80%+ 성공 → AL 심사 자격
      │
[5차 분류] 공통형 주제 5~6문항 → 어휘 폭 + 사회적 대화
  ├─ 어휘 부족 / 구조 없음 → AL 감점
  └─ 폭넓은 어휘 + 패러그래프 → AL 감점 없음
      │
[6차 분류] 14번(비교변화) + 15번(이슈) → 토론 능력
  ├─ 스킵 / 답변 부실 → IH (14/15 없어도 IH 가능)
  └─ 토론 + 결론 + 고급 어휘 → AL 확정
```

### 2-2. 문항별 역할 매핑

| 문항 | 역할 | 측정 대상 | 등급 영향 |
|------|------|----------|----------|
| **1번** | 자기소개 | (참고만) | 미미 |
| **2, 5, 8번** | 현재 묘사 | **Skeleton Paragraph** | IM↔IH/AL 대분류 |
| **3, 6, 9번** | 선택 주제 확장 | 시제 + 내용 | IH 지지 |
| **4, 7, 10번** | 과거 시제 | **시제 정확도** | IH↔AL |
| **11번** | 롤플레이 질문 | **질문 수/복잡도** | IM3 관문 |
| **12번** | 롤플레이 문제해결 | **상황설명 + 해결책** | IH 관문 (12배 가중) |
| **13번** | 실제 경험 | 시제 | IH 지지 |
| **14번** | 비교/변화/대조 | **토론 + 어휘** | AL 전용 |
| **15번** | 이슈/뉴스 | **토론 + 어휘** | AL 전용 |

### 2-3. AL 전용 감점 항목

채점표에 표시되면 AL 불가. 표시 안 되면 AL 가능.

| 항목 | 내용 | 감점 원인 | 대응 |
|------|------|----------|------|
| **Agreement** | 수일치 | some question(s), one of the biggest issue(s) | 규칙 엔진으로 체크 |
| **Preposition** | 전치사 | problem about → with, on Friday | 규칙 엔진으로 체크 |
| **Syntax** | 문장 구조 단조 | SVO만 반복 | 분사구문(-ing) 사용 유도 |
| **Vocabulary** | 어휘 반복 | many/good/use 반복 | 어휘 치환 시스템 |
| **Breath of Vocabulary** | 어휘 폭 부족 | 주제별 고급 어휘 미사용 | Phase 4~5에서 집중 |
| **Connectedness** | 연결어 부족 | Transition words 미사용 | Phase 1에서 기본 확보 |

## 3. 6 Phase 커리큘럼

### Phase 0: 진단

**목표**: 학생의 현재 위치를 파악하고 어디서 시작할지 결정

**진단 방법**: 모의고사 데이터 분석 (기존 시스템의 Prompt C와 유사하되, 관점이 다름)

| 진단 항목 | 측정 방법 | 분기 |
|----------|----------|------|
| Skeleton Paragraph | 2/5/8번 답변에서 구조 감지 (Topic + 3 Sections + Conclusion) | 없음 → Phase 1 필수 |
| 롤플레이 질문 능력 | 11번 답변에서 질문 수, 꼬리질문, 복잡한 질문 존재 여부 | 부족 → Phase 2 필수 |
| 롤플레이 해결 능력 | 12번 답변에서 상황설명 충분성 + 해결책 수 | 부족 → Phase 2 필수 |
| 시제 정확도 | 과거 답변(4/7/10번)에서 시제 오류 비율 | 50% 미만 → Phase 3 필수 |
| 어휘 다양성 | 전체 답변에서 고빈도 단순 어휘(many, good, use) 비율 | 높음 → Phase 4 필수 |
| 14/15번 대응 | 비교변화/이슈 답변에서 토론 구조 + 결론 존재 여부 | 부재 → Phase 5 필수 |

**진단 결과 → 학습 경로 자동 설정:**

```
IM 계열 학생: Phase 1 → 2 → 3 (→ 4 → 5 선택)
IH 학생 (AL 목표): Phase 1 확인 → 4 → 5
IH 반복 학생: 진단에서 취약 Phase 식별 → 해당 Phase 집중
```

### Phase 1: Skeleton Paragraph (구조 확보)

**목표**: **모든 답변**에서 문단 구조(Skeleton Paragraph)를 일관되게 사용하는 능력 확보 → IH/AL 심사 자격 획득

**왜 가장 먼저인가:**
> "스켈레톤 패러그래프를 쓰면 반드시 IH와 어드밴스의 심사를 받게 된다"
> "2/5/8에서 문단을 측정하여 대분류를 한다. 문단을 못 하면 나머지 답변이 아무리 좋아도 성적에 반영되지 않는다"

#### 학습 내용

**258 구조 프레임:**

| 문항 | 시작 (Topic) | 전환어 | 본론 구분자 | 결론 | 마무리 |
|------|-------------|-------|-----------|------|--------|
| 2번 | To talk about [주제] | To get into more details | The first/second/last thing is that... | Overall, ... | That's about it |
| 5번 | Speaking of [주제] | Speaking of that | One/Another/The last thing is that... | The conclusion is that... | That's pretty much about it |
| 8번 | When it comes to [주제] | Speaking of which | The good/Another good/The best thing is that... | The bottom line is that... | That's all I can say |

**AL 강화 요소:**
- Topic Sentence를 **가치 판단이 담긴 문장**으로 강화
  - ❌ "I live in an apartment."
  - ✅ "Living in an apartment offers both convenience and a strong sense of community."
- Concluding Sentence에 **요약 + 평가** 포함
  - ❌ "That's about it."
  - ✅ "My apartment provides a comfortable and vibrant living environment that perfectly suits my needs. That's about it."

#### 드릴 구조

```
Step 1. 구조 학습
  → 258 프레임 설명 + 예시 음성 제시
  → 학생이 자기 선택 주제로 한글 키워드 3개 준비

Step 2. 2번 문항 연습
  → 프레임(To talk about / first-second-last / Overall) 보면서 녹음
  → AI 구조 판정: Topic? Transition? 3 Sections? Conclusion?
  → 부족 요소 교정 메시지 + 재발화

Step 3. 5번 문항 연습
  → 같은 주제를 5번 프레임(Speaking of / one-another-last / The conclusion is)으로
  → 판정 + 재발화

Step 4. 8번 문항 연습
  → 같은 주제를 8번 프레임(When it comes to / good-another good-best / The bottom line is)으로
  → 판정 + 재발화

Step 5. 다른 주제로 반복 (선택 주제 전체를 순회)
```

#### 판정 기준

| 요소 | 판정 방법 | Pass 기준 |
|------|----------|----------|
| Topic Sentence | "To talk about" / "Speaking of" / "When it comes to" 감지 | 존재 |
| Transition Word | "To get into more details" / "Speaking of that/which" 감지 | 1개 이상 |
| 3 Sections | "first/second/last" 또는 "one/another/last" 또는 "good/another good/best" | 3개 구분자 |
| Concluding Sentence | "Overall" / "The conclusion is" / "The bottom line is" 감지 | 존재 |
| AL Topic 강화 | Topic Sentence에 가치 판단 문장 존재 | (AL 목표 시) |
| AL Conclusion 강화 | 요약 문장 존재 | (AL 목표 시) |

#### 졸업 기준
- 선택 주제 3개 이상에서 258 구조 Pass
- 2/5/8번 각각 다른 마커 사용 확인

### Phase 2: 롤플레이 (IM3 → IH 심사 자격)

**목표**: 11번에서 IM3 확보 + 12번에서 IH 확정

**왜 구조 다음인가:**
> "모든 낮은 득점의 원인은 11번에서 기인한다"
> "12번은 12배 가중치. 공통형 주제과 시제 12문항이 12번 1문항과 동일한 비중"

#### 11번 (질문하기) 학습 내용

**핵심 원칙**: 3~4개 질문이 아닌 **8~10개의 연속적이고 복잡한 질문**

**질문 카테고리 시스템:**

| 카테고리 | 업체 예시 | 친구 예시 |
|---------|---------|---------|
| 종류 | What kinds of movies do you have? | What kind of movies do you like? |
| 시간 | When do you open/close? | What time is good for you? |
| 가격 | How much are they? | - |
| 결제 | Can I pay by credit card? | - |
| 할인 | Is there a discount if I use a certain card? | - |
| 위치 | Where is your location? | Where should we meet? |
| 주차 | Do you have a parking lot? Is it indoors? | - |
| 활동 | - | What do you want to do after? |
| 선호 | Do you have any recommendations that suit me? | Do you have any preferences you're in the mood for? |
| **만능 마무리** | Is there anything I need to prepare or make sure of? | Is there anything I have to bring? |

**복잡한 질문 패턴 (IM3→IH 필수):**
```
관계절:     ...that is newly released / ...that suits me
간접의문문: I was wondering if I could...
가정법:     Would it be possible to receive a discount if I use...?
꼬리질문:   Oh, you do. Then what about...?
```

**친구 문제 만능 전략**: 무조건 **만남 유도** → 시간/장소/활동 질문으로 수량 확보

#### 11번 드릴 구조

```
Step 1. 상황 제시 (업체 또는 친구)
Step 2. 학생 자유 답변 (녹음)
Step 3. AI 판정:
  - 질문 총 수량 (목표: 8개+)
  - 꼬리질문 존재 여부
  - 복잡한 질문 존재 여부 (that절, if절, 간접의문문)
  - 만능 마무리 사용 여부
Step 4. 부족 요소별 교정:
  - 수량 부족 → 카테고리 확장 안내
  - 꼬리질문 없음 → "Oh, you do. Then ~" 패턴 교수
  - 복잡한 질문 없음 → 만능 마무리 교수
Step 5. 재발화
```

#### 12번 (문제해결) 학습 내용

**4단계 구조:**
```
1. [도입]     I'm calling to let you know I have a problem, which is that...
2. [상황설명]  (충분한 디테일 — 짧으면 감점)
3. [당황]     I don't know what to do. What am I supposed to do?
4. [해결책]   2~3개 제시 + 만능 마무리
```

**AL 필수 표현:**
- `There must have been some misconception or miscommunication.` (상대 실수 완곡 표현)
- `You should have double-checked that.` (should have PP)
- `I was wondering if I could get a refund.` (간접의문문)
- 만능 마무리: `Do you have any other solutions you may come up with?`

**문제 유형별 해결책 패턴:**

| 유형 | 친구 해결책 | 업체 해결책 |
|------|-----------|-----------|
| 약속 늦음 | 시간 변경 / 대기 요청 / 카페 추천 | - |
| 약속 불가 | rain check / 다른 사람 추천 / 재조정 | - |
| 물건 파손 | 돈 보상 / 새 것 구매 / 수리 | - |
| 구매 문제 | - | exchange for / refund / return |
| 예약 문제 | - | 변경 / 취소 / 다른 옵션 |

#### 12번 드릴 구조

```
Step 1. 문제 상황 제시 (11번에 이어서)
Step 2. 학생 답변 (녹음)
Step 3. AI 판정:
  - 상황설명 충분성 (문장 수, 디테일 양)
  - 당황 표현 존재 여부
  - 해결책 개수 (2개 이상)
  - AL 표현 사용 여부 (should have PP, there must have been, I was wondering if)
Step 4. 교정 + AL 표현 주입
Step 5. 재발화
```

#### Phase 2 졸업 기준
- 11번: 질문 8개 이상 + 복잡한 질문 2개 이상 + 만능 마무리
- 12번: 상황설명 3문장 이상 + 당황 + 해결책 2개 이상

### Phase 3: 시제 Template (IH 확정)

**목표**: 과거 시제 6문항에서 정확도 확보 → IH 확정, AL 자격 접근

**왜 롤플레이 다음인가:**
> "시제가 여섯 문항이나 출제되는데 거의 다 잡아야 어드밴스가 된다"
> "내용의 길이보다 실수를 줄이는 것이 중요하다"

#### Template A: 어릴적 / first / child 문제

**출제 위치**: 4번, 6번, 9번 (7번에 나올 때도 있음). 1~2문항 출제.

```
When I was young, I used to [ACTIVITY].
  → Unlike now / Unlike my current [SITUATION],
    [어릴적 디테일 — 기억나는 것 위주]

As for now, [요즘 얘기 — 자유롭게]

Overall, I still remember the times when I had fun [ACTIVITY] as a child.
  I miss those days. I wish I could go back to those moments.
```

**Slot 교체 예시:**

| 주제 | ACTIVITY | 어릴적 디테일 | 요즘 |
|------|----------|-------------|------|
| 집 | live in a small house | 단독주택, 작은 마당 | 아파트, 편리 |
| 음악 | listen to children's songs | 동요, 부모님 영향 | K-pop, 다양한 장르 |
| 영화 | watch animation movies | 극장 처음, 무서웠음, 팝콘 | 모든 장르, OTT |
| 여행 | go to local places | 부모님과 가까운 곳 | 국내외, 혼자도 |
| 운동 | play outside | 공원, 친구들과 | 조깅, 헬스 |

#### Template B: 계기 / 변화 문제

**출제 위치**: 동일한 과거 위치. Template A가 이미 나왔을 때 사용.

```
The moment I first began [ACTIVITY] was a long long time ago.
  I can't recall the exact moment.
  However, I do remember the excitement from my childhood
    when I used to [ACTIVITY].

My parents introduced me to [TOPIC] during my early years,
  that significantly influenced my initial [TOPIC] preferences.

As I grew older, my passion for [TOPIC] grew.
  Being Korean, I developed a special affection for it
    because it was easily approachable.

Nowadays, I [현재 취향].
  I would say my [TOPIC] taste has developed positively.
```

#### 드릴 구조

```
Step 1. 과거 문제 제시 (선택 주제 기반)
Step 2. Template A 프레임 보면서 답변 (녹음)
Step 3. AI 판정:
  - 시제 전환 정확도 (과거↔현재 혼용 여부)
  - Template 구조 준수 (When I was young → Unlike now → As for now → 향수)
  - Skeleton Paragraph 유지 여부
Step 4. 시제 오류 교정 + 재발화

Step 5. 다른 주제로 Template A 반복
Step 6. 같은 주제로 Template B 연습 (변형 대응력)
```

#### 판정 기준

| 요소 | 판정 | Pass |
|------|------|------|
| 과거 시제 사용 | used to, was, had, went 등 과거형 | 과거 구간에서 80%+ 정확 |
| 현재 전환 | as for now, nowadays, these days, currently | 명확한 전환 마커 |
| 시제 혼용 오류 | 과거 구간에서 현재형 사용 / 반대 | 2개 이하 오류 |
| 향수 마무리 | I miss / I wish I could go back | (AL 목표 시) |

#### Phase 3 졸업 기준
- Template A + B 각 2개 주제 이상 완료
- 시제 전환 정확도 80% 이상

### Phase 4: 공통형 주제 + AL 어휘 (감점 제거)

**목표**: 공통형 주제 주제 대응력 + AL 감점 요소(어휘/문법) 제거

**왜 시제 다음인가:**
> "공통형 주제 문제는 어드밴스와 IH를 구별하기 위한 심사"
> "고른 주제가 아닌데도 이야기할 수 있어야 한다"

#### 공통형 주제 만능 구조

4가지 공통형 주제 카테고리에 공통 적용되는 구조:

```
[어렵다]   It's a tough question. I haven't thought about it.
           But I'll give my best shot.

[많다]     There are numerous [FACILITIES/THINGS] in Korea.
           Wherever you go, you can visually spot at least one.

[분류]     We refer to [TYPE A] and [TYPE B].
           What they do is provide [SERVICES].

[이용]     A large portion of the population frequents [PLACES]
           for various purposes.

[요즘]     It is often said that these days
           a large number of people use [ONLINE SERVICE]
           instead of physically visiting in person.

[자긍심]   When foreigners come to Korea,
           they are often fascinated by the sophisticated [SERVICE] services.
           I'm proud of that as a Korean.

[결론]     We are performing well based on the trend so far.
           We hope to achieve even greater success in the future.
```

**카테고리별 키워드:**

| 카테고리 | 주제 예시 | 주제별 필수 키워드 |
|---------|---------|-----------------|
| **시설** | 은행, 호텔, 식당, 교통 | central/private, financial products, hospitality, dining services |
| **환경** | 리사이클링, 온난화, 지리, 날씨 | separate waste, global warming, peninsula, four seasons |
| **산업** | 기술, 산업 | Samsung, 30% market share, foldable, high resolution |
| **개인** | 휴일, 모임, 자유시간, 약속 | Lunar New Year, gathering, Chuseok, quality time |

#### 어휘 업그레이드 시스템

**IH → AL 어휘 치환 테이블:**

| IH 표현 | AL 대체 표현 |
|---------|------------|
| many | numerous, countless, a large number of, a great deal of |
| see | spot, observe, visually spot |
| have | refer to, feature |
| use | take advantage of, utilize, make use of |
| people | a large portion of the population, individuals |
| good | high-level, sophisticated, highly advanced |
| do well | perform well based on the trend so far |
| do better | achieve even greater success |
| many reasons | for various purposes |
| get | obtain, acquire |
| find | navigate, locate, explore |
| go to | frequent, patronize |
| famous | widely recognized, gained worldwide fame |
| popular | booming, on the rise, trendy and sensational |
| surprised | fascinated, captivated |

**AI 교정 비교법 (핵심 기능):**

```
1. 학생 답변 transcript (STT)
2. AI가 동일 내용을 AL 수준 어휘로 교정한 버전 자동 생성
3. 원본 vs 교정본을 나란히 표시, 달라진 어휘를 하이라이트
4. 학생이 교정 어휘 중 1~2개를 선택적으로 채택
5. 채택한 어휘를 포함하여 전체 답변 재발화
```

> "내용의 길이는 크게 다르지 않아요. 어휘만 달라지는 거예요."
> "하나 넣으셔도 확률은 급 상승하게 됩니다."

#### 고급 문법 패턴 교수

| 패턴 | 예시 | 효과 |
|------|------|------|
| **분사구문 (-ing/면서)** | I watch movies eating popcorn | Syntax 감점 제거 |
| **가정법 과거** | If I had to choose one, it would be... | 1% 사용자 고급 시제 |
| **대절 구문** | It is commonly said that... / It is inevitably true that... | 문장 복잡성 |
| **-wise 패턴** | Location wise, ... but taste wise, ... | 어휘 다양성 |

#### 드릴 구조

```
Step 1. 랜덤 공통형 주제 주제 제시 (시설/환경/산업/개인)
Step 2. 학생 자유 답변 (녹음)
Step 3. AI 판정:
  - Skeleton Paragraph 구조 유지 여부
  - 어휘 수준 (IH 어휘 반복 횟수)
  - 문법 오류 (agreement, preposition, syntax)
  - 분사구문 사용 여부
Step 4. 교정 비교법 실행:
  - 원본 vs AL 교정본 diff 제시
  - 핵심 치환 1~2개 선택 안내
Step 5. 재발화 (선택한 어휘 포함)
Step 6. 다른 공통형 주제 주제로 반복
```

### Phase 5: 14/15번 AL 저격 (AL 확정)

**목표**: 14번(비교변화) + 15번(이슈) 답변으로 AL 확정

**왜 마지막인가:**
> "14/15번을 스킵해도 IH는 가능하다. 하지만 AL은 반드시 14/15를 잘 풀어야 한다"

#### 14번: 비교/변화/대조 — 스마트폰 만능 전략

**90%의 문제가 "옛날과 지금"** → 답: **스마트폰/기술 발전**

```
[Topic] There are both similarities and differences
        between the past and the present.

[과거]  When looking back at the past,
        we had to physically visit [PLACE] in person individually on sight.
        Which was a waste of time and energy,
        as they had to wait in long lines.

[전환]  However, with technological advancements,
        this is no longer the case.

[현재]  Now all you have to do is download an application
        on your smartphone and [ACTION] with just a few clicks.
        Without having to park your car or spend time on the road.

[결론]  Perhaps the most significant difference is that
        we have more options available today,
        all thanks to the invention of smartphones.
        What a world! How convenient it is!
```

**Slot 교체:**

| 주제 | PLACE | ACTION |
|------|-------|--------|
| 영화 | nearby theater | purchase tickets, watch via OTT |
| 여행 | travel agency | book flights and hotels |
| 쇼핑 | department store | order online |

| 음악 | music store | stream unlimited music |
| 식당 | restaurant | order food delivery |
| 운동 | gymnasium | follow online workout programs |

**10% 대조 문제 (두 대상 비교)** → 사람/장르/장소 비교 시 **극대화 전략**:
- 체격 큰 사람 vs 작은 사람
- 액션 영화 vs 로맨스 영화
- 도시 vs 시골
→ 공통: 구조는 동일, 내용만 슬롯 교체

#### 15번: 이슈/뉴스 — 경제 만능 전략

**모든 주제에 "돈/경제"로 연결** → 답: **인플레이션/경기침체**

```
        I hope this trend improves in the near future.
```

**주제별 연결:**

| 주제 | "Making [TOPIC] more challenging" 연결 |
|------|--------------------------------------|
| 영화 | Making watching movies at theaters more expensive |
| 여행 | Making traveling abroad more costly |
| 집 | Making housing more unaffordable |
| 음악 | Making attending concerts more expensive |
| 건강 | Making healthcare less accessible |
| 환경 | Making environmental protection less of a priority |

**AL 핵심 요소:**
- **답변 길이**: 90초 이상 = AL 황금 구간
- **토론 구조**: 주장 → 근거 → 반론 방어("It's not everything, but...") → 결론

## 4. 드릴 세션 설계

### 4-1. 단일 드릴의 재발화 사이클

모든 드릴은 동일한 **재발화 사이클**을 따른다:

```
┌─────────────────────────────────────────┐
│  1. 문제 제시 (음성 또는 텍스트)           │
│  2. 학생 자유 답변 (녹음)                  │
│  3. AI 분석 + 판정                        │
│     ├─ 구조 판정 (Skeleton, Template)     │
│     ├─ 문법 판정 (agreement, preposition) │
│     ├─ 어휘 판정 (IH/AL 수준)             │
│     └─ 내용 판정 (Phase별 필수 요소)       │
│  4. 피드백 제시                            │
│     ├─ Pass 요소: ✅ 체크리스트             │
│     ├─ 교정 요소: ❌ + 올바른 표현          │
│     └─ 업그레이드 제안: 💡 AL 대체 어휘     │
│  5. 학생 재발화 (교정 반영하여 다시 녹음)    │
│  6. 재판정 → Pass이면 다음 드릴로           │
│           → 2회 실패 시 상세 교정 (LLM)    │
└─────────────────────────────────────────┘
```

### 4-2. 세션 흐름 (1회 세션)

하나의 튜터링 세션은 다음과 같이 진행:

```
[세션 시작]
  → 이전 세션 복습 확인 (숙제 체크)
  → 오늘의 Phase/드릴 안내

[드릴 1] 문제 제시 → 답변 → 판정 → 교정 → 재발화
[드릴 2] 다른 문제 → 답변 → 판정 → 교정 → 재발화
[드릴 3] (선택) 취약 부분 보강 드릴

[세션 마무리]
  → 오늘의 핵심 정리 (교정된 표현 요약)
  → 다음 세션 과제 안내
  → 연습할 주제/표현 목록
```

### 4-3. 피드백 2단계 시스템

| 단계 | 조건 | 방법 | 응답 시간 |
|------|------|------|----------|
| **경량 판정** | 모든 답변 | 규칙 엔진 (구조 감지 + 마커 매칭 + 문법 체크) | 1~2초 |
| **상세 교정** | 2회 실패 / 학생 요청 / 복잡한 오류 | LLM (GPT-4.1) — 구체적 교정 + 모범 표현 + 재발화 지시 | 3~5초 |

**경량 판정이 커버하는 범위:**
- Skeleton Paragraph 구조 요소 존재 여부
- Template 핵심 마커 존재 여부 (When I was young, Unlike now, As for now 등)
- 질문 개수 (롤플레이)
- 수일치 오류 (some + 복수, one of the + 복수)
- 기본 시제 오류 (과거 구간에 현재형)
- IH 어휘 반복 횟수 (many, good, use 카운트)

**상세 교정이 필요한 경우:**
- 구조는 있지만 내용이 빈약할 때
- 복잡한 문법 오류 (시제 혼용이 맥락적일 때)
- 어휘 대체 제안이 맥락에 맞는지 확인이 필요할 때
- AI 교정 비교법 실행 시

## 5. Template 시스템

### 5-1. Template 설계 원칙

> "전부 외우는 건 불가능. 틀만 외워서 체점받도록"
> "여기 자리만 바꿔 끼면 됩니다"

**원칙:**
1. **하나의 Template으로 모든 주제 커버** — 주제별 답변을 따로 만들지 않음
2. **바꿔 끼는 Slot은 최소화** — 1~2개 단어만 교체
3. **구조(프레임)는 고정** — 매번 동일한 순서로 말함
4. **내용은 애드립** — 키워드만 준비하고 나머지는 자유
5. **암기 금지** — 읽어서 익숙해지는 것까지만

### 5-2. Template 목록

| ID | 이름 | 적용 문항 | 핵심 구조 |
|----|------|---------|----------|
| SKP-A | Skeleton 변형A | 콤보① description | To talk about → first/second/last → Overall |
| SKP-B | Skeleton 변형B | 콤보② description | Speaking of → one/another/last → The conclusion is |
| SKP-C | Skeleton 변형C | 콤보③ description | When it comes to → good/another good/best → The bottom line is |
| SKP-ALL | Skeleton 공통 | **모든 답변** | 위 3가지를 번갈아 사용하여 모든 유형(routine, comparison, 과거경험 등)에 적용 |
| RP-11-BIZ | 롤플레이 11 업체 | roleplay_11 | I'm calling to → 카테고리별 질문 → 만능 마무리 |
| RP-11-FRD | 롤플레이 11 친구 | roleplay_11 | Hey, what's up → 취향 → 시간 → 장소 → 활동 → 만능 마무리 |
| RP-12 | 롤플레이 12 (= OPIc 공식 "돌발") | roleplay_12 | Problem which is that → 상황설명 → 당황 → 해결책 → 만능 마무리 |
| PAST-A | 과거 어릴적 | past_experience 3종 | When I was young → Unlike now → As for now → I wish |
| PAST-B | 과거 계기 | past_experience 3종 | The moment I first → parents → grew older → positively |
| COMMON | 공통형 주제 만능 | 공통형 description/routine | 어렵다 → 많다 → 분류 → 이용 → 요즘 → 자긍심 → 결론 |

## 6. 판정(피드백) 시스템

### 6-1. 구조 판정 (모든 Phase 공통)

**Skeleton Paragraph 감지 규칙:**

```yaml
topic_sentence_markers:
  - "to talk about"
  - "speaking of"
  - "when it comes to"

transition_markers:
  - "to get into more details"
  - "to give you more details"
  - "speaking of that"
  - "speaking of which"

section_markers:
  group_2: ["the first thing", "the second thing", "the last thing"]
  group_5: ["one thing", "another thing", "the last thing"]
  group_8: ["the good thing", "another good thing", "the best thing"]

conclusion_markers:
  - "overall"
  - "the conclusion is"
  - "the bottom line is"

closing_markers:
  - "that's about it"
  - "that's pretty much about it"
  - "that's all I can say"
```

### 6-2. 문법 판정

```yaml
agreement_rules:
  - pattern: "some + singular_noun"
    correction: "some + plural_noun"
    example: "some question → some questions"

  - pattern: "one of the + singular"
    correction: "one of the + plural"
    example: "one of the biggest issue → issues"

  - pattern: "many/several/various + singular"
    correction: "+ plural"

  - pattern: "a large portion of population + plural_verb"
    correction: "+ singular_verb (goes, uses)"

preposition_rules:
  - "problem about → problem with"
  - "on 시간 → at 시간" (at 5pm, at noon)
  - "in Friday → on Friday"

syntax_check:
  - ing_usage: 분사구문 사용 여부 (Syntax 감점 방지)
  - that_clause: 관계절 사용 여부
  - if_clause: 조건절 사용 여부
```

### 6-3. 어휘 판정

```yaml
ih_vocabulary_flags:
  # 이 단어들이 3회 이상 반복되면 "어휘 폭 부족" 경고
  - many (→ numerous, countless, a large number of)
  - good (→ sophisticated, high-level, highly advanced)
  - use (→ take advantage of, utilize, make use of)
  - people (→ individuals, a large portion of the population)
  - see (→ spot, observe, visually spot)
  - find (→ navigate, locate, explore)
  - famous (→ widely recognized, gained worldwide fame)
  - go to (→ frequent, patronize)

al_bonus_expressions:
  # 이 표현이 감지되면 AL 가점
  - "there must have been"
  - "should have PP"
  - "I was wondering if"
  - "it is commonly said that"
  - "it is inevitably true that"
  - "if I had to choose one, it would be"
  - "regardless of age or gender"
  - "based on the trend so far"
  - "-wise" pattern (location wise, taste wise)
  - 분사구문 (-ing at sentence level)
```

### 6-4. 피드백 메시지 설계

**구조 피드백:**
```
✅ Topic Sentence — "To talk about my house" 잘 사용했어요
✅ 3 Sections — first/second/last 구분이 명확해요
❌ Concluding Sentence — 결론 없이 끝났어요
   → "Overall, my apartment provides..." 로 마무리해 보세요
```

**어휘 피드백:**
```
💡 "many" 를 3번 사용했어요. 하나를 바꿔볼까요?
   → "numerous" 또는 "countless" 로 교체하면 AL 어휘가 됩니다
   예: "There are numerous banks in Korea"
```

**문법 피드백:**
```
❌ "some question" → "some questions" (s 필요)
   → some 뒤에는 항상 복수 명사가 와요
```

**재발화 지시:**
```
이번에는 [교정 사항]을 반영해서 다시 말해보세요.
프레임은 그대로 두고, [특정 부분]만 바꿔서 녹음해 주세요.
```

## 8. 현재 구현에 대한 보강 포인트

> **핵심 결론**: 현재 구현(C→D→QSE→E→L1→F 파이프라인, 병목 기반 개인화)의 **방향이 맞다.**
> 강사 분석의 가치는 이 아키텍처를 **대체**하는 것이 아니라, **판정 범위를 넓히고 등급별 난이도를 조절**하는 데 있다.

### 8-1. 현재 구현의 방향이 맞는 이유

1. **병목은 학생마다 다르다** — 같은 IM2라도 어떤 학생은 roleplay_12가 문제고, 다른 학생은 시제가 문제. 등급으로 경로를 고정하면 안 됨. Prompt C가 학생 개인의 반복 패턴을 보고 병목을 찾으니까 개인화가 됨.
2. **병목 우선순위가 맞다** — foundation > type > skill > target-gap. 기초가 안 되면 고급 훈련은 무의미.
3. **3회 데이터로 습관 vs 실수를 구분한다** — D-2 결정이 정확히 이 목적.
4. **점진적 자립(Q1→Q2→Q3)이 맞다** — 같은 병목을 힌트 줄여가며 연습, 미니 재평가로 수행력 검증.

### 8-2. 보강 포인트 6가지

현재 구현에 **빠져 있는 것**을 강사 분석에서 보강:

#### 보강 1: Layer 1에 Skeleton Paragraph 구조 판정 추가

**현재**: Layer 1은 내용 마커(comparison: past/present/difference/reason)만 판정. "답변이 문단 구조인지"는 보지 않음.

**보강**: 모든 답변에서 Skeleton Paragraph 요소를 감지하는 규칙 추가

```yaml
skeleton_check:
  topic_markers: ["to talk about", "speaking of", "when it comes to"]
  transition_markers: ["to get into more details", "speaking of that", "speaking of which"]
  section_markers: ["first thing", "second thing", "last thing", "one thing", "another thing", "good thing", "best thing"]
  conclusion_markers: ["overall", "the conclusion is", "the bottom line is"]
  closing_markers: ["that's about it", "that's pretty much", "that's all I can say"]
```

**적용 위치**: `lib/tutoring/layer1-engine.ts` — evaluateLayer1 함수에 skeleton_check 추가
**대응 체크박스**: F2-05, ADV-2-SP5

#### 보강 2: Layer 1에 어휘 판정 추가

**현재**: 어휘 관련 판정이 전혀 없음.

**보강**: IH→AL 어휘 반복 카운트 + AL 대체어 제안

```yaml
vocabulary_check:
  # 3회 이상 반복 시 "어휘 폭 부족" 경고
  ih_repeats:
    - many → [numerous, countless, a large number of]
    - good → [sophisticated, high-level, highly advanced]
    - use → [take advantage of, utilize, make use of]
    - people → [individuals, a large portion of the population]
    - see → [spot, observe]
    - find → [navigate, locate, explore]
    - famous → [widely recognized, gained worldwide fame]

  # 감지 시 AL 가점
  al_bonus:
    - "there must have been"
    - "should have"
    - "I was wondering if"
    - "it is commonly said that"
    - "if I had to choose"
    - 분사구문 (-ing at sentence start)
```

**적용 위치**: `lib/tutoring/layer1-engine.ts` — 새 함수 `evaluateVocabulary()` 추가
**대응 체크박스**: ADV-3-V1, ADV-5-G2

#### 보강 3: Layer 1에 문법 판정 추가

**현재**: 수일치, 전치사, 구문 복잡성 판정 없음.

**보강**: AL 감점 항목(Agreement, Preposition, Syntax) 규칙 추가

```yaml
grammar_check:
  agreement:
    - "some + singular" → "some + plural"  (some question → questions)
    - "one of the + singular" → "one of the + plural"
    - "many/several + singular" → plural
  preposition:
    - "problem about" → "problem with"
    - "listen + 목적어" → "listen to"
  syntax:
    - 분사구문(-ing) 사용 여부 체크 → Syntax 감점 방지
    - 관계절(that/which/who) 사용 여부
```

**적용 위치**: `lib/tutoring/layer1-engine.ts` — 새 함수 `evaluateGrammar()` 추가
**대응 체크박스**: ADV-5-G2 (Syntax), ADV-5-G4 (Prepositions), ADV-5-G5 (Agreement)

#### 보강 4: Layer 2에 AI 교정 비교법 추가

**현재**: Layer 2(Prompt F)는 escalation 시 "칭찬 + 교정 + retry 지시".

**보강**: 학생 원본 vs AL 수준 교정본을 나란히 제시하는 모드 추가

```
학생 원본: "There are many banks in Korea. People go to the bank for many reasons."
AL 교정본: "There are numerous banks in Korea. A large portion of the population frequents banks for various purposes."
                     ^^^^^^^^                  ^^^^^^^^^^^^^^^^^^^^^^^^^^^         ^^^^^^^^^    ^^^^^^^^^^^^^^^
→ 달라진 어휘만 하이라이트 → "이 중 1~2개만 골라서 다시 말해보세요"
```

**적용 위치**: `supabase/functions/tutoring-evaluate` — Layer 2 분기에 "교정 비교 모드" 추가
**트리거**: 목표 등급이 AL인 학생 + ADV-3-V1 미통과 병목

#### 보강 5: Prompt E 드릴 생성 시 등급별 난이도 조절

**현재**: Prompt E가 `level_modifiers` 테이블의 등급별 설정을 참조하긴 하지만, frame 구조 자체는 동일.

**보강**: Prompt E에 등급별 Template 난이도 지시 추가

```
IM1~IM2 학생: "frame은 1~2 slot만. sample_answer는 3~4문장. 서론/마무리 포함."
IM3 학생: "frame은 2 Section. sample_answer는 5~6문장. 서론-본론(2개)-마무리."
IH 학생: "frame은 3 Section + Transition. sample_answer는 7~8문장."
AL 학생: "frame은 3 Section + Transition + 가치 판단 Topic. sample_answer에 AL 어휘 2개+, 분사구문 1개+ 포함."
```

**적용 위치**: `supabase/functions/tutoring-generate-drills` — Prompt E user 메시지에 등급별 지시 추가
**참조**: `level_modifiers` 테이블의 기존 데이터 활용

#### 보강 6: Prompt C 입력에 체크박스 통과율 추가

**현재**: Prompt C는 transcript + speech meta만 읽고 병목을 독자 판단. 기존 평가 체크박스는 의도적으로 제외(D-4 결정).

**보강**: 체크박스 **통과율 요약**을 Prompt C 입력에 **참고 신호**로 추가

```
기존 입력: final_level + transcript + speech meta
보강 입력: + 체크박스 통과율 요약 {
  "ADV-2-SP5": "3/3 통과 (Skeleton 안정)",
  "ADV-4": "0/3 통과 (Q12 게이트 반복 실패)",
  "ADV-3-V1": "1/3 통과 (어휘 폭 불안정)",
  "ADV-1-D3": "0/3 통과 (미래 시제 전무)"
}
```

Prompt C가 transcript를 직접 읽어 독자 판단하는 것은 유지하되, 체크박스 통과율을 **추가 참고 신호**로 제공하면 진단 정확도가 높아짐. 특히 "3회 연속 ADV-4 = 0%"는 "Q12 구조를 모른다"라는 명확한 신호.

> **주의**: D-4 결정("기존 consults의 weak_points는 사용하지 않는다")은 유지. weak_points가 아닌 **통과율 숫자**만 제공.

### 8-3. 보강 우선순위

시뮬레이션(mt_8099015f, mt_ea64ebca)에서 확인된 임팩트 순:

| 순위 | 보강 | 임팩트 | 난이도 |
