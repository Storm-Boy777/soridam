# 모의고사 성장 리포트 설계

> **상태**: 설계 초안 (GPT-5.4 OPIc 전문가 자문 기반)
> **작성일**: 2026-03-10
> **관련 문서**: `모의고사.md`, `모의고사-평가-v3.md`, `튜터링.md`

---

## 1. 개요

### 목적
- 사용자가 **자신의 성장을 한눈에 체감**하고
- **왜 등급이 올랐는지 / 안 올랐는지**를 이해하며
- **다음에 무엇을 훈련해야 하는지** 바로 행동으로 이어지게 하는 것

### 핵심 철학 (GPT 일타강사)
> "등급 추이 그래프는 단순 시각화가 아니라 **동기부여 장치**이고,
> 성장 리포트는 **자기 인식 + 처방 연결 장치**이다."

### 위치
- **등급 추이 그래프**: 모의고사 > "나의 이력" 탭 상단
- **성장 리포트**: 모의고사 > "결과" 탭 하단 (개별 세션 결과 아래 추가 섹션)

## 2. 등급 추이 그래프

### 2.1 3단 복합 뷰 구조

#### A. 상단: 예상 등급 계단 그래프
- **계단형(step chart)** — OPIc 등급은 연속값이 아닌 서열형이므로
- Y축: NH / IL / IM1 / IM2 / IM3 / IH / AL (7단계)
- X축: 응시 회차 (1회, 2회, 3회...)
- 각 점에 시험일 + 모드(훈련/실전) 표시
- 최근 3회 이동 평균선(점선) — 변동성 완화

#### B. 중단: 등급 내 준비도(Readiness) 바
- **가장 중요한 요소** — "맨날 IM인데 성장 안 하나?" 문제 해결
- total_score(0~100)를 현재 등급 구간 내 위치로 표현
- 프로그레스 바 형태: `[IM ====●====== IH]`
- 다음 등급까지의 거리 표시
- 문구 예:
  - "등급은 유지됐지만, IH 진입 가능성이 높아졌어요."
  - "현재 IM 상위권 수준입니다."

#### C. 하단: FACT 4영역 미니 추이
- 4색 라인 차트 (F: 파랑, A: 초록, C: 주황, T: 보라)
- 각 회차별 증감 표시: `F +8 / A +2 / C +1 / T +10`
- **병목 태그** 표시: "현재 병목: C" / "상승 동력: F"
- 레이더 차트는 "직전 회차 비교" 용도로만 별도 제공

### 2.2 데이터 소스
```
mock_test_reports: session_id, final_level, total_score, score_f, score_a, score_c, score_t
mock_test_sessions: started_at, mode
```
- 추가 DB 없이 기존 데이터로 구현 가능
- `total_score`가 준비도 바의 원천 데이터

### 2.3 OPIc 등급 체계 주의사항
1. **등급 간 난이도 불균등** — NH→IL과 IH→AL은 다른 차원. Y축 등간격이지만 실제 난이도는 아님을 인지
2. **예상 등급 변동성** — 주제 친숙도, 롤플레이 적응도, 컨디션에 따라 회차별 변동 큼. "예상 등급"임을 명시
3. **병목 영역이 등급을 결정** — 평균보다 최저 FACT가 중요. 병목 태그 필수

### 2.4 1회차 사용자 처리
- 그래프 대신 "현재 상태 카드" 표시
- 현재 등급 + FACT 프로필 + "다음 응시 후 성장 추이를 확인할 수 있습니다"

## 3. 성장 리포트

### 3.1 핵심 원칙
- 단순 숫자 나열이 아닌 **원인-결과 연결형 리포트**
- "왜 좋아졌는가" + "왜 아직 안 올랐는가" 둘 다 설명
- 리포트 끝에 반드시 **튜터링 처방 CTA** 연결

### 3.2 리포트 7개 섹션

#### ① 한 줄 요약
- 리포트 최상단, 가장 먼저 눈에 들어와야 함
- 예: "이번 회차는 T와 F가 개선되며 IM 안정권에 진입했습니다."
- 예: "등급은 유지됐지만 IH 진입까지의 거리는 줄었습니다."
- 예: "C 정체와 role-play 약세로 등급 상승은 제한됐습니다."

#### ② 결과 변화 표
| 항목 | 이전 | 현재 | 변화 |
|------|---:|---:|---:|
| 예상 등급 | IM2 | IM2 | 유지 |
| 준비도 | 54 | 71 | +17 ↑ |
| F 말하기흐름 | 62 | 71 | +9 ↑ |
| A 문법정확성 | 68 | 70 | +2 → |
| C 내용풍부도 | 58 | 60 | +2 → |
| T 질문수행력 | 55 | 69 | +14 ↑ |

#### ③ 등급 변화 원인 분석
- **상승 시**: 3개 상승 요인 명확히 (예: "질문 수행력 상승: 복수 요구사항 누락 감소")
- **유지/하락 시**: 정체 원인 분석 (예: "C 정체: 답변 길이는 늘었지만 디테일 부족")
- 학습자가 가장 궁금한 것: "나 분명 더 잘했는데 왜 등급 그대로지?" → 여기서 답을 줌

#### ④ FACT별 해석 코멘트
- 숫자가 아닌 **의미** 설명
- F: 머뭇거림 감소, 전개 자연스러움, 끊김/자기수정 빈도
- A: 시제 일관성, 문장 다양성, 반복 오류 재발
- C: 구체적 예시 수, 정보 밀도 (길이 ≠ 풍부도)
- T: 요구사항 누락, role-play 수행, 질문 의도 직접 응답

#### ⑤ question_type 비교
- 10가지 유형별 강/약/흔들림 분류
- 전 회차 대비 유형별 상승/하락
- "특정 유형 익숙함의 착시 vs 실제 실력 성장" 구분

#### ⑥ 병목 분석
- 다음 등급을 막는 1~2개 요인 명확히
- 예: "현재 가장 큰 병목은 C입니다."
- "학습자는 여러 약점을 한꺼번에 고치려다 실패. 가장 큰 병목 제거가 핵심"

#### ⑦ 추천 행동 3가지 + 튜터링 CTA
- **이번 주 반드시** / **다음 모의고사 전** / **시험장에서 주의**
- 각 병목에 1:1 매핑된 튜터링 훈련 버튼
  - "F 정체 → 15분 흐름 강화 훈련 시작"
  - "C 병목 → 예시·경험 확장 훈련 시작"
- "이 훈련은 다음 등급 진입에 가장 직접적인 영향을 줍니다" 문구

### 3.3 GPT 생성 vs 규칙 기반

| 섹션 | 방식 | 이유 |
|------|------|------|
| ① 한 줄 요약 | **GPT 생성** | 자연어 맥락 필요 |
| ② 결과 변화 표 | 규칙 기반 | 순수 데이터 계산 |
| ③ 등급 변화 원인 | **GPT 생성** | 원인-결과 해석 필요 |
| ④ FACT별 해석 | **GPT 생성** | 개별 체크박스 데이터 기반 해석 |
| ⑤ question_type 비교 | 규칙 기반 | 유형별 점수 집계 |
| ⑥ 병목 분석 | 규칙 + GPT | 최저 FACT 규칙 + 해석 |
| ⑦ 추천 행동 | **GPT 생성** | 맞춤형 액션 필요 |

### 3.4 GPT 호출 시점
- **mock-test-report EF**에서 종합 리포트 생성 시 함께 생성
- 이전 세션 데이터를 컨텍스트로 전달
- 1회차: 성장 리포트 미생성 (비교 대상 없음)
- 2회차부터: 직전 세션과 비교하여 생성

## 4. 응시 횟수별 리포트 깊이 차등

| 회차 | 리포트 이름 | 포커스 | 포함 섹션 |
|------|------------|--------|----------|
| 1회 | 기준선 진단 | 현재 수준 파악 | ②④⑤⑥ (비교 없이 현재 상태만) |
| 2회 | Before/After 비교 | 첫 변화 확인 | ①②③④⑤⑥⑦ 전체 |
| 3~4회 | 패턴 확인 | 반복 약점, 실제 성장 vs 착시 | + 3회 이동 추세, 반복 약점 강조 |
| 5회+ | 도약 전략 | 정체 돌파, 다음 등급 조건 | + 가장 오래 정체된 영역, 진입 조건 |

사용자가 "데이터가 쌓일수록 더 똑똑한 리포트를 받는다"는 경험

## 5. 실제 OPIc 성장 패턴 (리포트 반영용)

GPT 일타강사 자문 기반, 리포트에서 "현재 당신의 패턴은 ~입니다"로 안내

### 패턴 1: F 빠른 상승 + C 느린 상승
- 가장 흔함. 템플릿/연결어로 F는 빠르지만 C는 경험 축적 필요
- 안내: "현재는 자연스러운 초기 성장 패턴입니다. 지금부터는 '길게 말하기'보다 '구체적으로 말하기'가 중요합니다."

### 패턴 2: T 급상승 후 흔들림
- 질문 구조 이해하면 빠르지만 압박 시 다시 누락
- 안내: "T는 크게 개선됐지만 안정화가 필요합니다. 압박 환경에서 연습하세요."

### 패턴 3: A plateau (화석화)
- 특정 문법 오류가 고착되어 교정이 어려움
- 안내: "반복되는 문법 패턴을 집중 교정하면 돌파할 수 있습니다."

### 패턴 4: 등급 정체 + FACT 상승
- FACT는 오르는데 등급은 그대로 → 준비도 바로 설명
- 안내: "등급은 유지됐지만 다음 등급에 충분히 가까워졌습니다."

## 6. 튜터링 처방 연결 구조

### 리포트 → 튜터링 흐름
```
성장 리포트 확인 → 병목 인식 → 1순위 처방 CTA 클릭 → 튜터링 세션 시작
```

### FACT 기반 처방 분기
| 병목 FACT | 튜터링 처방 | 프로토콜 |
|-----------|-----------|---------|
| F 약함 | 연결어/chunking 훈련 | EPP + Timed Practice |
| A 약함 | 반복 오류 교정 | Oral Transformation |
| C 약함 | 예시·경험 확장 | Forced Variation |
| T 약함 | 질문 해체 + 체크리스트 | EPP + Self-repair |

### CTA 우선순위
- **1순위**: 등급 진입에 가장 직접적인 영향 (1개)
- **2순위**: 보조 개선 (1개)
- **3순위**: 유지 관리 (선택, 1개)

## 7. DB 변경 사항

### 기존 테이블 활용 (추가 테이블 불필요)
- `mock_test_reports`: FACT 점수, 등급, total_score → 추이 그래프 + 준비도
- `mock_test_sessions`: 날짜, 모드 → X축
- `mock_test_evaluations`: 문항별 평가 → question_type 비교

### mock_test_reports 컬럼 추가 (검토 필요)
```sql
-- 성장 리포트 GPT 생성 결과 저장
growth_summary TEXT,        -- ① 한 줄 요약
growth_analysis JSONB,      -- ③④⑥⑦ GPT 생성 분석 (원인/해석/병목/추천행동)
growth_comparison JSONB,    -- ② 이전 대비 변화 데이터 (규칙 기반 계산)
```

### growth_analysis JSONB 스키마 (초안)
```json
{
  "level_change_reason": "T와 F 개선으로 IM 안정권 진입",
  "fact_comments": {
```

## 8. 구현 Phase (검토 필요)

### Phase A: 등급 추이 그래프 (프론트엔드만)
- "나의 이력" 탭 상단에 3단 뷰 추가
- 기존 `mock_test_reports` 데이터만 사용
- 차트 라이브러리: Recharts (이미 번들에 포함 여부 확인)
- 준비도 바: total_score 기반 CSS 프로그레스 바

### Phase B: 성장 리포트 GPT 생성
- `mock-test-report` EF 확장: 이전 세션 데이터 포함하여 GPT 호출
- `mock_test_reports` 테이블에 growth 컬럼 추가
- 2회차부터 자동 생성

### Phase C: 결과 탭 UI
- 기존 ResultSummary 아래에 성장 리포트 섹션 추가
- 튜터링 CTA 버튼 연결

### Phase D: 응시 횟수별 차등 + 패턴 안내
- 회차에 따라 리포트 깊이 차등
- 성장 패턴 감지 + 안내 문구
