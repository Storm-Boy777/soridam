## 1. 배경 & 목표

- **대상**: `/study-group` = Talklish — 오프라인 영어 스터디 진행 화면(큰 모니터용). 월/수/금 운영, **월요일 = 팟캐스트 스터디**.
- **자료 영상**: 항상 **EnglishPod** 형식. 정형 10단계 구조:
  인트로 → Vocabulary Preview → **대화 1차** → Language Takeaway → Putting It Together → 대화 2차 → Fluency Builder → 대화 3차 → 잡담 → Audio Review
- **문제**: 어휘·표현 학습이 약했음 — `key_expressions`가 `{english, korean, example}` 3필드뿐, "표를 띄우는" 수준.
- **목표**: ① 자료 생성 자동화 (영문 스크립트 수동 복붙 → YouTube URL만) ② 어휘를 "훈련 카드"로 강화.

## 2. 핵심 설계 결정

| 항목 | 결정 |
|------|------|
| 자막 추출 | **Supadata API** — `GET https://api.supadata.ai/v1/transcript`, `x-api-key` 헤더, `text=false`(타임스탬프 포함). 무료 100건/월 → 소리담 사용량(월 4~8개)이면 비용 $0 |
| 탈락한 방식 | 레거시 captionTracks worker·`youtube-transcript` npm — YouTube **PO Token 정책**으로 막힘 (실제 테스트로 확인). `youtube-transcript-api`(Python)는 되지만 EF(Deno)에서 못 씀 |
| 대화 구간 | EF가 자막 타임스탬프로 `dialogue_segment {start_sec, end_sec}` 추출 → 학습 화면에서 YouTube `?start=&end=`로 **대화 1차 구간만 재생** (EnglishPod는 동일 대화를 3회 반복) |
| 어휘 훈련 카드 | 7필드: `expression`, `meaning_ko`(뜻+뉘앙스), `meaning_en`(영영정의), `examples[{en,ko}]`, `similar_expressions[]`, `speaking_prompt`, `level`("core"\|"stretch") |

## 4. ⚠️ 미해결 — 런타임 에러

```
Runtime TypeError: Cannot read properties of undefined (reading 'length')
components/study-group/talklish/podcast-stage.tsx (617) @ VocabBoard
  const e = card.examples.length > 0 ? Math.min(exIdx, card.examples.length - 1) : 0;
```

- **원인**: 새 `VocabBoard`는 `card.examples`(7필드 어휘 카드)를 기대하는데, 렌더 대상 자료의 `key_expressions`에 `examples`가 없음.
- **추정**: ① 기존 9개 `study_podcasts` 자료가 구버전 3필드 구조 / ② 또는 새 EF 미배포 상태라 자료가 옛 구조로 생성됨.
- **해결 방향**:
  1. **`VocabBoard` 방어 코드** — `card.examples ?? []`, `card.similar_expressions ?? []` 등으로 undefined 방어. 자료가 불완전해도 크래시 방지 (podcast-stage.tsx의 `VocabBoard`, 약 605행~).
  2. **근본 해결** — 새 EF 배포 확인 + 기존 9개 자료를 새 폼으로 재생성(구버전 폐기). 구버전 데이터엔 `expression`/`meaning_ko`도 없으므로 방어만으론 빈 카드가 됨.
