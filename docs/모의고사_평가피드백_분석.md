### 2-2. 개별 평가 프롬프트 구조 (eval_routine 대표)

**GPT에게 생성 요청하는 데이터:**
1. **checkboxes** (18~42개): question_type별 INT/ADV/AL 체크박스, 각각 pass/fail + 한국어 evidence
2. **sentences**: transcript를 문장별 분할 + group 번호 부여 (의미 맥락별 그루핑)
3. **corrections**: 교정 사항 (sentence_index, error_parts, tip_korean, corrected_segment)
4. **deep_analysis** (5섹션):
   - overall_assessment: 종합 피드백 (잘한 점 + 부족한 점)
   - linguistic_analysis: 언어 패턴 분석 (오류 유형 분류 + L1 간섭 원인)
   - communicative_effectiveness: 의사소통 효과 (전달된 내용 vs 흐려진 부분)
   - proficiency_gap: 레벨업 포인트 (현재 능력 → target_level 도달 요구 능력 + 예시 문장)
   - recommendation: 맞춤 연습 (이 답변의 오류에 직접 연결된 구체적 연습)

**주입되는 변수:**
- transcript, question_english, question_id, question_number
- topic_category, category, nickname, target_level
- word_count, wpm, filler_word_count, long_pause_count, audio_duration
- accuracy_score, prosody_score, fluency_score, mispronunciations
- checkbox_type, question_type

### 2-3. 종합 평가 프롬프트 구조 (eval_comprehensive)

**GPT에게 생성 요청하는 데이터:**
1. **overall_comments_ko**: 8-12문장 전문가 진단서 (등급별 ACTFL 템플릿 기반 커스터마이징)
2. **comprehensive_feedback** (5파트):
   - overall_assessment: 등급 맥락 + Floor/Ceiling 참조 (3-4문장)
   - strengths_analysis: 상위 3개 강점 (2-3문장)
   - weaknesses_analysis: 상위 3개 약점 패턴 (2-3문장)
   - gap_analysis: 현재 등급 → target_level 갭 (2-3문장)
   - action_plan: 3가지 우선 액션 아이템 (3-4문장)
3. **recurring_mistakes** (최대 5개): 14문항 교차 분석 반복 실수 패턴
   - pattern_name_ko, category, severity, frequency, affected_questions
   - description_ko, example_wrong, example_correct, improvement_tip_ko
4. **training_recommendations** (3개): 우선순위별 훈련 권장
   - category_ko, weak_areas_ko, practice_tip, recommended_type
   - model_sentence, tutoring_connection (tutor_type, session_focus, expected_improvement)

**주입되는 변수:**
- final_level, floor_status, floor_level, ceiling_status, sympathetic_listener
- int_pass_rate, adv_pass_rate, valid_question_count
- aggregated_int_checkboxes, aggregated_adv_checkboxes
- avg_fluency_score, avg_prosody_score, avg_accuracy_score
- question_summaries (14문항 요약)
- nickname, target_level, score_f/a/c/t, total_score

## 4. UI 표시 현황 — 문제점 분석

### 4-1. 개별 평가 UI (TrainingEvalPanel, 627줄)

**렌더링 순서:**
1. PassRateBar — 체크박스 통과율 (INT) 78%
2. TranscriptSection — 원본 transcript + 오디오 플레이어 + 메타(WPM, 시간, 필러, 침묵)
3. PronunciationSection — 3점수(정확도/유창성/운율) + 오발음 단어
4. CheckboxSection — 실패 항목(코드+evidence) / 통과 항목(접힘)
5. CorrectionsSection — error_parts(취소선) → corrected_segment + tip_korean
6. **DeepAnalysisSection — 기본 접힘(collapsed)** ← 핵심 문제

**문제점:**

| # | 문제 | 상세 | 영향도 |
|---|------|------|--------|
| A-1 | 심층 분석 기본 접힘 | `expanded=false` 기본값. 5섹션 피드백이 숨겨져 있음 | 치명적 |
| A-2 | 체크박스 코드 직접 노출 | `INT-1-1`, `INT-4-F2` 등 내부 코드를 font-mono로 표시 | 높음 |
| A-3 | 통과율이 최상단 | "체크박스 통과율 78%" — 사용자에게 비직관적 | 중간 |
| A-4 | sentences 데이터 미활용 | GPT가 문장별 분할+group 번호까지 생성하지만, 원본 transcript만 표시 | 높음 |
| A-5 | 교정-문장 연결 부재 | corrections.sentence_index 있지만, 해당 문장과 연결 표시 안 함 | 높음 |
| A-6 | 메타 정보 해석 없음 | WPM 76, 필러 2개 — 좋은지 나쁜지 해석 없이 수치만 나열 | 낮음 |

### 4-2. 종합 평가 UI (ResultSummary, 449줄)

**렌더링 순서:**
1. 등급 배지 + FACT 총점 + 이전 비교
2. FACT 영역별 점수 (F/A/C/T 바)
3. 영역별 분석 (INT/ADV/AL 통과율 + Floor/Ceiling)
4. 발음 평균 점수
5. 훈련 권장 (reason_ko + question_type + "스크립트 만들기" CTA)
6. **종합 피드백 — 기본 접힘(collapsed)** ← 핵심 문제
7. 문항별 상세 — 기본 접힘

**문제점:**

| # | 문제 | 상세 | 영향도 |
|---|------|------|--------|
| B-1 | 종합 피드백 기본 접힘 | overall_comments_ko + comprehensive_feedback 모두 접힘 | 치명적 |
| B-2 | comprehensive_feedback 미파싱 | JSON 5파트를 하나의 문자열로 표시 | 높음 |
| B-3 | recurring_mistakes 미저장/미표시 | 프롬프트가 생성하지만 EF에서 DB 저장 안 함 + UI 없음 | 높음 |
| B-4 | training_recommendations 단순화 | reason_ko + question_type만 표시. practice_tip, model_sentence 무시 | 높음 |
| B-5 | FACT 설명 부재 | F, A, C, T 알파벳만. 무엇을 의미하는지 설명 없음 | 중간 |
| B-6 | 내부 용어 직접 노출 | Sympathetic: ALWAYS_NEEDED, Floor: ESTABLISHED 등 ACTFL 영어 원문 | 중간 |

## 5. 개선 제안 (우선순위별)

### P1: 즉시 개선 (코드 수정만으로 큰 효과)

1. **심층 분석 기본 펼침 + 최상단 이동** — expanded 기본값 true, 렌더링 순서 변경
2. **종합 피드백 기본 펼침** — showFeedback 기본값 true
3. **체크박스 코드 → 한국어 라벨** — checkbox ID → 한국어 설명 매핑
4. **FACT 라벨 추가** — F: 유창성, A: 정확성, C: 내용, T: 과제수행
5. **내부 용어 한국어 변환** — Floor/Ceiling/Sympathetic → 한국어

### P2: 중요 개선 (UI 리디자인 + 데이터 활용)

6. **인라인 교정 뷰** — sentences + corrections 연결, 문장별 오류 하이라이트
7. **comprehensive_feedback 5섹션 파싱** — JSON 구조를 섹션별 카드로 표시
8. **training_recommendations 상세화** — practice_tip + model_sentence 표시
9. **WPM/필러 해석 추가** — 범위별 라벨 (느림/적절/빠름)

### P3: 고도화 (EF + DB 수정 필요)

10. **recurring_mistakes 저장 + 표시** — DB 컬럼 추가 + EF 저장 + UI 카드
11. **tutoring_connection 활용** — 튜터링 모듈 연결 CTA
12. **개별 평가 정보 계층 재설계** — overall_assessment → 교정 → 발음 → 상세 순서
