## 1. 프로젝트 개요

### 서비스 소개
- **서비스명**: 소리담 (Soridam) — AI 기반 OPIc 영어 말하기 학습 플랫폼
- **URL**: https://soridamhub.com
- **대상**: OPIc 시험 준비 한국인 학습자
- **철학**: 무료 플랫폼. 크레딧은 AI API 실비용 충당이지 수익 목적 아님. 개발자의 열정 프로젝트.

### 브랜드 메시지
> "소리담 덕분에 정말 큰 도움이 되었어요." — 이 한마디면 즐거움의 배움비용입니다.

- 사용자와 개발자가 직접 소통하는 **유일한 채널**
- 버그 신고, 기능 건의, 질문, 그리고 감사의 인사가 오가는 공간
- 상업적 고객 지원 센터가 아닌, **따뜻한 커뮤니티**

## 3. 소통함 구조

### 3개 탭
1. **공지사항** — 관리자가 작성하는 공지 (읽기 전용)
2. **게시판** — 사용자 피드백 보드 (건의/버그/질문)
3. **1:1 문의** — 비공개 문의 (본인만 열람)

### 게시판(피드백 보드) 기능
- **카테고리 필터**: 전체 / 🐛 버그 / 💡 건의 / ❓ 질문
- **정렬**: 최신순 / 공감순
- **글쓰기**: 카테고리 선택 + 제목 + 내용
- **목록 항목 정보**: 제목, 작성자 닉네임, 작성 시간, 카테고리, 공감 수, 댓글 수
- **상세 보기**: 제목, 작성자, 시간, 본문, 공감 버튼, 댓글 스레드
- **댓글**: 일반 사용자 댓글 + 관리자 답변 (관리자 답변은 구분 표시)
- **공감(Upvote)**: 하트 토글 방식

### 기술 스택
- Next.js App Router + TypeScript + Tailwind CSS
- TanStack React Query (데이터 페칭)
- Lucide React (아이콘)

## 4. 디자인 요구사항

### 핵심 방향
1. **다른 학습 페이지와 차별화** — 시험후기/스크립트/모의고사는 기능적·도구적. 소통함은 **감성적·관계적**이어야 함
2. **프리미엄 커뮤니티 보드 수준** — Canny, Linear, Notion 커뮤니티, GitHub Discussions 등 참고
3. **소리담 철학이 느껴지는 따뜻함** — 상업적 고객지원이 아닌, 열정 프로젝트의 소통 공간
4. **시각적 노이즈 최소화** — 선(border) 최소화, 여백과 타이포그래피로 위계 구분

### 화면별 요구사항

#### A. 전체 레이아웃
- 좁은 폭 (게시판 특성에 맞게)
- 헤더 영역에 소리담 철학/따뜻한 메시지 공간
- 탭 전환 네비게이션

#### B. 게시판 목록
- 글 목록은 심플하되 인터랙션 세련
- 정보 계층 명확: 제목 > 메타(작성자·시간·카테고리) > 공감/댓글
- 공감(Upvote) UI 처리 방식 제안 필요
- 카테고리 필터 + 정렬 + 글쓰기 CTA

#### C. 글쓰기 폼
- 인라인 또는 모달 형태
- 카테고리 선택 + 제목 + 내용 입력
- 세련된 입력 포커스 상태

#### D. 상세 보기
- 글 제목/본문의 가독성 극대화
- 공감 인터랙션
- 댓글 스레드 (관리자 답변 구분)

#### E. 댓글 섹션
- 관리자 답변 vs 일반 댓글 시각적 구분
- 줄바꿈 지원 입력창
- 가벼운 디자인 (채팅 버블 X)

### 하지 말아야 할 것
- ❌ 관리자 패널 같은 딱딱한 느낌
- ❌ 선(border)으로 모든 영역 구분
- ❌ 채팅앱 말풍선 스타일 댓글
- ❌ 너무 무거운 카드 디자인
- ❌ 학습 페이지와 동일한 톤
